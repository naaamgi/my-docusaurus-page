Security Onion

네트워크 보안 모니터링(NSM)을 위한 오픈소스 Linux 배포판. IDS, 로그 분석, PCAP 분석을 통합한 보안 운영 플랫폼.

개요​

Security Onion이란?​

항목설명
정식 명칭Security Onion
유형네트워크 보안 모니터링 (NSM) 플랫폼
라이선스GPL (오픈소스)
기반Ubuntu Linux
주요 용도침입 탐지, 네트워크 분석, 로그 관리, 위협 헌팅
공식 사이트https://securityonionsolutions.com/

핵심 구성 요소​

1. Suricata (IDS/IPS)​

시그니처 기반 침입 탐지 시스템:

# 규칙 위치
/opt/so/rules/suricata/

# 규칙 업데이트
sudo so-rule-update

# Suricata 로그
/opt/so/log/suricata/

# 커스텀 규칙 추가
sudo nano /opt/so/saltstack/local/pillar/minions/<hostname>.sls

Suricata 규칙 예시​

# 예: SSH 브루트포스 탐지
alert tcp any any -> any 22 (msg:"Possible SSH Brute Force"; \
  flow:to_server; threshold:type both, track by_src, count 5, seconds 60; \
  sid:1000001; rev:1;)

# 예: SQL Injection 탐지
alert http any any -> any any (msg:"SQL Injection Attempt"; \
  content:"union"; nocase; content:"select"; nocase; \
  sid:1000002; rev:1;)

2. Zeek (네트워크 분석)​

네트워크 메타데이터 추출:

# Zeek 로그 위치
/opt/so/log/zeek/

# 주요 로그 파일
conn.log      # 연결 로그
http.log      # HTTP 트래픽
dns.log       # DNS 쿼리
ssl.log       # SSL/TLS 연결
files.log     # 파일 전송
notice.log    # 경고/알림

Zeek 로그 필드 (conn.log)​

필드설명
ts타임스탬프
uid연결 고유 ID
id.orig_h출발지 IP
id.orig_p출발지 포트
id.resp_h목적지 IP
id.resp_p목적지 포트
proto프로토콜
duration연결 지속시간
orig_bytes전송 바이트
resp_bytes수신 바이트

Hunt 인터페이스​

위협 헌팅을 위한 검색 인터페이스:

# 기본 검색
event.module:suricata AND event.severity:1

# IP 검색
source.ip:192.168.1.100 OR destination.ip:192.168.1.100

# 시간 범위 지정
@timestamp:[now-24h TO now]

# 특정 포트 검색
destination.port:445

# 프로토콜 검색
network.protocol:http

검색 쿼리​

Elasticsearch Query DSL​

기본 검색​

{
  "query": {
    "match": {
      "event.module": "suricata"
    }
  }
}

복합 검색​

{
  "query": {
    "bool": {
      "must": [
        { "match": { "event.module": "zeek" }},
        { "match": { "network.protocol": "http" }}
      ],
      "filter": [
        { "range": { "@timestamp": { "gte": "now-1h" }}}
      ]
    }
  }
}

자주 사용하는 검색어​

Suricata 알림 검색​

# 모든 알림
event.module:suricata AND event.type:alert

# 높은 심각도 알림
event.module:suricata AND event.severity:1

# 특정 시그니처
suricata.eve.alert.signature:"ET MALWARE"

# 특정 카테고리
suricata.eve.alert.category:"Attempted Administrator Privilege Gain"

Zeek 로그 검색​

# HTTP 트래픽
event.module:zeek AND event.dataset:zeek.http

# DNS 쿼리
event.module:zeek AND event.dataset:zeek.dns

# SSL/TLS
event.module:zeek AND event.dataset:zeek.ssl

# 파일 다운로드
event.module:zeek AND event.dataset:zeek.files

# 특정 도메인
dns.question.name:*.malware.com

네트워크 검색​

# 특정 IP의 모든 트래픽
source.ip:10.0.0.50 OR destination.ip:10.0.0.50

# 외부 IP로의 연결
NOT destination.ip:10.0.0.0/8 AND NOT destination.ip:192.168.0.0/16

# 비표준 포트 HTTP
destination.port:8080 AND network.protocol:http

# 대용량 전송
source.bytes:>1000000

위협 헌팅 시나리오​

1. 비콘 탐지 (C2 Communication)​

# 주기적인 연결 패턴 검색
event.module:zeek AND event.dataset:zeek.conn

# 분석 포인트
- 동일 목적지로 일정 간격 연결
- 작은 크기의 규칙적인 데이터 전송
- 비표준 포트 사용

2. 내부 정찰 (Lateral Movement)​

# SMB 스캔
destination.port:445 AND event.module:zeek

# RDP 스캔
destination.port:3389

# 다수 호스트 접근 시도
source.ip:X.X.X.X AND destination.port:(22 OR 445 OR 3389)

3. 데이터 유출 (Exfiltration)​

# 대용량 아웃바운드 트래픽
source.bytes:>10000000 AND NOT destination.ip:10.0.0.0/8

# DNS 터널링 의심
event.dataset:zeek.dns AND dns.question.name:*.*.*.*

# 비정상 시간대 트래픽
@timestamp:[2024-01-01T22:00:00 TO 2024-01-02T06:00:00]

4. 악성코드 다운로드​

# 실행 파일 다운로드
event.dataset:zeek.files AND file.mime_type:"application/x-dosexec"

# 의심스러운 User-Agent
http.request.user_agent:*curl* OR http.request.user_agent:*wget*

# 비표준 포트 HTTP
network.protocol:http AND NOT destination.port:80

트러블슈팅 체크리스트​

웹 인터페이스 접속 불가​

  • sudo so-status로 서비스 상태 확인
  • sudo so-restart로 서비스 재시작
  • 방화벽에서 443 포트 허용 확인
  • /opt/so/log/ 디렉토리에서 에러 로그 확인

알림이 생성되지 않음​

  • Suricata 컨테이너 상태 확인
  • 네트워크 인터페이스가 올바르게 설정되었는지 확인
  • 규칙이 활성화되어 있는지 확인
  • 트래픽이 센서를 통과하는지 확인 (미러링/TAP)

Elasticsearch 문제​

  • 디스크 공간 확인 (최소 15% 여유 필요)
  • 클러스터 상태 확인 (_cluster/health)
  • JVM 힙 메모리 확인
  • 노드 간 통신 확인

성능 문제​

  • CPU/메모리 사용량 확인
  • 디스크 I/O 확인
  • Elasticsearch 쿼리 최적화
  • 불필요한 로그 비활성화

센서가 트래픽을 수집하지 않음​

  • 모니터링 인터페이스가 프로미스큐어스 모드인지 확인
  • 네트워크 TAP/미러링 설정 확인
  • Zeek/Suricata 컨테이너 상태 확인
  • tcpdump -i <interface> 로 트래픽 확인
