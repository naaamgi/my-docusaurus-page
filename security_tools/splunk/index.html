Splunk

빅데이터 기반 로그 수집, 분석, 시각화 플랫폼. 보안 모니터링(SIEM), IT 운영, 비즈니스 분석에 활용되는 엔터프라이즈급 솔루션.

개요​

Splunk란?​

항목설명
정식 명칭Splunk Enterprise / Splunk Cloud
유형SIEM, 로그 분석, 빅데이터 플랫폼
라이선스상용 (무료 버전: 일 500MB)
개발사Splunk Inc. (Cisco 인수)
주요 용도보안 모니터링, 로그 분석, 위협 탐지
공식 사이트https://www.splunk.com/

제품 라인업​

제품용도특징
Splunk Enterprise온프레미스자체 인프라 운영
Splunk Cloud클라우드SaaS 형태
Splunk Free학습/테스트일 500MB 제한
Splunk Enterprise SecuritySIEM보안 특화 앱
Splunk SOAR자동화보안 오케스트레이션

설치​

시스템 요구사항​

구성최소 사양권장 사양
단일 서버8GB RAM, 4 CPU16GB RAM, 8 CPU
분산 환경Indexer: 16GB RAMSearch Head: 32GB RAM
디스크SSD 권장RAID 구성

Linux 설치​

# RPM 설치 (RHEL/CentOS)
wget -O splunk.rpm "https://download.splunk.com/products/splunk/releases/<version>/linux/splunk-<version>-linux-2.6-x86_64.rpm"
sudo rpm -i splunk.rpm

# DEB 설치 (Ubuntu/Debian)
wget -O splunk.deb "https://download.splunk.com/products/splunk/releases/<version>/linux/splunk-<version>-linux-2.6-amd64.deb"
sudo dpkg -i splunk.deb

# TGZ 설치
wget -O splunk.tgz "https://download.splunk.com/products/splunk/releases/<version>/linux/splunk-<version>-Linux-x86_64.tgz"
tar xvzf splunk.tgz -C /opt

초기 설정​

# Splunk 시작 (라이선스 동의)
/opt/splunk/bin/splunk start --accept-license

# 관리자 계정 설정
# Username: admin
# Password: <설정>

# 부팅 시 자동 시작
/opt/splunk/bin/splunk enable boot-start

# 웹 인터페이스 접속
# http://<server-ip>:8000

Docker 설치​

# Docker 실행
docker run -d -p 8000:8000 -p 8089:8089 \
  -e SPLUNK_START_ARGS="--accept-license" \
  -e SPLUNK_PASSWORD="<password>" \
  --name splunk \
  splunk/splunk:latest

핵심 개념​

데이터 구조​

주요 용어​

용어설명
Event하나의 로그 레코드
Index이벤트 저장소 (DB 테이블과 유사)
Source데이터 출처 (파일 경로 등)
Sourcetype데이터 형식 (syslog, json 등)
Host데이터 발생 호스트
Field이벤트 내 추출된 키-값
AppSplunk 기능 확장 패키지

인덱스 구조​

# 기본 인덱스
main        # 기본 데이터 인덱스
_internal   # Splunk 내부 로그
_audit      # 감사 로그
summary     # 요약 인덱스

# 인덱스 생성
Settings → Indexes → New Index

SPL (Search Processing Language)​

기본 검색​

# 모든 이벤트 검색
index=main

# 키워드 검색
index=main error

# 필드 검색
index=main status=500

# 시간 범위 지정
index=main earliest=-24h latest=now

# 와일드카드
index=main source="/var/log/*"

비교 연산자​

연산자설명예시
=같음status=200
!=다름status!=200
<, >미만/초과bytes>1000
<=, >=이하/이상bytes>=1000
LIKE패턴 매칭user LIKE "admin%"
IN포함status IN (200, 301, 302)

논리 연산자​

# AND (기본값, 생략 가능)
index=main error AND status=500
index=main error status=500

# OR
index=main status=200 OR status=301

# NOT
index=main NOT status=200

# 괄호로 그룹화
index=main (status=500 OR status=503) error

파이프라인 명령어​

# 기본 구조
index=main | command1 | command2 | command3

주요 명령어​

명령어용도예시
stats통계 계산stats count by status
table테이블 출력table _time, host, status
sort정렬sort -count
head/tail상위/하위 N개head 10
where조건 필터링where count>100
eval필드 계산/생성eval kb=bytes/1024
rex정규식 추출rex field=_raw "(?<ip>\d+\.\d+)"
rename필드 이름 변경rename src_ip AS source_ip
dedup중복 제거dedup host
top/rare빈도 분석top 10 status
timechart시간 기반 차트timechart count by host

검색 예제​

기본 검색​

# 최근 24시간 에러 로그
index=main sourcetype=syslog error earliest=-24h

# 특정 호스트의 로그
index=main host="webserver01"

# 상태 코드별 카운트
index=main sourcetype=access_combined
| stats count by status

# 상위 10개 IP
index=main sourcetype=access_combined
| top 10 clientip

보안 분석 검색​

로그인 실패 탐지​

# SSH 로그인 실패
index=main sourcetype=linux_secure "Failed password"
| stats count by src_ip, user
| where count > 5
| sort -count

# Windows 로그인 실패 (이벤트 ID 4625)
index=main sourcetype=WinEventLog EventCode=4625
| stats count by src_ip, user
| where count > 10

이상 트래픽 탐지​

# 대용량 데이터 전송
index=main sourcetype=firewall
| stats sum(bytes_out) as total_bytes by src_ip
| where total_bytes > 1000000000
| sort -total_bytes

# 비정상 포트 접근
index=main sourcetype=firewall dest_port!=80 dest_port!=443 dest_port!=22
| stats count by src_ip, dest_port
| where count > 100

악성코드 탐지​

# 의심스러운 프로세스 실행
index=main sourcetype=WinEventLog EventCode=4688
| search CommandLine="*powershell*" OR CommandLine="*cmd.exe*"
| stats count by Computer, CommandLine

# 레지스트리 변경
index=main sourcetype=WinEventLog EventCode=4657
| stats count by ObjectName, ProcessName

사용자 활동 분석​

# 비정상 시간대 로그인
index=main sourcetype=linux_secure "Accepted"
| eval hour=strftime(_time, "%H")
| where hour < 6 OR hour > 22
| table _time, user, src_ip

# 권한 상승 시도
index=main sourcetype=linux_secure "sudo"
| stats count by user, command
| sort -count

통계 및 시각화​

# 시간별 이벤트 카운트
index=main
| timechart span=1h count

# 소스별 이벤트 분포
index=main
| stats count by sourcetype
| sort -count

# 평균, 최대, 최소
index=main sourcetype=access_combined
| stats avg(response_time), max(response_time), min(response_time)

데이터 수집​

Universal Forwarder 설치​

# 설치 (Linux)
wget -O splunkforwarder.tgz "https://download.splunk.com/products/universalforwarder/releases/<version>/linux/splunkforwarder-<version>-Linux-x86_64.tgz"
tar xvzf splunkforwarder.tgz -C /opt

# 시작
/opt/splunkforwarder/bin/splunk start --accept-license

# 수신 서버 설정
/opt/splunkforwarder/bin/splunk add forward-server <indexer-ip>:9997

# 모니터링 추가
/opt/splunkforwarder/bin/splunk add monitor /var/log/

inputs.conf 설정​

# /opt/splunkforwarder/etc/system/local/inputs.conf

# 파일 모니터링
[monitor:///var/log/syslog]
disabled = false
sourcetype = syslog
index = main

# 특정 로그 파일
[monitor:///var/log/apache2/access.log]
disabled = false
sourcetype = access_combined
index = web

# Windows 이벤트 로그
[WinEventLog://Security]
disabled = false
index = windows

outputs.conf 설정​

# /opt/splunkforwarder/etc/system/local/outputs.conf

[tcpout]
defaultGroup = splunk_indexers

[tcpout:splunk_indexers]
server = indexer1:9997, indexer2:9997

Syslog 수신​

# Splunk에서 Syslog 수신 설정
Settings → Data inputs → UDP → New
Port: 514
Sourcetype: syslog
Index: main

알림 설정​

Alert 생성​

1. 검색 실행
2. Save As → Alert
3. 설정:
   - Title: 알림 이름
   - Trigger Conditions: 조건 (결과 수, 임계값 등)
   - Trigger Actions: 동작 (이메일, 웹훅 등)

Alert 조건 예시​

# 로그인 실패 5회 이상
index=main sourcetype=linux_secure "Failed password"
| stats count by src_ip
| where count >= 5

# 조건: Number of Results > 0
# 동작: Send email, Run a script

Webhook 알림​

1. Alert Actions → Add Actions → Webhook
2. URL: https://your-webhook-endpoint.com
3. Payload: JSON 형식

유용한 SPL 패턴​

필드 추출​

# 정규식으로 필드 추출
index=main
| rex field=_raw "user=(?<username>\w+)"

# JSON 필드 추출
index=main sourcetype=json
| spath input=_raw
| table user, action, timestamp

# 키-값 쌍 추출
index=main
| kv

룩업 활용​

# CSV 룩업
index=main
| lookup user_lookup username AS user OUTPUT department

# 자동 룩업 설정
Settings → Lookups → Lookup definitions

서브서치​

# 서브서치 사용
index=main
| search [search index=threat_intel | fields ip | rename ip AS src_ip]

# 결과 조인
index=main sourcetype=access_combined
| join clientip [search index=geoip | table ip, country]

매크로​

# 매크로 정의 (Settings → Advanced Search → Search Macros)
# 이름: get_errors
# 정의: index=main sourcetype=* error

# 매크로 사용
`get_errors` | stats count by sourcetype

빠른 명령어 참조​

자주 쓰는 검색​

# 최근 1시간 에러
index=main error earliest=-1h

# 상태 코드별 카운트
index=main | stats count by status | sort -count

# 시간별 이벤트 추이
index=main | timechart span=1h count

# 상위 소스 IP
index=main | top 10 src_ip

# 유니크 값 카운트
index=main | stats dc(user) as unique_users

보안 분석​

# 실패 로그인 IP
index=main "failed" OR "failure" | stats count by src_ip | where count>5

# 비정상 프로세스
index=main sourcetype=WinEventLog EventCode=4688 | rare CommandLine

# 데이터 유출 의심
index=main | stats sum(bytes_out) by src_ip | where 'sum(bytes_out)'>1000000000

관리 명령어 (CLI)​

# Splunk 시작/중지
/opt/splunk/bin/splunk start
/opt/splunk/bin/splunk stop
/opt/splunk/bin/splunk restart

# 상태 확인
/opt/splunk/bin/splunk status

# 라이선스 정보
/opt/splunk/bin/splunk list licenser-pools

# 인덱스 목록
/opt/splunk/bin/splunk list index

# 사용자 관리
/opt/splunk/bin/splunk add user <username> -role <role>

트러블슈팅 체크리스트​

검색 결과가 없음​

  • 시간 범위 확인 (Time Range Picker)
  • 인덱스 이름 확인
  • sourcetype 확인
  • 권한 확인 (해당 인덱스 접근 권한)

Forwarder 데이터 미수신​

  • Forwarder 상태 확인: ./splunk list forward-server
  • 네트워크 연결 확인: telnet <indexer> 9997
  • inputs.conf 설정 확인
  • 방화벽 포트 확인 (9997)

검색 속도 느림​

  • 검색 시간 범위 축소
  • 인덱스 지정 (index=...)
  • 필드 추출 최적화
  • Summary Indexing 활용

디스크 공간 부족​

  • 오래된 데이터 삭제/아카이브
  • 인덱스 보존 정책 설정
  • Frozen 경로 설정
  • 데이터 압축 설정 확인

라이선스 초과​

  • 일일 인덱싱 볼륨 확인
  • 불필요한 데이터 소스 비활성화
  • 데이터 필터링 설정
