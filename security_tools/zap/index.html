OWASP ZAP (Zed Attack Proxy)

오픈소스 웹 애플리케이션 보안 테스트 도구. 웹 취약점 자동 스캔, 수동 테스트, API 테스트를 지원하는 통합 보안 플랫폼.

개요​

ZAP이란?​

항목설명
정식 명칭OWASP Zed Attack Proxy
유형웹 애플리케이션 보안 스캐너 (DAST)
라이선스Apache 2.0 (오픈소스)
개발OWASP Foundation
주요 용도웹 취약점 스캔, 프록시, 퍼징, API 테스트
공식 사이트https://www.zaproxy.org/

핵심 기능​

설치​

Kali Linux (기본 설치됨)​

# 버전 확인
zap.sh -version

# 업데이트
sudo apt update && sudo apt install zaproxy

Docker 설치​

# 기본 실행
docker pull zaproxy/zap-stable
docker run -u zap -p 8080:8080 -i zaproxy/zap-stable zap.sh -daemon -host 0.0.0.0 -port 8080

# GUI 모드 (X11 필요)
docker run -u zap -p 8080:8080 -p 8090:8090 \
  -v $(pwd):/zap/wrk:rw \
  zaproxy/zap-stable zap-webswing.sh

Windows/Mac (수동 설치)​

# 다운로드 후 실행
# https://www.zaproxy.org/download/

# Mac (Homebrew)
brew install --cask owasp-zap

아키텍처 이해​

동작 원리​

주요 컴포넌트​

컴포넌트설명단축키
Sites Tree탐색된 사이트 구조 트리-
History모든 요청/응답 기록-
Active Scan능동적 취약점 스캔-
Spider자동 URL 크롤링-
Alerts발견된 취약점 목록-
Request/Response상세 요청/응답 보기-

기본 사용법​

1. 프록시 설정​

ZAP 프록시 시작​

# GUI 모드
zap.sh

# Daemon 모드 (API 사용)
zap.sh -daemon -port 8080 -config api.key=your-api-key

브라우저 프록시 설정​

프록시 주소: localhost (또는 127.0.0.1)
포트: 8080 (기본값)

Firefox 설정 경로:

Settings → Network Settings → Manual proxy configuration
HTTP Proxy: 127.0.0.1, Port: 8080
SSL Proxy: 127.0.0.1, Port: 8080

SSL 인증서 설치​

# ZAP CA 인증서 내보내기
# Tools → Options → Dynamic SSL Certificates → Save

# 브라우저에 인증서 가져오기
# Firefox: Settings → Privacy & Security → Certificates → Import

2. 수동 탐색 (Manual Explore)​

1. 브라우저에서 대상 웹사이트 접속
2. 로그인, 폼 제출 등 모든 기능 사용
3. ZAP의 Sites Tree에서 탐색된 URL 확인
4. History 탭에서 모든 요청 확인

3. 자동 스파이더 (Spider)​

1. Sites Tree에서 대상 사이트 우클릭
2. Attack → Spider 선택
3. 옵션 설정:
   - Max Depth: 크롤링 깊이 (기본: 5)
   - Max Children: 하위 URL 최대 개수
4. Start Scan 클릭

Spider 옵션​

옵션설명권장값
Max Depth크롤링 깊이5-10
Max Children디렉토리당 최대 URL100
Thread Count동시 요청 수5
Parse CommentsHTML 주석 파싱On
Parse robots.txtrobots.txt 파싱On

4. AJAX Spider​

JavaScript로 렌더링되는 SPA(Single Page Application) 크롤링:

1. Sites Tree에서 대상 사이트 우클릭
2. Attack → AJAX Spider 선택
3. 브라우저 선택 (Firefox/Chrome)
4. Start Scan 클릭

취약점 스캔​

Active Scan (능동 스캔)​

실제 공격 페이로드를 전송하여 취약점 탐지:

1. Sites Tree에서 대상 선택
2. Attack → Active Scan 선택
3. Policy 선택 또는 커스터마이징
4. Start Scan 클릭

Scan Policy 설정​

카테고리포함 취약점위험도
InjectionSQL Injection, Command Injection, XSSHigh
AuthenticationSession Fixation, Weak AuthHigh
Information DisclosureDirectory Listing, Error MessagesMedium
MisconfigurationMissing Headers, CORS IssuesLow-Medium

커스텀 스캔 정책 생성​

1. Analyze → Scan Policy Manager
2. Add 클릭
3. 필요한 테스트 항목 선택/해제
4. Threshold 설정 (Low/Medium/High)
5. Strength 설정 (Low/Medium/High/Insane)

Passive Scan (수동 스캔)​

트래픽 분석만으로 취약점 탐지 (공격 페이로드 없음):

자동으로 실행됨 (프록시 통과 시)
탐지 항목:
- 민감정보 노출
- 보안 헤더 누락
- 쿠키 설정 문제
- 정보 노출

고급 기능​

1. Fuzzer 사용​

특정 파라미터에 다양한 페이로드 주입:

1. History에서 요청 우클릭
2. Attack → Fuzz 선택
3. 퍼징할 위치 선택 (Highlight)
4. Add 클릭 → Payloads 추가
5. Start Fuzzer 클릭

내장 Wordlist 위치​

# ZAP 설치 디렉토리
/usr/share/zaproxy/fuzzers/

# 주요 리스트
dirbuster/  # 디렉토리/파일 목록
jbrofuzz/   # 다양한 퍼징 페이로드

커스텀 페이로드 추가​

1. Fuzz 창에서 Add 클릭
2. Type: File 선택
3. Select 클릭 → 워드리스트 파일 선택

2. 인증 설정 (Authentication)​

로그인이 필요한 사이트 스캔:

Form-based 인증​

1. Sites Tree에서 로그인 페이지 찾기
2. 로그인 요청 우클릭 → Flag as Context → New Context
3. Context 더블클릭 → Authentication 선택
4. Method: Form-based Authentication
5. Login Form Target URL 입력
6. Username/Password Parameter 설정
7. Login Indicator (로그인 성공 문자열) 설정

Session 관리​

1. Context → Session Management
2. Method: Cookie-based Session Management
3. 필요시 Session Token 파라미터 지정

3. API 스캔​

OpenAPI/Swagger 스캔​

1. Import → Import an OpenAPI Definition
2. OpenAPI URL 또는 파일 선택
3. Target URL 입력 (실제 API 서버)
4. Import 클릭
5. Active Scan 실행

GraphQL 스캔​

1. Import → Import a GraphQL Schema
2. Schema URL 입력
3. Import 후 Active Scan 실행

4. 스크립트 활용​

Zest 스크립트 (녹화)​

1. 요청 시퀀스 녹화
2. Sites → Record → Start Recording
3. 작업 수행
4. Stop Recording
5. 스크립트로 저장

Python 스크립트​

# standalone 스크립트 예시
# 모든 요청에 커스텀 헤더 추가

def sendingRequest(msg, initiator, helper):
    msg.getRequestHeader().setHeader("X-Custom-Header", "test-value")
    return msg

CLI / Automation​

ZAP CLI 기본 명령​

# Baseline Scan (빠른 스캔)
zap-baseline.py -t https://target.com -r report.html

# Full Scan (전체 스캔)
zap-full-scan.py -t https://target.com -r report.html

# API Scan
zap-api-scan.py -t https://target.com/api/v1/openapi.json -f openapi -r report.html

Baseline Scan 옵션​

zap-baseline.py -t <target> [options]

# 주요 옵션
-t    # Target URL
-r    # HTML 리포트 파일명
-w    # Markdown 리포트 파일명
-x    # XML 리포트 파일명
-J    # JSON 리포트 파일명
-a    # Alpha 룰 포함
-d    # Debug 모드
-P    # 프록시 포트 지정
-D    # Delay (초)
-i    # Info 레벨 Alert도 포함
-j    # AJAX Spider 사용
-m    # 최대 실행 시간 (분)

Full Scan 옵션​

zap-full-scan.py -t <target> [options]

# 추가 옵션
-a    # Spider/AJAX Spider 스킵
-s    # Safe 모드 (덜 공격적)

Docker 자동화​

# Baseline Scan
docker run -v $(pwd):/zap/wrk:rw -t zaproxy/zap-stable \
  zap-baseline.py -t https://target.com -r report.html

# Full Scan with Authentication
docker run -v $(pwd):/zap/wrk:rw -t zaproxy/zap-stable \
  zap-full-scan.py -t https://target.com \
  -r report.html \
  -z "-config auth.loginurl=https://target.com/login \
      -config auth.username=testuser \
      -config auth.password=testpass"

API 사용​

# API Key 확인
# Tools → Options → API → API Key

# 스캔 시작
curl "http://localhost:8080/JSON/ascan/action/scan/?url=https://target.com&apikey=YOUR-API-KEY"

# 스캔 상태 확인
curl "http://localhost:8080/JSON/ascan/view/status/?apikey=YOUR-API-KEY"

# 결과 가져오기
curl "http://localhost:8080/JSON/core/view/alerts/?apikey=YOUR-API-KEY"

트러블슈팅 체크리스트​

HTTPS 사이트 접근 불가​

  • ZAP CA 인증서가 브라우저에 설치되었는지 확인
  • Tools → Options → Dynamic SSL Certificates에서 인증서 재생성
  • 브라우저 캐시 및 쿠키 삭제

Spider가 제대로 동작하지 않음​

  • 로그인 필요 사이트인지 확인 (Authentication 설정)
  • JavaScript 렌더링 필요시 AJAX Spider 사용
  • robots.txt로 차단된 경로인지 확인

스캔 속도가 느림​

  • Thread 수 조정 (Tools → Options → Active Scan)
  • Delay between requests 줄이기
  • 불필요한 스캔 정책 비활성화

오탐 (False Positive) 발생​

  • Alert 우클릭 → Mark as False Positive
  • Context 설정으로 특정 URL 제외
  • Scan Policy에서 해당 룰 Threshold 조정

API 연결 실패​

  • API Key 확인 (Tools → Options → API)
  • Daemon 모드로 실행되었는지 확인
  • 방화벽에서 포트 허용 확인
