실습 환경 구성

보안 관제 실습을 위한 VMware 네트워크 환경 구성 가이드입니다.

실습 환경 개요​

전체 구성도​

IP 대역 구성표​

가상머신IP 주소역할
Security Onion192.168.2.240IDS/IPS (Suricata)
Splunk Server192.168.2.210SIEM 서버
Wazuh Server192.168.2.10HIDS 서버
Windows 10192.168.2.13클라이언트 / Agent

1. VMware 네트워크 설정​

NAT 네트워크 구성​

VMware에서 모든 VM이 같은 IP 대역을 사용하도록 NAT 설정을 변경합니다.

1.1 Virtual Network Editor 열기​

Edit → Virtual Network Editor → Change Settings (관리자 권한)

1.2 VMnet8 (NAT) 설정 변경​

Subnet IP 변경:

Subnet IP: 192.168.2.0
Subnet mask: 255.255.255.0

NAT Settings:

Gateway IP: 192.168.2.2

DHCP Settings:

Start IP: 192.168.2.200
End IP: 192.168.2.250
IP 대역 통일 이유

분석할 때 IP 주소가 다르면 헷갈릴 수 있어서 모두 같은 대역(192.168.2.x)으로 맞춥니다.

1.3 Windows 호스트 네트워크 어댑터 확인​

Win + R → ncpa.cpl

VMnet8 어댑터에서:

  • TCP/IPv4 더블클릭
  • IP 주소 자동 할당으로 설정 (또는 192.168.2.1 고정)
  • DNS: 8.8.8.8

2. 가상머신 네트워크 설정​

2.1 Linux VM 고정 IP 설정​

Security Onion 예시:

# 터미널 열기 (바탕화면 우클릭 → Open Terminal)
sudo -iㅌ

# IP 확인
ip addr
# 또는
ifconfig

네트워크 설정:

IP: 192.168.2.240
Subnet: 255.255.255.0
Gateway: 192.168.2.2
DNS: 8.8.8.8

2.2 Windows VM 고정 IP 설정​

Win + R → ncpa.cpl → 이더넷 더블클릭 → 속성 → TCP/IPv4

Windows 10 예시:

IP: 192.168.2.13
Subnet: 255.255.255.0
Gateway: 192.168.2.2
DNS: 8.8.8.8

2.3 연결 테스트​

각 VM에서 게이트웨이와 외부 연결 테스트:

# 게이트웨이 연결 확인
ping 192.168.2.2

# DNS 확인 (IP가 아닌 도메인으로)
ping google.com
게이트웨이 Ping 실패 시

게이트웨이까지 안 나가면 외부 통신은 당연히 안 됩니다:

  1. VM 네트워크 어댑터가 NAT인지 확인
  2. VMnet8 설정이 올바른지 확인
  3. IP 대역이 맞는지 확인

3. 필수 가상머신 준비​

3.1 Security Onion 16​

특징:

  • Ubuntu 기반에 보안 도구가 미리 설치됨
  • Suricata, Snort 등 IDS 포함
  • 별도 OS 설치 없이 바로 사용 가능

VM 생성:

File → New Virtual Machine → Custom
- Guest OS: Linux, Ubuntu 64-bit
- Name: Security Onion
- Processors: 4 cores
- Memory: 8 GB
- Network: NAT
- Disk: 100 GB

설치 후 계정:

Username: suguil (또는 설정한 계정)
Password: 123456

3.2 Metasploitable 2​

특징:

  • 의도적으로 취약하게 만든 서버
  • 웹 해킹, 서비스 취약점 테스트용
  • 절대 인터넷에 노출시키면 안 됨

기본 계정:

Username: msfadmin
Password: msfadmin

IP 확인:

ifconfig
# 또는
ip addr

3.3 Kali Linux​

특징:

  • 모의해킹 전문 배포판
  • Metasploit, Nmap, Nikto 등 도구 포함

기본 계정:

Username: kali
Password: kali

3.4 Ubuntu Forwarder​

용도:

  • Splunk Universal Forwarder 설치
  • 로그 전송 테스트

필수 패키지:

sudo apt update
sudo apt install -y nginx  # 웹 로그 테스트용

4. 네트워크 진단 명령어​

Windows​

# IP 설정 확인
ipconfig /all

# 게이트웨이 연결 테스트
ping 192.168.2.2

# DNS 테스트
nslookup google.com

Linux​

# IP 확인
ip addr
ifconfig

# 게이트웨이 테스트
ping -c 4 192.168.2.2

# DNS 테스트
ping -c 4 google.com

# 라우팅 테이블 확인
route -n
ip route

5. 원격 접속 도구​

TeamViewer (팀뷰어)​

실습 중 문제가 있을 때 원격 지원을 받을 수 있습니다.

설치:

검색창 → teamviewer → 다운로드 → 설치

주의사항:

  • TeamViewer가 실행 중이면 인터넷 뱅킹이 안 될 수 있음
  • 백도어처럼 원격 접속을 허용하므로 사용 후 종료 권장
  • 비밀번호 공유 후 반드시 변경

빠른 참조​

네트워크 설정 체크리스트​

# 1. VMware NAT 설정
Subnet: 192.168.2.0/24
Gateway: 192.168.2.2
DHCP: 192.168.2.200 ~ 250

# 2. 각 VM 설정
Security Onion: 192.168.2.240
Splunk: 192.168.2.210
Wazuh: 192.168.2.10
Win10: 192.168.2.13
Kali: 192.168.2.210
Meta: 192.168.2.209
Ubuntu: 192.168.2.213

# 3. 모든 VM 공통
Subnet: 255.255.255.0
Gateway: 192.168.2.2
DNS: 8.8.8.8

트러블슈팅​

증상원인해결
Ping이 안 나감네트워크 어댑터 설정NAT 모드인지 확인
게이트웨이만 안 됨IP 대역 불일치VMnet8 설정 확인
DNS만 안 됨DNS 설정 누락8.8.8.8 설정
VM간 통신 안 됨방화벽방화벽 임시 비활성화
