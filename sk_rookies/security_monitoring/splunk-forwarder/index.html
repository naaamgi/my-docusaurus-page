Splunk Forwarder 연동

Ubuntu와 Windows에 Universal Forwarder를 설치하고 Splunk 서버로 로그를 전송하는 실습입니다.

실습 구성​

1. Splunk 서버 준비​

1.1 수신 포트 확인​

Splunk 서버에서 9997 포트가 열려있는지 확인:

# Splunk 서버에서
sudo /opt/splunk/bin/splunk display listen

# 또는
netstat -tlnp | grep 9997

포트가 안 열려있다면:

sudo /opt/splunk/bin/splunk enable listen 9997

1.2 인덱스 생성 (선택사항)​

웹 UI에서:

Settings → Indexes → New Index

권장 인덱스:

  • linux_logs: Linux 시스템 로그
  • web_logs: Nginx/Apache 로그
  • firewall_logs: 방화벽 로그

2. Linux Universal Forwarder​

2.1 Forwarder 다운로드​

Ubuntu Forwarder(192.168.2.213)에서:

# 작업 디렉토리
mkdir -p ~/splunk && cd ~/splunk

# 다운로드
wget -O splunkforwarder.deb "https://download.splunk.com/products/universalforwarder/releases/9.0.0/linux/splunkforwarder-9.0.0-linux-2.6-amd64.deb"

2.2 설치​

# 설치
sudo dpkg -i splunkforwarder.deb

# 설치 위치: /opt/splunkforwarder

2.3 초기 설정​

# 첫 시작 및 라이선스 동의
sudo /opt/splunkforwarder/bin/splunk start --accept-license

# 관리자 계정 생성
# Username: admin
# Password: 12345678

2.4 서버 연결 설정​

# Splunk 서버 추가
sudo /opt/splunkforwarder/bin/splunk add forward-server 192.168.2.210:9997

# 인증 정보 입력
# admin / 12345678

2.5 모니터링 대상 추가​

인증 로그 (auth.log):

sudo /opt/splunkforwarder/bin/splunk add monitor /var/log/auth.log \
  -index main \
  -sourcetype linux_secure

Nginx 액세스 로그:

# Nginx 설치 (없으면)
sudo apt install -y nginx

# 로그 모니터링 추가
sudo /opt/splunkforwarder/bin/splunk add monitor /var/log/nginx/access.log \
  -index main \
  -sourcetype nginx_access

시스템 로그:

sudo /opt/splunkforwarder/bin/splunk add monitor /var/log/syslog \
  -index main \
  -sourcetype syslog

2.6 부팅 자동 시작​

sudo /opt/splunkforwarder/bin/splunk enable boot-start

2.7 확인​

# 전송 서버 확인
sudo /opt/splunkforwarder/bin/splunk list forward-server

# 모니터링 목록
sudo /opt/splunkforwarder/bin/splunk list monitor

# 서비스 상태
sudo /opt/splunkforwarder/bin/splunk status

3. Windows Universal Forwarder​

3.1 다운로드​

Windows 10(192.168.2.13)에서:

  1. splunk.com 접속
  2. Windows 64-bit 다운로드
  3. 또는 PowerShell:
# 다운로드 (관리자 PowerShell)
Invoke-WebRequest -Uri "https://download.splunk.com/products/universalforwarder/releases/9.0.0/windows/splunkforwarder-9.0.0-x64-release.msi" -OutFile "splunkforwarder.msi"

3.2 설치​

GUI 설치:

  1. MSI 파일 더블클릭
  2. 라이선스 동의
  3. 관리자 계정 생성: admin / 12345678
  4. Receiving Indexer: 192.168.2.210:9997
  5. 설치 완료

무인 설치 (CLI):

msiexec /i splunkforwarder.msi RECEIVING_INDEXER="192.168.2.210:9997" AGREETOLICENSE=yes /quiet

3.3 이벤트 로그 모니터링​

관리자 PowerShell에서:

# Splunk Forwarder 디렉토리 이동
cd "C:\Program Files\SplunkUniversalForwarder\bin"

# Windows Security 이벤트 로그 추가
.\splunk add monitor "WinEventLog://Security" -index main

# Windows System 이벤트 로그
.\splunk add monitor "WinEventLog://System" -index main

# Windows Application 이벤트 로그
.\splunk add monitor "WinEventLog://Application" -index main

3.4 확인​

# 전송 서버 확인
.\splunk list forward-server

# 모니터링 목록
.\splunk list monitor

# 서비스 상태
.\splunk status

4. 로그 발생 테스트​

4.1 Linux에서 로그 발생​

SSH 로그인 실패 시뮬레이션:

# 잘못된 비밀번호로 로그인 시도 (다른 터미널에서)
ssh invaliduser@localhost

# auth.log 확인
sudo tail -f /var/log/auth.log

Nginx 접속 로그:

# 로그 삭제 (기존 로그 정리)
sudo sh -c '> /var/log/nginx/access.log'

# 접속 테스트 (호스트 PC에서)
# 브라우저: http://192.168.2.213

# 로그 확인
sudo tail -f /var/log/nginx/access.log

4.2 공격 트래픽 시뮬레이션​

PowerShell 스크립트 (호스트 PC에서):

# 디렉토리 탐색 공격 시뮬레이션
$target = "http://192.168.2.213"

# 공격 코드 1: 일반 페이지 요청
for ($i = 1; $i -le 100; $i++) {
    Invoke-WebRequest -Uri "$target/" -ErrorAction SilentlyContinue
}

# 공격 코드 2: 없는 경로 요청 (404 발생)
$paths = @("/admin", "/backup", "/config", "/test", "/.env", "/wp-admin")
foreach ($path in $paths) {
    for ($i = 1; $i -le 20; $i++) {
        Invoke-WebRequest -Uri "$target$path" -ErrorAction SilentlyContinue
    }
}

# 공격 코드 3: SQL Injection 시도
$sqli = @("' OR '1'='1", "'; DROP TABLE users;--", "1; SELECT * FROM users")
foreach ($payload in $sqli) {
    Invoke-WebRequest -Uri "$target/search?q=$payload" -ErrorAction SilentlyContinue
}
로그 발생 팁
  • 포트폴리오용으로 하루 정도 스크립트를 돌리면 충분한 로그가 쌓임
  • 로그 삭제: > /var/log/nginx/access.log (꺽쇠 하나면 덮어쓰기)

5. Splunk에서 로그 확인​

5.1 검색​

Splunk 웹 UI에서:

# 모든 로그
index=main

# Linux 인증 로그
index=main sourcetype=linux_secure

# Nginx 로그
index=main sourcetype=nginx_access

# Windows Security 로그
index=main sourcetype=WinEventLog:Security

5.2 시간 필터​

# 최근 15분
index=main earliest=-15m

# 오늘
index=main earliest=@d

# 특정 시간대
index=main earliest="01/22/2026:09:00:00" latest="01/22/2026:18:00:00"

5.3 분석 쿼리 예시​

404 에러 분석:

index=main sourcetype=nginx_access status=404
| stats count by uri
| sort -count
| head 10

로그인 실패 분석:

index=main sourcetype=linux_secure "Failed password"
| stats count by src_ip
| sort -count

시간별 트래픽:

index=main sourcetype=nginx_access
| timechart span=1h count

6. 트러블슈팅​

6.1 연결 확인​

Forwarder에서:

# Linux
sudo /opt/splunkforwarder/bin/splunk list forward-server

# 연결 상태 확인
telnet 192.168.2.210 9997

Splunk 서버에서:

# 연결된 Forwarder 확인
sudo /opt/splunk/bin/splunk list forward-server

6.2 로그 확인​

Forwarder 로그:

# Linux
sudo tail -f /opt/splunkforwarder/var/log/splunk/splunkd.log
# Windows
Get-Content "C:\Program Files\SplunkUniversalForwarder\var\log\splunk\splunkd.log" -Tail 50

빠른 명령어 참조​

Linux Forwarder​

# 서버 추가
sudo /opt/splunkforwarder/bin/splunk add forward-server 192.168.2.210:9997

# 모니터 추가
sudo /opt/splunkforwarder/bin/splunk add monitor /var/log/auth.log -index main

# 모니터 목록
sudo /opt/splunkforwarder/bin/splunk list monitor

# 모니터 삭제
sudo /opt/splunkforwarder/bin/splunk remove monitor /var/log/auth.log

# 재시작
sudo /opt/splunkforwarder/bin/splunk restart

Windows Forwarder​

# 경로 이동
cd "C:\Program Files\SplunkUniversalForwarder\bin"

# 서버 추가
.\splunk add forward-server 192.168.2.210:9997

# 이벤트 로그 추가
.\splunk add monitor "WinEventLog://Security"

# 목록 확인
.\splunk list monitor

# 재시작
.\splunk restart

SPL 검색​

# 소스타입별 검색
index=main sourcetype=linux_secure
index=main sourcetype=nginx_access
index=main sourcetype=WinEventLog:Security

# 필드 추출
| stats count by field
| timechart count
| top 10 field

트러블슈팅 체크리스트​

데이터가 안 들어올 때​

  • Splunk 서버 9997 포트 열려있는지 확인
  • Forwarder가 실행 중인지 확인
  • 서버 연결 설정 확인 (list forward-server)
  • 방화벽 설정 확인
  • 모니터링 파일 경로 정확한지 확인
  • 파일 읽기 권한 확인

Forwarder 연결 안 될 때​

  • 네트워크 연결 확인 (ping)
  • 포트 연결 테스트 (telnet)
  • Splunk 서버 상태 확인
  • 인증 정보 확인

로그 파일 권한 문제​

# Linux에서 권한 확인
ls -la /var/log/auth.log

# Splunk 사용자에게 읽기 권한 부여
sudo chmod 644 /var/log/auth.log
