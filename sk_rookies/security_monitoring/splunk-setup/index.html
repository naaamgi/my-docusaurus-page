Splunk 설치 및 설정

Ubuntu 20.04에 Splunk Enterprise를 설치하고 기본 설정하는 가이드입니다.

Splunk 개요​

아키텍처​

구성 요소​

구성 요소역할
Search Head검색 및 대시보드
Indexer데이터 인덱싱 및 저장
Forwarder데이터 수집 및 전송
Splunk 비용
  • 현업에서 억대 단위 비용
  • 교육용 무료 라이선스 (500MB/일 제한)

1. 사전 준비​

1.1 시스템 요구사항​

항목최소 사양
OSUbuntu 20.04 LTS
CPU2 cores
RAM4 GB
Disk20 GB

1.2 호스트명 설정​

# 현재 호스트명 확인
hostname

# 호스트명 변경
sudo hostnamectl set-hostname splunk-server

# 확인
hostname

1.3 네트워크 설정​

# IP 확인
ip addr

# 정적 IP 설정 (Netplan)
sudo nano /etc/netplan/00-installer-config.yaml
network:
  ethernets:
    ens33:
      addresses:
        - 192.168.2.210/24
      gateway4: 192.168.2.2
      nameservers:
        addresses:
          - 8.8.8.8
  version: 2
# 네트워크 적용
sudo netplan apply

# 연결 테스트
ping -c 4 google.com

2. Splunk 설치​

2.1 패키지 다운로드​

# 작업 디렉토리 생성
mkdir -p ~/splunk && cd ~/splunk

# wget으로 다운로드 (버전 확인 후 변경)
wget -O splunk.deb "https://download.splunk.com/products/splunk/releases/9.0.0/linux/splunk-9.0.0-linux-2.6-amd64.deb"

또는 splunk.com에서 직접 다운로드

2.2 설치​

# deb 패키지 설치
sudo dpkg -i splunk.deb

# 설치 위치
# /opt/splunk

2.3 초기 설정​

# Splunk 시작 (최초 실행 시 설정)
sudo /opt/splunk/bin/splunk start --accept-license

# 관리자 계정 설정
# Username: admin
# Password: 12345678 (8자리 이상)

2.4 부팅 시 자동 시작​

# 자동 시작 설정
sudo /opt/splunk/bin/splunk enable boot-start

3. 웹 인터페이스 접속​

3.1 접속​

브라우저에서:

http://192.168.2.210:8000

로그인:

Username: admin
Password: 12345678

3.2 초기 화면 구성​

  • Search & Reporting: 검색 및 분석
  • Settings: 설정
  • Apps: 앱 관리

4. 인덱스 설정​

4.1 인덱스 개념​

인덱스설명
main기본 인덱스 (488GB 제한)
_internalSplunk 내부 로그
사용자 정의용도별 분리

4.2 새 인덱스 생성​

Settings → Indexes → New Index

설정:

Index Name: security_logs
Max Size: 500 MB
무료 라이선스 제한

무료 버전은 500MB까지로 표시되지만 실제로는 더 적을 수 있습니다.

5. 데이터 입력 설정​

5.1 수신 포트 설정​

Forwarder로부터 데이터를 받을 포트를 설정합니다.

방법 1: CLI

sudo /opt/splunk/bin/splunk enable listen 9997
# admin 비밀번호 입력

방법 2: 웹 UI

Settings → Forwarding and receiving → Configure receiving → New Receiving Port
Port: 9997

5.2 로컬 로그 모니터링​

Settings → Data inputs → Files & Directories → New Local File & Directory

예시: syslog 모니터링

File or Directory: /var/log/syslog
Source type: syslog
Index: main

6. SPL 기본 문법​

6.1 기본 검색​

# 모든 이벤트
index=main

# 최근 15분
index=main earliest=-15m

# 특정 시간
index=main earliest="01/20/2026:00:00:00" latest="01/20/2026:23:59:59"

6.2 필터링​

# 키워드 검색
index=main error

# 필드 검색
index=main sourcetype=syslog

# AND 조건
index=main error AND warning

# OR 조건
index=main error OR failed

# NOT 조건
index=main NOT debug

6.3 통계​

# 개수 세기
index=main | stats count

# 소스별 개수
index=main | stats count by source

# 상위 10개
index=main | top 10 source

# 유니크 값
index=main | stats dc(source)

6.4 시계열​

# 시간별 이벤트 수
index=main | timechart count

# 1시간 단위
index=main | timechart span=1h count

# 소스별 시계열
index=main | timechart count by source

7. 대시보드 생성​

7.1 새 대시보드 생성​

Search & Reporting → Dashboards → Create New Dashboard

설정:

Title: Security Dashboard
Permissions: Shared in App

7.2 패널 추가​

  1. Edit 클릭
  2. Add Panel 클릭
  3. 검색 쿼리 입력
  4. 시각화 유형 선택 (차트, 테이블 등)

예시 패널:

이벤트 수 (Single Value):

index=main | stats count

소스별 분포 (Pie Chart):

index=main | stats count by source

시간별 추이 (Line Chart):

index=main | timechart span=1h count

8. 서비스 관리​

8.1 Splunk 서비스 명령어​

# 시작
sudo /opt/splunk/bin/splunk start

# 중지
sudo /opt/splunk/bin/splunk stop

# 재시작
sudo /opt/splunk/bin/splunk restart

# 상태 확인
sudo /opt/splunk/bin/splunk status

8.2 systemd 서비스 (부팅 자동 시작 후)​

sudo systemctl start splunk
sudo systemctl stop splunk
sudo systemctl restart splunk
sudo systemctl status splunk

빠른 명령어 참조​

설치 및 설정​

# 설치
sudo dpkg -i splunk.deb

# 첫 시작
sudo /opt/splunk/bin/splunk start --accept-license

# 부팅 자동 시작
sudo /opt/splunk/bin/splunk enable boot-start

# 수신 포트 열기
sudo /opt/splunk/bin/splunk enable listen 9997

SPL 기본​

# 기본 검색
index=main

# 시간 필터
earliest=-1h
earliest=-24h@h

# 필드 검색
sourcetype=syslog
source="/var/log/*"

# 통계
| stats count
| stats count by field
| top 10 field

# 시계열
| timechart count
| timechart span=1h count by source

주요 경로​

# Splunk 홈
/opt/splunk

# 설정 파일
/opt/splunk/etc/system/local/

# 로그
/opt/splunk/var/log/splunk/

# 앱
/opt/splunk/etc/apps/

트러블슈팅 체크리스트​

웹 UI 접속 안 될 때​

  • Splunk 서비스 실행 중인지 확인
  • 8000 포트 열려있는지 확인 (netstat -tlnp | grep 8000)
  • 방화벽 설정 확인 (sudo ufw status)
  • IP 주소 올바른지 확인

데이터가 안 들어올 때​

  • 9997 포트 열려있는지 확인
  • Forwarder 설정 확인
  • 인덱스 설정 확인
  • 권한 문제 확인

라이선스 경고​

  • 일일 500MB 제한 확인
  • 인덱스 크기 확인
  • 오래된 데이터 삭제 고려
