Wazuh Agent 등록

Windows와 Linux에 Wazuh Agent를 설치하고 서버에 등록하는 가이드입니다.

Agent vs Agentless​

방식AgentAgentless
설치클라이언트에 프로그램 설치설치 없음 (SSH)
모니터링실시간주기적 (스케줄)
기능전체 기능제한적 (무결성 검사 등)
용도일반 서버/워크스테이션네트워크 장비, 제한 환경

1. Windows Agent 설치​

1.1 대시보드에서 설치 명령 확인​

  1. Wazuh 대시보드 접속: https://192.168.2.10
  2. Agents 메뉴 클릭
  3. Deploy new agent 클릭
  4. Windows 선택
  5. Server address 입력: 192.168.2.10

생성된 PowerShell 명령어:

Invoke-WebRequest -Uri https://packages.wazuh.com/4.x/windows/wazuh-agent-4.7.0-1.msi -OutFile ${env:tmp}\wazuh-agent.msi; msiexec.exe /i ${env:tmp}\wazuh-agent.msi /q WAZUH_MANAGER='192.168.2.10' WAZUH_AGENT_GROUP='default'

1.2 Windows에서 설치​

관리자 권한 PowerShell:

# 다운로드 및 설치
Invoke-WebRequest -Uri https://packages.wazuh.com/4.x/windows/wazuh-agent-4.7.0-1.msi -OutFile ${env:tmp}\wazuh-agent.msi

msiexec.exe /i ${env:tmp}\wazuh-agent.msi /q WAZUH_MANAGER='192.168.2.10' WAZUH_AGENT_GROUP='default'
설치 실패 시

명령어 설치가 안 되면 MSI 파일을 직접 더블클릭해서 GUI로 설치합니다:

  1. C드라이브에서 "wazuh-agent" 검색
  2. MSI 파일 더블클릭
  3. 설치 진행

1.3 서비스 시작​

# 서비스 시작
NET START WazuhSvc

# 또는 서비스 관리자에서
# services.msc → Wazuh → 시작

1.4 등록 확인​

Wazuh 대시보드:

Agents → 에이전트 목록에서 Windows 확인

Windows에서:

# 에이전트 상태 확인
& "C:\Program Files (x86)\ossec-agent\agent-control.exe" -s

2. Linux Agent 설치​

2.1 대시보드에서 명령 확인​

  1. Deploy new agent 클릭
  2. DEB amd64 선택 (Ubuntu)
  3. Server address: 192.168.2.10

2.2 Ubuntu에서 설치​

# GPG 키 추가
curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | sudo apt-key add -

# 저장소 추가
echo "deb https://packages.wazuh.com/4.x/apt/ stable main" | sudo tee /etc/apt/sources.list.d/wazuh.list

# 패키지 업데이트 및 설치
sudo apt update
sudo apt install -y wazuh-agent

# 또는 직접 다운로드
curl -so wazuh-agent.deb https://packages.wazuh.com/4.x/apt/pool/main/w/wazuh-agent/wazuh-agent_4.7.0-1_amd64.deb
sudo dpkg -i wazuh-agent.deb

2.3 서버 설정​

# ossec.conf에서 서버 주소 설정
sudo nano /var/ossec/etc/ossec.conf
<ossec_config>
  <client>
    <server>
      <address>192.168.2.10</address>
    </server>
  </client>
</ossec_config>

2.4 서비스 시작​

# 서비스 시작
sudo systemctl daemon-reload
sudo systemctl enable wazuh-agent
sudo systemctl start wazuh-agent

# 상태 확인
sudo systemctl status wazuh-agent

3. Agentless 설정​

3.1 SSH 키 설정 (서버에서)​

# Wazuh 서버에서 SSH 키 생성
sudo -u ossec ssh-keygen -t rsa -N "" -f /var/ossec/.ssh/id_rsa

# 대상 서버에 공개키 복사
sudo -u ossec ssh-copy-id admin@192.168.2.100

3.2 Agentless 호스트 등록​

# register_host.sh 스크립트 사용
sudo /var/ossec/agentless/register_host.sh add admin@192.168.2.100 NOPASS

3.3 ossec.conf에 설정 추가​

sudo nano /var/ossec/etc/ossec.conf
<ossec_config>
  <agentless>
    <type>ssh_integrity_check_linux</type>
    <frequency>36000</frequency>
    <host>admin@192.168.2.100</host>
    <state>periodic</state>
  </agentless>
</ossec_config>

3.4 서비스 재시작​

sudo systemctl restart wazuh-manager

4. Windows Sysmon 연동​

4.1 Sysmon 개요​

Sysmon은 Windows에서 상세한 시스템 활동을 로깅하는 도구입니다.

로깅 항목:

  • 프로세스 생성/종료
  • 네트워크 연결
  • 파일 생성 시간 변경
  • 드라이버/DLL 로드
  • 레지스트리 변경

4.2 Sysmon 설치​

다운로드:

https://docs.microsoft.com/en-us/sysinternals/downloads/sysmon

설치:

# 다운로드 후 압축 해제
# 관리자 PowerShell에서

# 기본 설치
.\Sysmon64.exe -accepteula -i

# 설정 파일 사용 설치
.\Sysmon64.exe -accepteula -i sysmonconfig.xml

4.3 Wazuh에서 Sysmon 로그 수집​

ossec.conf 수정 (Windows Agent):

notepad "C:\Program Files (x86)\ossec-agent\ossec.conf"
<ossec_config>
  <localfile>
    <location>Microsoft-Windows-Sysmon/Operational</location>
    <log_format>eventchannel</log_format>
  </localfile>
</ossec_config>

서비스 재시작:

NET STOP WazuhSvc
NET START WazuhSvc

4.4 Sysmon 이벤트 확인​

대시보드에서:

Agents → Windows Agent → Security Events

필터:

data.win.system.channel:Microsoft-Windows-Sysmon/Operational

5. 에이전트 관리​

5.1 에이전트 목록 확인​

CLI:

# Wazuh 서버에서
sudo /var/ossec/bin/agent_control -l

대시보드:

Agents → 전체 목록 확인

5.2 에이전트 상태 확인​

# 특정 에이전트 상태
sudo /var/ossec/bin/agent_control -i 001

# 모든 에이전트 상태
sudo /var/ossec/bin/agent_control -l

5.3 에이전트 삭제​

# 에이전트 삭제
sudo /var/ossec/bin/manage_agents -r 001

6. 대시보드에서 이벤트 분석​

6.1 보안 이벤트 확인​

Agents → (에이전트 선택) → Security Events

6.2 필터 사용​

룰 레벨별:

rule.level:>=7

이벤트 타입별:

rule.groups:authentication_failed
rule.groups:sysmon
rule.groups:windows

시간별:

  • 우측 상단 시간 선택기 사용
  • Today, Last 24 hours 등

6.3 주요 필드​

필드설명
rule.id룰 ID
rule.level심각도 (0-15)
rule.description룰 설명
agent.name에이전트 이름
data.srcip출발지 IP
data.dstip목적지 IP

빠른 명령어 참조​

Windows Agent​

# 설치
msiexec.exe /i wazuh-agent.msi /q WAZUH_MANAGER='192.168.2.10'

# 서비스 시작/중지
NET START WazuhSvc
NET STOP WazuhSvc

# 설정 파일
C:\Program Files (x86)\ossec-agent\ossec.conf

# 로그 위치
C:\Program Files (x86)\ossec-agent\ossec.log

Linux Agent​

# 설치
sudo dpkg -i wazuh-agent.deb

# 서비스 관리
sudo systemctl start wazuh-agent
sudo systemctl stop wazuh-agent
sudo systemctl status wazuh-agent

# 설정 파일
/var/ossec/etc/ossec.conf

# 로그
/var/ossec/logs/ossec.log

Wazuh Server​

# 에이전트 목록
sudo /var/ossec/bin/agent_control -l

# 에이전트 상태
sudo /var/ossec/bin/agent_control -i 001

# Agentless 등록
sudo /var/ossec/agentless/register_host.sh add user@host NOPASS

# 서비스 재시작
sudo systemctl restart wazuh-manager

트러블슈팅 체크리스트​

Agent가 등록 안 될 때​

  • Wazuh 서버 IP 올바른지 확인
  • 1514, 1515 포트 열려있는지 확인
  • 방화벽 설정 확인
  • Agent 서비스 실행 중인지 확인
  • Agent 로그 확인 (ossec.log)

Agent가 Disconnected일 때​

  • 네트워크 연결 확인
  • Agent 서비스 상태 확인
  • 서버 연결 설정 확인
  • 인증 키 문제 확인

Sysmon 로그 안 올 때​

  • Sysmon 서비스 실행 중인지 확인
  • ossec.conf에 Sysmon 설정 있는지 확인
  • Agent 재시작 했는지 확인
  • 이벤트 뷰어에서 Sysmon 로그 생성되는지 확인

Agentless 연결 안 될 때​

  • SSH 키 복사 완료됐는지 확인
  • SSH 비밀번호 없이 로그인 가능한지 테스트
  • ossec 사용자 권한 확인
  • 대상 서버 SSH 서비스 실행 확인

다음 단계​

Agent 설정이 완료되면:

  1. 대시보드에서 실시간 보안 이벤트 모니터링
  2. 커스텀 룰 작성으로 탐지 정확도 향상
  3. 알림 설정으로 중요 이벤트 즉시 통보
  4. 프로젝트에 활용할 보안 관제 시나리오 설계
