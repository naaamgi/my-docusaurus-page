Wazuh 설치

Ubuntu 20.04에 Wazuh 서버를 올인원으로 설치하는 가이드입니다.

Wazuh 개요​

구성 요소​

주요 기능​

구성요소역할
Wazuh Manager에이전트 관리, 룰 처리
Wazuh Indexer로그 저장 및 검색 (Elasticsearch)
Wazuh Dashboard웹 UI (Kibana)
Filebeat로그 전송

1. 사전 준비​

1.1 시스템 요구사항​

항목최소 사양권장 사양
CPU2 cores4 cores
RAM4 GB8 GB
Disk20 GB50 GB
OSUbuntu 20.04Ubuntu 20.04

1.2 호스트명 설정​

# 호스트명 변경
sudo hostnamectl set-hostname wazuh-server

# 확인
hostname

1.3 네트워크 설정​

# IP 확인
ip addr

# 정적 IP 설정 (예: 192.168.2.10)
sudo nano /etc/netplan/00-installer-config.yaml
network:
  ethernets:
    ens33:
      addresses:
        - 192.168.2.10/24
      gateway4: 192.168.2.2
      nameservers:
        addresses:
          - 8.8.8.8
  version: 2
# 적용
sudo netplan apply

# 연결 테스트
ping -c 4 google.com

2. Wazuh 올인원 설치​

2.1 설치 스크립트 다운로드​

# 작업 디렉토리 생성
mkdir -p ~/wazuh && cd ~/wazuh

# 설치 스크립트 다운로드
curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh

2.2 올인원 설치 실행​

# 설치 실행 (시간 소요)
sudo bash wazuh-install.sh -a

설치되는 구성요소:

  • Wazuh Indexer (Elasticsearch)
  • Wazuh Manager
  • Wazuh Dashboard (Kibana)
  • Filebeat
설치 시간

네트워크 속도에 따라 10-30분 소요됩니다.

2.3 설치 완료 후 정보​

설치 완료 시 다음 정보가 표시됩니다:

Wazuh dashboard URL: https://192.168.2.10
Username: admin
Password: [자동 생성된 복잡한 비밀번호]
비밀번호 저장

자동 생성된 비밀번호를 반드시 기록해두세요!

3. 비밀번호 변경 (선택)​

실습 편의를 위해 간단한 비밀번호로 변경합니다.

3.1 비밀번호 변경 도구 다운로드​

cd ~/wazuh

# 비밀번호 변경 도구 다운로드
curl -so wazuh-passwords-tool.sh https://packages.wazuh.com/4.7/wazuh-passwords-tool.sh

3.2 비밀번호 변경​

# 실행 권한 부여
chmod +x wazuh-passwords-tool.sh

# 비밀번호 변경
sudo ./wazuh-passwords-tool.sh -u wazuh -p 123456

3.3 서비스 재시작​

sudo systemctl restart wazuh-manager
sudo systemctl restart wazuh-indexer
sudo systemctl restart wazuh-dashboard

4. 웹 대시보드 접속​

4.1 접속​

브라우저에서:

https://192.168.2.10

인증서 경고:

  • "고급" 클릭
  • "안전하지 않음 - 계속" 클릭

로그인:

Username: wazuh
Password: 123456 (또는 자동 생성된 비밀번호)

4.2 대시보드 구성​

주요 메뉴:

메뉴설명
Agents에이전트 목록 및 관리
Security Events보안 이벤트
Integrity monitoring파일 무결성 모니터링
Vulnerabilities취약점 탐지
Settings설정

5. 서비스 관리​

5.1 서비스 상태 확인​

# Wazuh Manager
sudo systemctl status wazuh-manager

# Wazuh Indexer (Elasticsearch)
sudo systemctl status wazuh-indexer

# Wazuh Dashboard (Kibana)
sudo systemctl status wazuh-dashboard

# Filebeat
sudo systemctl status filebeat

5.2 서비스 관리 명령어​

# 시작
sudo systemctl start wazuh-manager

# 중지
sudo systemctl stop wazuh-manager

# 재시작
sudo systemctl restart wazuh-manager

# 부팅 자동 시작 활성화
sudo systemctl enable wazuh-manager

6. 로그 확인​

6.1 Wazuh Manager 로그​

# 매니저 로그
sudo tail -f /var/ossec/logs/ossec.log

# 알림 로그
sudo tail -f /var/ossec/logs/alerts/alerts.log

# 아카이브 로그
sudo tail -f /var/ossec/logs/archives/archives.log

6.2 Wazuh Dashboard (Kibana) 검색​

기본 검색:

rule.level:>=7

시간대 설정:

  1. 우측 상단 시간 선택기 클릭
  2. "Today" 또는 원하는 범위 선택

7. 자체 모니터링 확인​

Wazuh 서버는 자기 자신도 모니터링합니다.

7.1 서버 이벤트 확인​

대시보드에서:

Agents → (wazuh-server 클릭) → Security Events

7.2 이벤트 필터링​

# 특정 룰 레벨 이상
rule.level:>=7

# 특정 그룹
rule.groups:authentication_failed

# 특정 에이전트
agent.name:wazuh-server

8. 주요 설정 파일​

8.1 파일 위치​

파일경로
Manager 설정/var/ossec/etc/ossec.conf
룰 파일/var/ossec/ruleset/rules/
에이전트 설정/var/ossec/etc/shared/
로그/var/ossec/logs/

8.2 ossec.conf 구조​

<ossec_config>
  <!-- 글로벌 설정 -->
  <global>
    <logall>yes</logall>
  </global>

  <!-- 알림 설정 -->
  <alerts>
    <log_alert_level>3</log_alert_level>
  </alerts>

  <!-- 에이전트리스 설정 -->
  <agentless>
    <type>ssh_integrity_check_linux</type>
    <frequency>36000</frequency>
    <host>admin@192.168.2.100</host>
  </agentless>
</ossec_config>

빠른 명령어 참조​

설치​

# 스크립트 다운로드
curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh

# 올인원 설치
sudo bash wazuh-install.sh -a

# 비밀번호 변경
sudo ./wazuh-passwords-tool.sh -u wazuh -p 123456

서비스 관리​

# 전체 재시작
sudo systemctl restart wazuh-manager
sudo systemctl restart wazuh-indexer
sudo systemctl restart wazuh-dashboard

# 상태 확인
sudo systemctl status wazuh-manager

로그 확인​

# Manager 로그
sudo tail -f /var/ossec/logs/ossec.log

# 알림 로그
sudo tail -f /var/ossec/logs/alerts/alerts.log

에이전트 관리​

# 에이전트 목록
sudo /var/ossec/bin/agent_control -l

# 에이전트 상태
sudo /var/ossec/bin/agent_control -i 001

트러블슈팅 체크리스트​

웹 대시보드 접속 안 될 때​

  • wazuh-dashboard 서비스 실행 중인지 확인
  • HTTPS로 접속했는지 확인 (http가 아님)
  • 443 포트 열려있는지 확인
  • 방화벽 설정 확인

설치 실패​

  • 인터넷 연결 확인
  • 디스크 공간 확인 (최소 20GB)
  • 메모리 확인 (최소 4GB)
  • 기존 Elasticsearch/Kibana 충돌 확인

비밀번호 분실​

# 비밀번호 재설정
sudo ./wazuh-passwords-tool.sh -u wazuh -p [새비밀번호]

# 서비스 재시작
sudo systemctl restart wazuh-manager
sudo systemctl restart wazuh-dashboard
