ZAP 웹 스캐닝

OWASP ZAP을 활용하여 Metasploitable 웹 서버의 취약점을 스캔하는 실습입니다.

실습 구성도​

핵심 개념:

  • ZAP이 브라우저와 웹서버 사이에서 프록시 역할
  • 모든 트래픽을 인터셉트하여 분석
  • Security Onion에서 스캔 트래픽 탐지

1. ZAP 설치 및 설정​

1.1 Java 설치 (필수)​

ZAP은 Java 기반이므로 먼저 Java를 설치합니다.

  1. Java 다운로드: Oracle JDK 또는 OpenJDK
  2. 설치 후 환경변수 설정:
    • JAVA_HOME: Java 설치 경로
    • Path에 %JAVA_HOME%\bin 추가

확인:

java -version
Java 경로 오류

ZAP이 실행 안 되면 Java 경로 설정이 잘못된 것입니다. 경로에 오타가 있으면 찾는게 어려우니 다시 설정하는게 빠릅니다.

1.2 ZAP 설치​

  1. zaproxy.org 에서 Windows 버전 다운로드
  2. 설치 진행 (기본 옵션)
  3. 실행 후 업데이트 확인

1.3 폰트 설정 (한글 깨짐 방지)​

Tools → Options → Display → Font Size

적당한 크기로 설정 후 ZAP 재시작

2. 프록시 설정​

2.1 ZAP 프록시 포트 확인​

Tools → Options → Network → Local Servers/Proxies

기본 설정:

Address: localhost
Port: 8080

2.2 Windows 프록시 설정​

방법 1: 인터넷 옵션

Win + R → inetcpl.cpl → 연결 → LAN 설정

설정:

☑ 프록시 서버 사용
주소: localhost (또는 127.0.0.1)
포트: 8080

방법 2: 설정 앱

설정 → 네트워크 → 프록시 → 수동 프록시 설정
프록시 설정 확인

프록시 설정 후 브라우저에서 웹사이트 접속 시 ZAP에 트래픽이 표시되면 정상입니다.

3. Add-ons 업데이트​

3.1 업데이트 확인​

Help → Check for Updates

3.2 스캔 관련 Add-ons 설치​

Installed 탭: 이미 설치된 것 중 업데이트 가능한 것

Marketplace 탭: 추가 설치 가능한 것

권장 설치 목록:

  • Active scanner rules
  • Passive scanner rules
  • Spider
  • Ajax Spider
  • Directory List (dirbuster)
  • SQL Injection Scanner
Add-on 설치 팁
  • 스캔이라는 글자가 있으면 체크
  • Spider 관련 기능 체크
  • 한 번에 10개 이상 설치하면 오래 걸림

4. Metasploitable 스캔 실습​

4.1 대상 확인​

# Metasploitable IP 확인
# msfadmin / msfadmin 로그인 후
ip addr
# 예: 192.168.2.209

4.2 브라우저에서 접속​

프록시 설정된 브라우저에서:

http://192.168.2.209

ZAP 확인:

  • 하단 Sites 패널에 192.168.2.209 추가됨
  • History 탭에 요청/응답 기록

4.3 Spider 실행 (사이트맵 수집)​

  1. Sites에서 대상 사이트 우클릭
  2. Attack → Spider 선택
  3. 완료까지 대기

Spider 결과:

  • 사이트의 모든 링크 수집
  • 숨겨진 페이지 발견
  • 디렉토리 구조 파악

4.4 Active Scan 실행​

  1. Sites에서 대상 사이트 우클릭
  2. Attack → Active Scan 선택
  3. 스캔 완료까지 대기 (시간 소요)

Active Scan 특징:

  • 실제 공격 페이로드 전송
  • SQL Injection, XSS 등 테스트
  • 타겟에 로그가 대량 발생

5. 결과 분석​

5.1 Alerts 탭 확인​

발견된 취약점이 심각도별로 표시됩니다:

아이콘심각도설명
🔴High즉시 조치 필요
🟠Medium빠른 조치 권장
🟡Low개선 권장
🔵Informational참고 정보

5.2 주요 취약점 확인​

Metasploitable에서 발견되는 주요 취약점:

취약점설명
SQL InjectionDB 쿼리 조작 가능
XSS스크립트 삽입 가능
Directory Listing디렉토리 목록 노출
Outdated Software구버전 소프트웨어

5.3 보고서 생성​

Report → Generate Report

형식 선택:

  • HTML (웹 보기용)
  • XML (데이터 처리용)
  • Markdown (문서화용)

6. Security Onion에서 탐지 확인​

6.1 스캔 트래픽 확인​

Security Onion 터미널에서:

sudo tail -f /var/log/suricata/fast.log

예상 탐지 내용:

  • SQL Injection 시도
  • Directory Traversal
  • 비정상적인 HTTP 요청

6.2 Sguil에서 이벤트 분류​

ZAP 스캔으로 인한 이벤트:

  • CAT VI: 정찰/스캐닝으로 분류
  • 다수의 HTTP 요청 기록
  • 공격 시도 패턴 탐지

7. 인터셉트 기능 활용​

7.1 요청 가로채기​

  1. Break 버튼 활성화 (녹색 원)
  2. 브라우저에서 요청 발생
  3. ZAP에서 요청 내용 확인/수정
  4. Forward 버튼으로 전송

7.2 파라미터 수정​

# 원본 요청
GET /login.php?user=admin&pass=1234

# 수정 후
GET /login.php?user=admin'--&pass=anything
실습 환경에서만

인터셉트 및 수정 기능은 허가된 테스트 환경에서만 사용하세요. 실제 서비스에 사용하면 불법입니다.

8. phpMyAdmin 접근 테스트​

8.1 phpMyAdmin 접속​

http://192.168.2.209/phpMyAdmin/

기본 계정:

Username: root
Password: (빈 값)

8.2 ZAP에서 인터셉트​

phpMyAdmin 페이지 접근 시:

  • 로그인 요청 캡처
  • 세션 쿠키 확인
  • 인증 우회 테스트

빠른 명령어 참조​

프록시 설정​

# 인터넷 옵션 열기
inetcpl.cpl

# 설정: 연결 → LAN 설정
# localhost:8080 프록시 설정

ZAP 주요 단축키​

단축키기능
Ctrl+Alt+SSpider 시작
Ctrl+Alt+AActive Scan
F11Break 토글
Ctrl+RResend 요청

스캔 순서​

1. 프록시 설정 확인
2. 대상 사이트 접속 (Sites에 추가)
3. Spider 실행 (사이트맵 수집)
4. Active Scan 실행 (취약점 탐지)
5. Alerts 확인 (결과 분석)
6. Report 생성 (보고서)

트러블슈팅 체크리스트​

ZAP이 실행 안 될 때​

  • Java 설치 확인 (java -version)
  • JAVA_HOME 환경변수 설정 확인
  • 경로에 한글/공백 있는지 확인

프록시가 안 잡힐 때​

  • ZAP 8080 포트 사용 중인지 확인
  • Windows 프록시 설정 확인
  • 방화벽 예외 추가

스캔이 느릴 때​

  • 동시 스레드 수 조정 (Options → Active Scan)
  • 불필요한 Add-on 비활성화
  • 대상 서버 상태 확인

인증서 오류​

  • ZAP 루트 인증서 설치
  • Tools → Options → Dynamic SSL Certificates
